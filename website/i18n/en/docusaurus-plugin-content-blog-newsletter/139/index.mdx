## ⚛️ React {#react}

[![Introducing react.dev](/emails/issues/139/reactdev.jpg)](https://react.dev/blog/2023/03/16/introducing-react-dev)

**[Présentation de react.dev](https://react.dev/blog/2023/03/16/introducing-react-dev)**

Le nouveau site React est enfin en ligne ! Un nouvel article accompagne ce lancement. C'est un projet très ambitieux (tant techniquement que par le contenu) qui a nécessité plusieurs années d'efforts et de nombreux collaborateurs.

Ce que j'apprécie particulièrement :

- L'accent mis sur les hooks et composants fonctionnels
- Les guides avec nombreux diagrammes, illustrations et défis interactifs
- La présentation de React comme solution cross-platform, pas seulement web
- Les pages API bien séparées (référence/usage, actuel/déprécié, client/serveur)
- La cohérence globale de l'expérience

Certains changements ont aussi fait débat cette semaine, notamment le fait de recommander uniquement une courte liste de frameworks pour démarrer un projet React (Next.js, Remix, Gatsby, Expo) sans recommander beaucoup d'autres (Vite, Create-React-App, SPA côté client...).

- 👀 [React PR - Suspend le commit sans bloquer le rendu](https://github.com/facebook/react/pull/26398) : une fonctionnalité qui sera utilisée en interne en premier, notamment pour les ressources (CSS, polices, images). Lors de la navigation avec startTransition, React pourra attendre un peu jusqu'à ce que les images de la page suivante soient chargées avant de valider la transition ([détails](https://twitter.com/acdlite/status/1637102880850059264)).
- 👀 [React PR - Supprimer l'avertissement layout effect côté serveur](https://github.com/facebook/react/pull/26395) : l'avertissement SSR pour useLayoutEffect sera supprimé. Dan Abramov commente sur [useIsomorphicLayoutEffect](https://twitter.com/dan_abramov/status/1636176793592905729).
- 🧵 [Tim Neutkens sur l'amélioration de `next export`](https://twitter.com/timneutkens/status/1636693508223270912) : compatible avec App Router, Server Components...
- 📜 [Future Proofing Your Remix App](https://remix.run/blog/future-flags) : Remix explique sa stratégie pour déployer des changements cassants lors des versions majeures. Ils prévoient d'introduire des "Future flags" permettant d'opter pour les changements cassants planifiés à l'avance. Si vous activez tous les flags en v1, la migration vers la v2 sera simple.
- 📜 [Internationalization In Next.js 13 With React Server Components](https://www.smashingmagazine.com/2023/03/internationalization-nextjs-13-react-server-components/) : Les Server Components auront un impact majeur sur la façon de traduire nos apps React, notamment sur les performances, le code splitting et la taille des bundles. Cet article présente bien le sujet. J'apporte des détails supplémentaires dans ce [thread](https://twitter.com/sebastienlorber/status/1638528549056311297).
- 📜 [Hooks You Probably Don't Need](https://reacttraining.com/blog/hooks-you-probably-dont-need) : présentation des hooks React moins connus, classés par probabilité d'utilité (bonjour `useInsertionEffect` 😅). J'ai aimé la dernière section présentant un anti-pattern courant et comment `useLayoutEffect` le prévient. Je pense que [useSyncExternalStore est génial](https://twitter.com/sebastienlorber/status/1615329010761842689) pour lire les APIs navigateur comme `localStorage`.
- 📜 [Front-Commerce is getting a Remix](https://developers.front-commerce.com/changelog/front-commerce-is-getting-a-remix) : après Shopify, c'est maintenant Front-Commerce qui décide de se baser sur Remix. Cela présente de nombreux avantages, comme la possibilité d'utiliser le progressive enhancement, un mode MPA, et un couplage moindre avec React.
- 📜 [Astro: How Good Is the New JavaScript Framework?](https://cult.honeypot.io/reads/astro-the-new-javascript-framework/)
- 📜 [Reducing 97% of the Open UI site's JavaScript by migrating to Astro](https://blog.andri.co/023-migrating-open-ui-site-from-gatsby-to-astro/)
- 📦 [React-Error-Boundary v4](https://twitter.com/sebastienlorber/status/1638465247886426113) : nouvelle API `useErrorBoundary` pour déclencher les limites d'erreur pour les erreurs non-render, et réinitialiser facilement la limite depuis n'importe où.
- 📦 [Wakuwork](https://github.com/dai-shi/wakuwork) : Daishi Kato travaille sur un framework React minimaliste avec support des React Server Components.
- 📦 [Framer-Motion 10.5](https://twitter.com/mattgperry/status/1636714500731592706) : `useAnimate()`
- 📦 [Remix-Sitemap](https://github.com/fedeya/remix-sitemap)
- 🎥 [Will React's New Cache Fix Its "Use" Hook?](https://www.youtube.com/watch?v=T3m-MZkuadU)
