## ⚛️ React {#react}

[![TanStack Router](/emails/issues/123/tanstack-router.png)](https://tanstack.com/router/v1)

**[TanStack Router](https://tanstack.com/router/v1)**

Tanner dévoile son nouveau projet : un routeur 100% typé et indépendant des frameworks. Une liste impressionnante de fonctionnalités inclut :

- Gestion d'état de l'URL
- Cache intégré
- Schéma/validation des paramètres de recherche
- Préchargement automatique
- Loaders/Actions
- Éléments asynchrones + Découpage de code

De prime abord, on y décèle des inspirations de Remix ou React-Router, mais l'API diffère sensiblement et n'est pas basée sur JSX, ce qui facilite probablement l'agnosticisme et la sécurité des types. Pour l'instant seul l'adaptateur React existe, et la documentation est incomplète, mais le projet semble très prometteur !

🔗 **Liens utiles** :

- 🐦 [Tanner Linsley](https://twitter.com/tannerlinsley/status/1590101866385489929) présente son projet sur Twitter avec une superbe bande-annonce !

[![Gatsby 5 : le Gatsby le plus rapide à ce jour](/emails/issues/123/gatsby5.png)](https://www.gatsbyjs.com/blog/gatsby-5/)

**[Gatsby 5 : le Gatsby le plus rapide à ce jour](https://www.gatsbyjs.com/blog/gatsby-5/)**

Gatsby 5.0 vient d'être officiellement publié. Il inclut l'API Gatsby Slice qui permet d'améliorer le temps de build lors de la modification d'un élément de mise en page (en-tête, pied de page...), et le support en version bêta des Server Components qui permettent une hydratation partielle. Il y a également de nouveaux composants Head/Script, GraphiQL v2, et plus encore.

Gatsby a également présenté [Valhalla](https://www.gatsbyjs.com/products/valhalla-content-hub/), un nouveau service sur leur cloud qui propose d'utiliser les plugins source de Gatsby pour créer un hub de contenu et une API GraphQL unifiée découplée de Gatsby. Il est [possible d'utiliser Valhalla avec Next.js](<https://github.com/gatsby-inc/valhalla-examples/blob/main/frameworks/nextjs/app/(pets)/page.tsx>).

Les générateurs de sites statiques ne sont pas très populaires actuellement, bien qu'ils conviennent encore à de nombreux sites. Je trouve positif qu'au moins un acteur majeur continue d'innover dans ce domaine. L'API Slice semble être une bonne idée. J'aurais souhaité que Valhalla puisse fonctionner localement, mais je pense que c'est un service exclusivement cloud.

## 📱 React-Native {#react-native}

[![Expo SDK 47](/emails/issues/123/expo47.png)](https://blog.expo.dev/expo-sdk-47-a0f6f5c038af)

**[Expo SDK 47](https://blog.expo.dev/expo-sdk-47-a0f6f5c038af)**

Le nouveau SDK 47 vient d'être officiellement publié, avec un article plus détaillé sur les nouveautés.

- React Native 0.70.5 et React 18.1.0
- Expo Modules API 1.0 permet de créer très facilement des modules natifs Fabric en Swift/Kotlin
- Hermes disponible sur Expo Go iOS, expérience de débogage simplifiée. Hermes sera le moteur par défaut dans le prochain SDK
- Prise en charge de Fabric dans plusieurs modules Expo
- Expo Router en bêta : routage basé sur le système de fichiers pour React-Navigation
- Diverses améliorations EAS : sécurité de bout en bout, workers M1...

En bref, une version très réussie ! Avec Expo Modules, il semble plus facile d'adopter la nouvelle architecture comparé aux outils officiels comme react-native-codegen qui ne prend pas encore en charge Swift.
