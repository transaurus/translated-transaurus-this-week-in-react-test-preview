## ⚛️ React {#react}

[![Bilan de la Next.js Conf 2024](/emails/issues/207/nextjsconf.jpg)](https://vercel.com/blog/recap-next-js-conf-2024)

[**Bilan de la Next.js Conf 2024**](https://vercel.com/blog/recap-next-js-conf-2024)

La semaine dernière, la communauté s'est réunie à la conférence Next.js. Ce bilan présente les points forts, notamment :

- Next.js 15 stable
- Turbopack Dev stable
- Mise en cache simplifiée, complexité réduite
- Améliorations de l'auto-hébergement et modèles

L'article de blog complémentaire [Notre parcours avec la mise en cache](https://nextjs.org/blog/our-journey-with-caching) est particulièrement intéressant, présentant de nouvelles sémantiques et API expérimentales de mise en cache comme `use cache`, `cacheTag` et `cacheLife`. C'est un système plus flexible, et particulièrement pratique car vous n'avez même pas besoin de fournir une clé de cache.

## 📱 React-Native {#react-native}

[![Nouvelle Architecture par défaut, DevTools et plus](/emails/issues/207/newarch.jpg)](https://reactnative.dev/blog/2024/10/23/release-0.76-new-architecture)

[**React Native 0.76 - Nouvelle Architecture par défaut, DevTools et plus**](https://reactnative.dev/blog/2024/10/23/release-0.76-new-architecture)

React Native 0.76 est sorti avec des mises à jour majeures ! La Nouvelle Architecture est désormais activée par défaut, un nouvel outil React Native DevTools améliore le débogage, et la prise en charge CSS pour `boxShadow` et `filter` permet des effets d'interface avancés (uniquement avec la nouvelle architecture). Attention aux changements cassants : les SDK minimum requis passent à iOS 15.1 et Android SDK 24 (Android 7). Il est temps de mettre à niveau et de l'essayer !

La v0.76 est cool, mais ce qui est encore mieux, c'est que la [Nouvelle Architecture est là](https://reactnative.dev/blog/2024/10/23/the-new-architecture-is-here) et officiellement déclarée prête pour la production ! Ce résultat couronne un effort de plusieurs années et constitue un jalon majeur pour l'écosystème React Native. Vous connaissez probablement déjà JSI, Fabric, Turbomodules, la communication synchrone sans pont à faible surcharge, le mode interop, etc. Au-delà des performances, cela résout des problèmes UX/DX cruciaux comme mesurer des vues synchronement pour positionner des infobulles ou faire fonctionner `useLayoutEffect` comme sur le web. Ce qui m'enthousiasme le plus, probablement le plus sous-estimé : React Native peut enfin utiliser les fonctionnalités concurrentes de React 18+, notamment les transitions, Suspense et les innovations à venir.
