## ⚛️ React {#react}

[![useSignal() is the Future of Web Frameworks](/emails/issues/135/signals.jpg)](https://www.builder.io/blog/usesignal-is-the-future-of-web-frameworks)

[**`useSignal()` est l'avenir des frameworks Web**](https://www.builder.io/blog/usesignal-is-the-future-of-web-frameworks)

Misko, créateur de Qwik et AngularJS, souligne l'intérêt des signaux et compare leur fonctionnement à celui de React. React utilise directement la valeur de l'état, tandis que les signaux utilisent un accesseur et créent des abonnements lors de son appel.

Les signaux permettent une optimisation automatique des ré-rendus avec une granularité fine, sans nécessiter de mémoïsation. Les composants intermédiaires faisant parfois uniquement du props drilling n'ont pas besoin de se réafficher.

Cet article est paru juste après l'[annonce d'Angular](https://twitter.com/angular/status/1625939902046117890) prévoyant d'utiliser les signaux (une [démo](https://twitter.com/sarah_edo/status/1628065696247857152) est désormais disponible).

Andrew Clark a [commenté](https://twitter.com/acdlite/status/1626590880126889984) l'introduction potentielle des signaux dans React. Il préfère actuellement miser sur le compilateur [React Forget](https://www.youtube.com/watch?v=lGEMwh32soc), dont nous devrions avoir des nouvelles via un article React Labs prochainement. Je trouve positif que React aille à contre-courant et continue d'[explorer sa direction actuelle](https://twitter.com/acdlite/status/1625599973420154881).

[![Announcing Sandpack 2.0 and a Node.js runtime for any browser](/emails/issues/135/sandpack2.jpg)](https://codesandbox.io/blog/announcing-sandpack-2)

**[Annonce de Sandpack 2.0 et d'un runtime Node.js pour n'importe quel navigateur](https://codesandbox.io/blog/announcing-sandpack-2)**

CodeSandbox a annoncé la nouvelle version de Sandpack, un outil qui permet de créer des expériences interactives de codage en direct dans le navigateur (éditeur de code + prévisualisation). Sandpack s'intègre bien avec React grâce à un package dédié. La v1 était déjà utilisée sur le nouveau site de documentation beta de React, par Josh Comeau et de nombreux autres.

Cette v2 est basée sur la nouvelle technologie NodeBox qui permet d'exécuter Node.js dans le navigateur. Il est possible d'exécuter Next.js, Vite, Astro et d'autres piles.

Cela rappelle la technologie de StackBlitz, qui vient d'[annoncer l'API WebContainers](https://blog.stackblitz.com/posts/webcontainer-api-is-here/). L'article de Sandpack explique les différences entre les technologies NodeBox et WebContainers.

Vous pouvez également **analyser les résultats des enquêtes** dans des tableaux de bord personnalisés, sauvegarder vos formulaires au format PDF, ou simplement **les convertir en fichiers PDF modifiables**. Il offre un support natif pour le **rendu React et Angular** et une totale liberté de choix quant au backend, car **toute combinaison serveur + base de données est entièrement compatible**.
