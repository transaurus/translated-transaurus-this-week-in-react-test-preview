## ⚛️ React {#react}

[![Les composants serveur React désormais dans RedwoodJS](/emails/issues/179/redwood.jpg)](https://redwoodjs.com/blog/rsc-now-in-redwoodjs)

[**Les composants serveur React désormais dans RedwoodJS**](https://redwoodjs.com/blog/rsc-now-in-redwoodjs)

Bighorn, la prochaine "ère" de Redwood démarrant en v8.0, misera entièrement sur les composants serveur React (RSC). Cet article explique pourquoi Redwood préfère désormais les RSC à GraphQL, qui n'offrait pas la meilleure expérience développeur (DX) pour démarrer la plupart des projets. GraphQL restera pris en charge et pourra être utilisé conjointement avec les RSC. Les Cellules Redwood permettent désormais d'exporter une nouvelle fonction "data" pour charger les données de la cellule.

Vous pouvez essayer Redwood Bighorn dès aujourd'hui en version canary v8, mais le billet de blog mentionne encore de nombreuses aspérités qui seront corrigées avant la sortie officielle majeure. Un autre article intéressant à lire [compare les conventions de routage de Redwood avec Next.js et Remix](https://redwoodjs.com/blog/whats-different-comparing-the-router-in-nextjs-app-api-nextjs-pages-api-remix-and-redwoodjs).

## 📱 React-Native {#react-native}

Cette section est désormais co-écrite avec [Benedikt](https://twitter.com/bndkt). N'hésitez pas à nous envoyer vos commentaires par email ou sur [Twitter](https://twitter.com/bndkt) !

[![React Navigation - Présentation de l'API Statique](/emails/issues/179/rnav.jpg)](https://reactnavigation.org/blog/2024/03/25/introducing-static-api)

[**React Navigation - Présentation de l'API Statique**](https://reactnavigation.org/blog/2024/03/25/introducing-static-api)

Sans doute l'une des bibliothèques React Native les plus utilisées et importantes, React Navigation voit sa version majeure actuelle (6) introduite il y a environ trois ans. Avec la v7 (actuellement en alpha), React Navigation réintroduit une API statique permettant de définir les écrans via une configuration objet. Cela permet l'inférence de types (si vous avez déjà créé des types pour un arbre React Navigation dynamique, vous savez le gain que cela représente) et facilite également la configuration des liens profonds (voire la rend partiellement automatique).

Consultez également le [guide de mise à niveau v7](https://reactnavigation.org/docs/7.x/upgrading-from-6.x/) pour connaître les autres changements à venir !
