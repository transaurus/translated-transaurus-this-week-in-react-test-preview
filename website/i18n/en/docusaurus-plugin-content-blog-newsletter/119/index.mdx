## ⚛️ React {#react}

**[Vercel OG Image Generation : Images de cartes sociales rapides et dynamiques à la périphérie](https://vercel.com/blog/introducing-vercel-og-image-generation-fast-dynamic-social-card-images)**

Vercel présente une solution très efficace pour générer des cartes sociales dynamiques dans les Edge Functions. Pratique : utilisation possible de JSX, composants React (sans état) et Tailwind sans configuration supplémentaire. Je recommande [ce fil](https://twitter.com/shuding_/status/1579607964549513217) expliquant son fonctionnement, très intéressant ! Ils utilisent même Yoga, le moteur de layout de React-Native 🤭

**[L'avenir du rendu dans React](https://prateeksurana.me/blog/future-of-rendering-in-react/)**

Article bien illustré présentant les modèles de rendu de React 18 et au-delà. Vous y trouverez des informations sur le fonctionnement du streaming SSR, Suspense, React lazy, et l'implémentation des feature flags avec les Server Components...

**[Surmonter les problèmes courants dans les projets React](https://dinosaurs-with-jetpacks.com/posts/react-problems)**

Encourage à découpler les applications JavaScript des frameworks (React ou autres). Prône un retour à plus de simplicité. Certaines positions radicales ne feront pas l'unanimité.

**[Material UI dans Storybook](https://storybook.js.org/blog/material-ui-in-storybook/)**

Tutoriel avec une section particulièrement intéressante expliquant comment automatiser la création des contrôles Storybook en réutilisant les types TypeScript de la bibliothèque UI sous-jacente. Très utile pour le transfert des props. Prend même en charge les helpers comme `Pick` et `Omit`.

- 👤 Sathya Gunasekaran (ex V8) [rejoint l'équipe React core](https://twitter.com/_gsathya/status/1578816177371250688), probablement pour [travailler sur le compilateur React-Forget](https://twitter.com/Huxpro/status/1578865666010263552)
- 👥 [Intervenants de Next.js Conf 2022](https://twitter.com/vercel/status/1578401243243741185)
- 📖 Docs bêta : [React.Children](https://twitter.com/dan_abramov/status/1578457525628047360) + [useId](https://twitter.com/dan_abramov/status/1577283961763467265) + [createElement](https://twitter.com/dan_abramov/status/1577463721193897985) + [useImperativeHandle](https://twitter.com/dan_abramov/status/1577717226269966336)
- 📖 [Améliorations de la documentation Next.js sur Vercel](https://twitter.com/leeerob/status/1577788730349436928)
- 🌶 [React et Next.js sont MORTS — Quelque chose de nouveau les remplace enfin (pour de bon)](https://javascript.plainenglish.io/react-and-next-js-is-dead-something-new-is-finally-replacing-it-for-good-c792c48806f6) : explique comment le framework Qwik présente un modèle supérieur à React. Titre racoleur, mais article très intéressant.
- 📜 [Créer un moteur de rendu React personnalisé - Partie 1](https://www.markcodes.dev/posts/build-react-custom-renderer-part-1) : nouvelle série montrant comment créer un moteur de rendu React. Pour l'instant facile à suivre, mais ne prend pas encore en charge les mises à jour.
- 📜 [Remix, React et l'état](https://blog.jim-nielsen.com/2022/remix-react-and-state/) : propose un modèle mental pour Remix qui réconcilie l'état React de manière fullstack.
- 📜 [Comment écrire sa propre bibliothèque de gestion d'état](https://judehunter.dev/blog/how-to-write-your-own-state-management-library) : utile pour comprendre le fonctionnement d'une bibliothèque de gestion d'état et implémenter un système de sélecteurs.
- 📜 [Comment récupérer des données dans React en pensant performance](https://www.developerway.com/posts/how-to-fetch-data-in-react) : explique pourquoi la récupération de données dans React n'est pas simple. La performance est liée à l'UX/perception. Détaille les waterfalls et limites de requêtes navigateur.
- 📜 [Éviter les états UI impossibles avec React, TypeScript et xState](https://whereisthemouse.com/avoid-impossible-ui-states-with-react-typescript-and-xstate) : les union-types discriminés de TypeScript évitent les états impossibles, et XState évite les transitions d'état impossibles.
- 📜 [J'ai changé d'avis sur l'écriture de nouveaux frameworks JavaScript](https://whitep4nth3r.com/blog/write-a-new-javascript-framework/) : seuls 4 frameworks supportent SSG + SSR : Eleventy, Next.js, Gatsby, Redwood.
- 📜 [Comment utiliser Sandpack pour des démos de code](https://www.ctnicholas.dev/articles/how-to-use-sandpack-for-code-demos) : article [mis à jour avec les dernières fonctionnalités Sandpack](https://twitter.com/ctnicholasdev/status/1577683748878884864), cet outil Codesandbox crée des playgrounds de code. Utilisé par la doc bêta de React, Josh Comeau...
- 📦 [Tremor 1.0](https://www.tremor.so/) : bibliothèque de composants React UI pour créer des tableaux de bord analytiques.
- 🐦 [W3C Trusted Types + React + XSS](https://twitter.com/sebastienlorber/status/1579485631260286977) : une nouvelle proposition HTML pourrait prévenir les vulnérabilités XSS nativement, rendant peut-être `dangerouslySetInnerHTML` obsolète ?
