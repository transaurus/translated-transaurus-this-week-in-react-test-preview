## ⚛️ React {#react}

[![Can React Server Actions finally fix forms?](/emails/issues/156/serveractions.jpg)](https://www.mux.com/blog/what-are-react-server-actions)

[**Les Server Actions de React peuvent-elles enfin résoudre les formulaires ?**](https://www.mux.com/blog/what-are-react-server-actions)

Une excellente introduction aux Server Actions, une fonctionnalité à venir de React déjà intégrée dans Next.js 13.4 (expérimental). Comprend un court tutoriel sur le téléchargement d'une vidéo avec titre. Les Server Actions peuvent renvoyer une réponse pour donner un retour à l'utilisateur, mais il est clair que ce n'est pas encore une expérience développeur idéale.

À mon avis, les Server Actions devraient viser à prendre pleinement en charge l'amélioration progressive. Cela nécessite probablement d'introduire une API "flash" pour gérer les retours sans aucun JavaScript côté client. Cela existe dans de nombreux frameworks backend, et Remix propose une API ([`session.flash()`](https://remix.run/docs/en/main/utils/sessions#sessionflashkey-value)). Information connexe qui pourrait vous intéresser : avec [`action.$$FORM_ACTION`](https://twitter.com/sebastienlorber/status/1689921046856474625), vous pouvez étendre le comportement d'amélioration progressive de React.
