## ⚛️ React {#react}

[![Astro 5.0](/emails/issues/212/astro.jpg)](https://astro.build/blog/astro-5/)

**[Astro 5.0](https://astro.build/blog/astro-5/)**

Une nouvelle version majeure d'Astro vient de sortir. Bien que techniquement ce ne soit pas un framework React pur, il est encore largement utilisé dans la communauté React, peut facilement intégrer des composants React interactifs et repose sur MDX. Les points forts sont :

- Content Layer pour rassembler tout votre contenu dans un magasin de données convivial et type-safe. Y compris depuis des CMS, des API et d'autres sources.
- Server Islands pour servir une réponse de page initiale aussi rapidement que possible, tout en conservant la possibilité d'entrelacer du contenu dynamique dans une mise en page principalement statique. Une alternative plus simple et assez intéressante aux React Server Components et au Prerendering Partiel de Next.js.
- Prerendering simplifié, supprimant l'option « hybride »
- Variables d'environnement type-safe
- Vite 6 avec la nouvelle API Environment
- Fonctionnalités expérimentales : recadrage d'image, mises en page d'images responsives, composants SVG

## 📱 React-Native {#react-native}

[![Radon IDE 1.0](/emails/issues/212/radon.jpg)](https://ide.swmansion.com/)

**[Radon IDE 1.0](https://ide.swmansion.com/)**

Software Mansion a créé une extension VSCode (prenant également en charge Cursor) qui transforme votre éditeur en un IDE complet pour React Native et Expo. Elle est désormais officiellement lancée en version stable. C'est un produit payant, mais compte tenu du prix abordable et des améliorations apportées à l'expérience développeur (DX), c'est une évidence qui soutient également les formidables efforts open source de l'entreprise. Jetez un œil à la [vidéo de lancement de Radon IDE](https://www.youtube.com/watch?v=07Un9EfE8D4) pour un aperçu rapide des fonctionnalités : panneau de simulateurs et réglages, débogage, inspection en un clic, navigation pilotée par URL, intégration de Storybook, et plus encore.
