## ⚛️ React {#react}

[![Bun 1.0](/emails/issues/159/bun.jpg)](https://bun.sh/blog/bun-v1.0)

[**Bun 1.0**](https://bun.sh/blog/bun-v1.0)

Impossible de passer à côté cette semaine : Bun vient de sortir en version stable v1.0. Cette nouvelle n'est pas directement liée à React, mais elle affectera sans aucun doute tous les développeurs React à un moment donné.

Bun est une chaîne d'outils complète pour JavaScript et TypeScript basée sur Webkit et écrite en Zig. C'est une alternative ultra-rapide à toute une série d'outils frontend que nous utilisons aujourd'hui : Node.js, npx, Babel, esbuild, swc, webpack, Jest, Vitest, npm, Yarn, pnpm... Les benchmarks sont extrêmement impressionnants et résolument réels.

Sa compatibilité avec les outils existants et sa prise en charge mixte ESM/CJS en font un remplacement facile à adopter. Bun est généralement capable d'exécuter votre code tel quel, sans aucune modification (sinon c'est [🐦 considéré comme un bug](https://twitter.com/jarredsumner/status/1700497869160268155)). Les frameworks React comme Next.js, Remix et Astro sont déjà pris en charge. L'utiliser en remplacement de npm/Yarn/pnpm ou de Jest/Vitest constitue probablement un bon point d'entrée avec un risque limité.

## 📱 React-Native {#react-native}

[![Static Hermes](/emails/issues/159/hermes.jpg)](https://speakerdeck.com/tmikov2023/static-hermes-react-native-eu-2023-announcement)

[**Static Hermes**](https://speakerdeck.com/tmikov2023/static-hermes-react-native-eu-2023-announcement)

Static Hermes a été annoncé lors de la React-Native EU la semaine dernière. Je n'y étais pas, donc j'essaie d'expliquer ce que j'ai compris en lisant les diapositives, car la vidéo n'est pas encore sur YouTube.

Static Hermes est un projet expérimental qui vous permet de compiler optionnellement une partie de votre code TypeScript (ou Flow) en code natif. Cela offre des gains de performance significatifs (x10-20) par rapport au mode habituel d'Hermes basé sur l'interprétation de bytecode. Il permet également une intégration facile avec les API natives directement depuis votre code TypeScript (FFI à coût nul, 15 à 80 fois plus rapide qu'un wrapper JSI).

Pour activer cela, TypeScript doit d'abord devenir un langage "sound". Static Hermes modifiera la sémantique de JavaScript pour garantir que les types déclarés correspondent aux types à l'exécution. Cette diapositive explique bien le concept :

[![Static Hermes rend TypeScript sound](/emails/issues/159/hermes2.jpg)](https://speakerdeck.com/tmikov2023/static-hermes-react-native-eu-2023-announcement?slide=14)

Une idée très intéressante qui pourrait peut-être un jour avoir un impact en dehors de l'écosystème React-Native ?
