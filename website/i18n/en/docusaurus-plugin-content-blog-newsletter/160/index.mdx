## ⚛️ React {#react}

[![Remix v2](/emails/issues/160/remix-v2.jpg)](https://remix.run/blog/remix-v2)

[**Remix v2**](https://remix.run/blog/remix-v2)

Comme prévu, Remix v2 devrait être une mise à jour relativement simple, car de nombreuses fonctionnalités v2 étaient déjà disponibles en v1 via des 'future flags' optionnels. Cet article donne un aperçu de tous les flags livrés en v1 et désormais activés par défaut en v2. Remix propose aussi des packages de rétrocompatibilité pour ceux préférant certaines conventions v1. Le support des React Server Components devrait arriver dans Remix v3.

## 📱 React-Native {#react-native}

[![Expo Router RFC - Routes API](/emails/issues/160/expo-api-routes.jpg)](https://blog.expo.dev/rfc-api-routes-cce5a3b9f25d)

[**Expo Router RFC - Routes API**](https://blog.expo.dev/rfc-api-routes-cce5a3b9f25d)

Je l'attendais avec impatience ! Expo se lance dans les fonctionnalités côté serveur en proposant de déployer des Routes API via une convention Expo Router. Vous pouvez déployer ces routes vous-même, et bientôt sur des hôtes populaires, mais il est clair que l'objectif à long terme est de suggérer d'héberger le runtime serveur dans EAS.

Cela marque le début d'une nouvelle ère pour Expo. Ayant résolu le problème du mobile natif, il est maintenant temps de faire du cross-platform une réalité grand public en comblant les lacunes de la prise en charge web avec le Server Side Rendering et les React Server Components.
