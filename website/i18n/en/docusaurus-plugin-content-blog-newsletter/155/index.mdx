## ⚛️ React {#react}

[![Outils de développement pour React Server Components](/emails/issues/155/rscdevtools.jpg)](https://www.alvar.dev/blog/creating-devtools-for-react-server-components)

[**Outils de développement pour React Server Components**](https://www.alvar.dev/blog/creating-devtools-for-react-server-components)

Suite à la sortie des React Server Components, Alvar a remarqué que la communication réseau entre client et serveur utilisait un format difficile à déchiffrer. Il a donc commencé à travailler sur un site web [RSC Parser](https://rsc-parser.vercel.app/) permettant d'inspecter plus facilement les payloads. Cela fonctionnait, mais l'expérience utilisateur n'était pas idéale. Depuis, il travaille sur une véritable extension de navigateur qui fonctionne un peu comme les Network DevTools de Chrome, avec une chronologie montrant les payloads au fil du temps et prenant même en charge le streaming avec Suspense. Cet outil lui a déjà permis de détecter des payloads trop volumineux sur son blog et de les optimiser.

C'est un outil prometteur, encore en développement et à suivre. L'extension n'est pas encore disponible dans les stores des navigateurs, mais le sera bientôt. Quelques démonstrations vidéo dans ce [fil Twitter](https://twitter.com/sebastienlorber/status/1689203357368365056).

[![Mise en cache dans Next.js](/emails/issues/155/cachinginnextjs.jpg)](https://nextjs.org/docs/app/building-your-application/caching)

[**Mise en cache dans Next.js**](https://nextjs.org/docs/app/building-your-application/caching)

Une nouvelle page de documentation Next.js fournit une explication complète du fonctionnement des 4 niveaux de cache, pourquoi ils existent et comment les contourner :

- Mémoïsation des requêtes
- Cache de données
- Cache de route complet
- Cache du routeur

Il n'est pas possible de contourner le cache du routeur : un problème mis en lumière dans [cette vidéo](https://www.youtube.com/watch?v=_yhSh4g0NSk).

[![Une alternative non-cloud à Google Forms qui a tout pour plaire](/emails/issues/155/surveyjs.jpg)](https://surveyjs.io/?utm_source=thisweekinreact&utm_medium=email)

**[Une alternative non-cloud à Google Forms qui a tout pour plaire](https://surveyjs.io/?utm_source=thisweekinreact&utm_medium=email)**

Si vous cherchez à mettre en place un système robuste de gestion de formulaires entièrement intégré à votre infrastructure informatique, qui vous permet de conserver un contrôle total sur le flux de données, sans passer des mois ni dépenser des milliers de dollars pour le déployer, alors SurveyJS est la solution idéale !

SurveyJS est une suite produit composée de quatre bibliothèques JavaScript open-source permettant de créer et modifier plusieurs formulaires dynamiques basés sur JSON via un créateur de formulaires par glisser-déposer, de les afficher dans votre application React et de stocker les données sensibles des enquêtes au sein de votre infrastructure interne. Vous pouvez ensuite générer vos enquêtes et formulaires personnalisés sous forme de fichiers PDF modifiables dans un navigateur et visualiser les résultats avec des graphiques et tableaux interactifs. [En savoir plus maintenant](https://surveyjs.io/?utm_source=thisweekinreact&utm_medium=email).
