[![Vercel réintègre tout le rendu edge dans Node.js](/emails/issues/183/r183.jpg)](https://x.com/leeerob/status/1780705942734331983)

🐦 [Vercel réintègre tout le rendu edge dans Node.js](https://x.com/leeerob/status/1780705942734331983)

Lee Rob, VP Produit de Vercel, a expliqué sur X que Vercel « réintègre tout le rendu edge dans Node.js ». Si vous utilisez Vercel (et probablement même si ce n'est pas le cas), vous savez que Vercel promeut le concept d'« edge » depuis longtemps. D'un côté, edge signifie que le calcul s'exécute près de l'emplacement de l'utilisateur accédant à un site hébergé sur Vercel. D'un autre côté, cela signifie aussi que ce calcul s'exécute dans un « runtime edge », basé sur V8 et non sur Node.js. Bien que le concept d'exécution proche des utilisateurs finaux semble séduisant, les avantages s'effacent rapidement si les données doivent être chargées depuis un emplacement central (ce qui est très souvent le cas). Parallèlement, les équipes de développement peinent souvent à adapter les charges de travail au sous-ensemble limité d'API disponibles dans le runtime edge. Vercel semble avoir conclu qu'au global, les défis pour les développeurs surpassent les avantages limités, et revient donc aux runtimes Node.js centralisés. J'apprécie leur transparence sur ce sujet ! Pour plus de détails, Theo a réalisé une [vidéo](https://youtu.be/lAGE-k1Zfrg?si=5SP4hyoFkixTsh-y) à ce sujet.

**[NLUX ― Nouvelle bibliothèque React pour simplifier l'ajout de fonctionnalités IA](https://nlux.dev/?c-twr2-h)**

[NLUX](https://nlux.dev?c-twr2-p1) est une bibliothèque React [open-source](https://github.com/nluxai/nlux) qui simplifie l'intégration de modèles de langage avancés (LLM) comme ChatGPT dans vos applications web. Avec quelques lignes de code, vous ajoutez des capacités conversationnelles IA.

Fonctionnalités clés 🌟 :

- **Créez des interfaces de chat IA en minutes** - Interfaces conversationnelles de qualité avec peu de code.
- **Composants React & Hooks** - `<AiChat />` pour l'UI et le hook `useChatAdapter` pour une intégration facile.
- **Adaptateurs flexibles** - Pour `ChatGPT`, `LangChain` et `HuggingFace`, avec une API pour **créer vos propres adaptateurs**.
- **Synchronisez l'état de l'UI** avec les backends IA de manière optimale.
- **Streaming Markdown** - Rendu du markdown généré par les LLM en temps réel.
- **Zéro dépendance** - Base de code légère.

Plus d'informations, exemples et documentation sur [nlux.dev](https://nlux.dev?c-twr2-p1)

## 📱 React-Native {#react-native}

[![React Native 0.74 publié, l'IDE React Native est désormais en bêta](/emails/issues/183/rn183.jpg)](https://reactnative.dev/blog/2024/04/22/release-0.74)

📦 [React Native 0.74 publié](https://reactnative.dev/blog/2024/04/22/release-0.74)

Cette semaine a vu la publication de React Native 0.74, une étape majeure ! Cette version rend le "mode sans pont" (bridgeless) par défaut pour les nouveaux projets utilisant la Nouvelle Architecture (dès la version 0.75, la Nouvelle Arch deviendra aussi le défaut). Les deux options restent désactivables, mais représentent clairement l'avenir de React Native. De nombreuses bibliothèques ont ajouté leur support ces derniers mois. Autre nouveauté : RN 0.74 intègre Yoga 3.0. Peu de changements cassants, mais consultez les [notes de version](https://reactnative.dev/blog/2024/04/22/release-0.74).

📦 [L'IDE React Native est désormais en bêta](https://ide.swmansion.com/)

Another exciting news this week is the closed beta for React Native IDE, developed by the team at Software Mansion. You can now sign up to get early access to the IDE. They also have a landing page up now, and the [docs](https://ide.swmansion.com/docs/getting-started) are publicly available for anyone curious about it.
