[![L'hydratation est un arbre, la reprise est une carte](/emails/issues/148/hydration.jpg)](https://www.builder.io/blog/hydration-tree-resumability-map)

**[L'hydratation est un arbre, la reprise est une carte](https://www.builder.io/blog/hydration-tree-resumability-map)**

Propose un modèle mental intéressant pour comprendre la différence entre l'hydratation (React) et la reprise (Qwik). Avec la reprise, tous les composants sont considérés comme statiques, et les gestionnaires d'événements sont le point d'entrée pour l'interactivité. Il n'est pas nécessaire de parcourir un arbre (`O(n)`) : avec la reprise, l'évolutivité est bonne, comme une recherche dans une table de hachage (`O(1)`).
