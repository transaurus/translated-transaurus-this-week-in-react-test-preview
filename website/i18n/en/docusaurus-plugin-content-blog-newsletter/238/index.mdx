Bonjour à tous !

Cette semaine est relativement calme dans l'écosystème React, mais nous avons tout de même divers articles de blog et versions intéressants. Peut-être aurons-nous bientôt des nouvelles passionnantes du [React Summit](https://reactsummit.com/) qui a lieu vendredi !

Côté React Native, tous les développeurs travaillent déjà sur la prise en charge d'iOS 26 Liquid Glass et d'autres éléments annoncés par Apple hier. Ce nouveau concept design est plutôt controversé et fait l'objet de nombreuses moqueries 😂. React Native 0.80 arrive bientôt, je garde cela pour plus tard, mais vous pouvez jeter un coup d'œil dans [la présentation d'Alex Hunt à App.js](https://www.youtube.com/live/UTaJlqhTk2g?t=6520s).

L'écosystème JS a été plutôt actif avec Oxlint 1.0 et diverses nouvelles intéressantes sur Node.js.

## ⚛️ React {#react}

- 💸 [Build professional design system in less than a day, using Hope AI, new model by Bit.Cloud.](https://bit.cloud/solutions/design-systems?utm_source=ThisWeekInReact&campaign=DesignSystem)
- 🦋 [React Conf \- First speakers announced](https://bsky.app/profile/react.dev/post/3lqxleaezcs2p): I guess we’ll have React Compiler news in October.
- 📣 [React Router Open Governance](https://remix.run/blog/rr-governance): React Router gets more community-driven, forming a Steering Committee initially composed of Remix team members. RFCs will follow stages inspired by the TC39 process. This also gives a sneak peek of what you can expect in v8. See also [video comments from Alem Tuzlak](https://www.youtube.com/watch?v=qh2TRkNnyVw).
- 📜 [How Imports Work in RSC](https://overreacted.io/how-imports-work-in-rsc/): Dan Abramov deep dives on module systems. The JS one creates singletons and is quite different from the C one, which creates duplication by default. RSCs permit creating a single program spanning across two computers, with two independent module systems. By the way, you can now have a [1:1 consulting call with Dan](https://overreacted.io/im-doing-a-little-consulting/).
- 📜 [Reactivity is easy](https://romgrk.com/posts/reactivity-is-easy/): Nice interactive article showing how to let cells of a data grid read from React context without useless re-renders.
- 📜 [useState should require a dependency array](https://bikeshedd.ing/posts/use_state_should_require_a_dependency_array/): Interesting interactive article advocating for `useState(initialState, deps)` with good arguments. I’m not sure I agree, though: using a `key` and splitting your UI into smaller components is probably enough, but curious to see what others think about this idea.
- 📜 [Beware The URL Type-Safety Iceberg](https://nuqs.47ng.com/blog/beware-the-url-type-safety-iceberg): The author of the `nuqs` library warns of the hidden dangers of search parameters. You’d rather design beautiful URLs, stay within size limits, deal with URL update rate limits, validate params, and be able to migrate old URLs to a new validation schema.
- 📜 [Use Async Local Storage to prevent props drilling in Next.js Route handlers](https://www.nico.fyi/blog/async-local-storage-to-prevent-props-drilling): It's like React Context but for Node.js functions.
- 📜 [How to Integrate Plausible Analytics in a Next.js App](https://mxd.codes/articles/how-to-integrate-plausible-analytics-in-a-next-js-app-without-getting-blocked): Proxying analytics API calls makes them less likely to trigger ad-blocking extensions.
- 📜 [How to Create Your Own Simple useState Hook](https://www.deepintodev.com/blog/how-to-create-your-own-simple-use-state-hook)
- 📜 [How React Suspense Works Under the Hood: Throwing Promises and Declarative Async UI](https://www.epicreact.dev/how-react-suspense-works-under-the-hood-throwing-promises-and-declarative-async-ui-plbrh)
- 📜 [Astro Integrations Explained](https://lou.gg/blog/astro-integrations-explained)
- 💸 [Build a working reward flow in minutes — with Dynamo](https://whitelabel-loyalty.com/products/dynamo/?utm_source=email&utm_medium=newsletter&utm_campaign=react)
- 📦 [TanStack Router 1.121](https://github.com/TanStack/router/discussions/2863#discussioncomment-13104960): Important milestone introducing a new `@tanstack/react-start` package. TanStack Start is now just a Vite plugin.
- 📦 [shadcn/ui \- June 2025 updates](https://ui.shadcn.com/docs/changelog): Upgrades `Calendar` to the latest `react-day-picker`, introduces a new CLI command to migrate imports to the new unified `radix-ui` package.
- 📦 [ForesightJS 2.0 \- Predict user intent to prefetch](https://foresightjs.com/): This library predicts mouse cursor and keyboard navigation to prefetch data earlier than mouse hover. Docs show how to integrate it with React Router and Next.js link components. TanStack will soon have a similar feature built in ([tweet](https://x.com/tannerlinsley/status/1908723776650355111)). This might also be useful in the future for pre-rendering with React `<Activity>`.
- 📦 [React Aria \- June 5 release \- Drag and drop support in React Aria Tree](https://react-spectrum.adobe.com/releases/2025-06-05.html)
- 📦 [Puck 0.19 \- Visual editor for React \- Slots API & performance gains](https://puckeditor.com/blog/puck-019)
- 📦 [Astro 5.9 \- Experimental CSP support, render Markdown in content loaders](https://astro.build/blog/astro-590/)
- 📦 [React Cosmos 7.0 \- Develop UI components in isolation \- Support React 19,](https://github.com/react-cosmos/react-cosmos/releases/tag/v7.0.0) [Next.js](http://Next.js) [15](https://github.com/react-cosmos/react-cosmos/releases/tag/v7.0.0)
- 🎥 [Youssef \- The Most Common React Query Mistake](https://www.youtube.com/watch?v=JVcZLEFaC2M)
- 🎥 [Jack Herrington \- Remix V3's Fatal Flaw](https://www.youtube.com/watch?v=5SPqO6NR_Bg)
- 🎥 [Theo \- Why Tech Companies Are Moving Off React](https://www.youtube.com/watch?v=HBpOzj-iBUg)
- 🎙️ [Syntax 908 \- Storybook Has Evolved w/ Jeppe Reinhold](https://syntax.fm/show/908/storybook-has-evolved-w-jeppe-reinhold)
