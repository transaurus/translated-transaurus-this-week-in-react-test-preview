## ⚛️ React {#react}

[![Wake up Remix](/emails/issues/236/remix.jpg)](https://remix.run/blog/wake-up-remix)

**[Wake up Remix - Le plan pour Remix v3](https://remix.run/blog/wake-up-remix)**

Cette semaine, vous avez peut-être entendu une rumeur selon laquelle [Remix abandonnerait React](https://x.com/AdamRackis/status/1925646909345058818). Ryan et Michael viennent de publier un article officiel décrivant l'avenir de Remix.

En bref, React Router continuera d'être entièrement maintenu et restera un choix sûr. En parallèle, Remix v3 sera un nouveau framework basé sur un fork de Preact, leur permettant de contrôler toute la pile.

Leur objectif avec Remix v3 est de « réinventer ce que peut être un framework web ». L'article décrit les principes directeurs de ce nouveau framework (optimisation pour les LLM, construction sur les API Web, composition...).

**[Démarrer avec Clerk Billing](https://go.clerk.com/KfnV7Rj)**

Apprenez à créer une expérience de facturation complète avec Clerk—sans interface personnalisée, webhooks ou gestion de session Stripe nécessaire.

- Créez une page de tarification avec le composant `<PricingTable />`
- Connectez-vous à Stripe — pas d'appels API manuels, pas de synchronisation manuelle
- Gérez les offres et les droits directement dans le tableau de bord Clerk
- L'assistant `has()` pour contrôler l'accès en fonction du forfait, des fonctionnalités et des permissions du client
- Gérez les données utilisateur et d'abonnement au même endroit

Activez la facturation, intégrez `<PricingTable/>` sur votre page `/pricing` et [commencez à monétiser votre application](https://go.clerk.com/LphCouG).
