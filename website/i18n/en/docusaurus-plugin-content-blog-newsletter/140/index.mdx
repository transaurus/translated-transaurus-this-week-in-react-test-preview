## 💸 Commanditaire {#sponsor1}

[![React-admin](/emails/issues/140/reactadmin.jpg)](https://marmelab.com/react-admin/?utm_source=newsletter&utm_term=this-week-in-react)

**[React-admin - Le framework open source pour les applications B2B](https://marmelab.com/react-admin/?utm_source=newsletter&utm_term=this-week-in-react)**

Les applications monopages (SPA) restent le meilleur choix pour créer des panneaux d'administration, des applications B2B et des systèmes ERP sur des API REST/GraphQL. [React-admin](https://marmelab.com/react-admin/?utm_source=newsletter&utm_term=this-week-in-react) est **le framework open source de référence pour construire des SPA**.

React-admin offre une bibliothèque de plus de 150 composants orientés métier. Ce n'est pas un kit d'interface : il exploite le meilleur kit d'interface pour React, MUI. React-admin booste la productivité des développeurs en assemblant les meilleures bibliothèques de leur catégorie, gérant toutes les tâches répétitives que vous ne devriez pas avoir à traiter.

Son point fort ? La meilleure expérience développeur que vous ayez jamais eue, soutenue par une documentation géniale et une communauté dynamique. [Et il ne faut que 30 minutes pour commencer](https://marmelab.com/react-admin/Tutorial.html) 🚀

React-admin est activement **maintenu depuis 5 ans**, et a déjà convaincu plus de 20 000 développeurs dans le monde. Venez l'essayer, c'est gratuit, pour toujours ! 😁

## ⚛️ React {#react}

[![React Labs](/emails/issues/140/reactlabs.jpg)](https://react.dev/blog/2023/03/22/react-labs-what-we-have-been-working-on-march-2023)

**[React Labs : Ce sur quoi nous avons travaillé – Mars 2023](https://react.dev/blog/2023/03/22/react-labs-what-we-have-been-working-on-march-2023)**

Une mise à jour officielle de la R&D autour de React qui aborde différents sujets :

- Composants Serveur React. Une belle implémentation est déjà disponible dans Next.js via App Router. Les RSC avec async/await devraient également pouvoir s'exécuter dans une application cliente. J'ai l'impression qu'ils veulent unifier les modèles côté client/serveur et je pense que [AsyncContext](https://twitter.com/sebastienlorber/status/1639183216799100928) sera génial pour ça !
- Chargement des ressources : intégration avec Suspense pour charger CSS, polices et images ([plus de détails](https://twitter.com/acdlite/status/1637102880850059264)).
- Métadonnées de document : support natif pour une API des métadonnées `<head>` avec support du streaming.
- Compilateur d'optimisation React : présenté non plus comme un compilateur de mémoïsation automatique, mais comme un compilateur de réactivité. [React Forget a atteint un jalon](https://twitter.com/en_JS/status/1641494974893281282), et ils se concentrent sur l'intégration.
- Rendu hors écran : une innovation que j'attends vraiment avec impatience, qui différencie React des autres frameworks. Meta teste le prérendu sur des apps React-Native avec des résultats positifs.
- Traçage des transitions : permet de détecter les transitions lentes.

## 📱 React-Native {#react-native}

- 📜 [Storybook pour React Native (6.5)](https://storybook.js.org/blog/storybook-for-react-native-6-5/) : excellente nouvelle pour les utilisateurs de Storybook sur React-Native. L'intégration rattrape Storybook web avec l'adoption de CSF, Controls, Args, et sera bientôt compatible avec Storybook 7.0. Grâce aux nouvelles fonctionnalités de Metro (`require.context`), la nécessité de générer un index de stories devrait bientôt disparaître.
- 📜 [Comment la migration de Redux vanilla à Redux Toolkit a amélioré la gestion d'état chez Shopify POS](https://shopify.engineering/react-redux-toolkit-migration) : retour d'expérience sur une migration progressive vers RTK, supprimant 3500 lignes.
- 👀 [RFC : APIs de traversée DOM et layout dans React Native](https://github.com/react-native-community/discussions-and-proposals/pull/607) : une nouvelle API avancée pourrait permettre des fonctionnalités comme `MutationObserver`, `IntersectionObserver`...
- 👀 [View Flattening sur iOS](https://twitter.com/SamuelSusla/status/1641086480675225601) : React-Native optimisait déjà sur Android en évitant de créer des vues natives inutiles. Samuel explique son fonctionnement et annonce que cela fonctionne désormais aussi sur iOS.
- 👀 [Démo Expo Router + Apple Handoff](https://twitter.com/Baconbrix/status/1641850192839180315) : innovation iOS pour basculer entre web et mobile, fonctionnant bien avec les apps universelles Expo Router 👌
- 🎥 [The Joy of Painting avec React Native](https://www.youtube.com/watch?v=GQqL1OCoOFM) : nouveau style de vidéo Skia ASMR par William Candillon 😄
- 📦 [React-Native WebView 12.0](https://twitter.com/titozzz/status/1642147926762815490) : support de la nouvelle architecture.
- 📦 [React-Native 0.76.1](https://github.com/facebook/react-native/releases/tag/v0.71.6) : nouvelles versions corrigeant les problèmes de build XCode.
