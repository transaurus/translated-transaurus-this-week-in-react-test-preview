## ⚛️ React {#react}

[![Sathya Gunasekaran - Quel avenir pour le Compilateur React ?](/emails/issues/208/compiler.jpg)](https://www.youtube.com/watch?v=qd5yk2gxbtg)

🎥 [**Sathya Gunasekaran \- Quel avenir pour le Compilateur React ?**](https://www.youtube.com/watch?v=qd5yk2gxbtg)

J'ai enfin pris le temps de regarder cette excellente conférence annonçant la [bêta du compilateur React](https://react.dev/blog/2024/10/21/react-compiler-beta-release), qui dévoile également d'autres aspects intéressants. Le compilateur ne se limite pas à la mémoïsation automatique, mais constitue une plateforme pour développer davantage d'optimisations. 3 fonctionnalités à venir ont été présentées :

- Configuration des types : Vous pouvez aider le compilateur à mieux fonctionner en fournissant des informations de type supplémentaires.
- Inlining JSX : Le compilateur peut supprimer les appels d'exécution JSX inutiles, les remplacer par des objets JSX préconstruits. `jsx('div')` devient `{type: 'div'}`.
- Outlining JSX : Le compilateur peut extraire automatiquement des sous-composants, qui peuvent ensuite être optimisés indépendamment. Ceci est particulièrement utile pour les éléments de liste lors de l'utilisation de `array.map` et de JSX intégré, le compilateur peut créer un composant `<Item>` (voir la capture d'écran du talk, sinon [cette PR](https://github.com/facebook/react/pull/30956) l'explique bien).

## 📱 React-Native {#react-native}

[![React Navigation 7.0](/emails/issues/208/reactnavigation7.jpg)](https://reactnavigation.org/blog/2024/11/06/react-navigation-7.0/)

[**React Navigation 7.0**](https://reactnavigation.org/blog/2024/11/06/react-navigation-7.0/)

React Navigation v7 est désormais disponible et devrait alimenter le prochain Expo Router v4 livré avec Expo SDK 52, tous deux en version bêta. Mes points forts préférés sont :

- Préchargement des écrans : Vous pouvez préparer les écrans à l'avance avant d'y naviguer et améliorer les performances perçues. Celui-ci est vraiment passionnant et pourrait utiliser les fonctionnalités Concurrent React à l'avenir.
- API statique : Simplifie la configuration du navigateur et facilite l'utilisation de TypeScript et du deep linking.

[**Nous venons de terminer la migration de l'application mobile Shopify vers React Native**](https://threadreaderapp.com/thread/1853619638141071573.html)

Mustafa (Directeur de l'ingénierie) a annoncé sur X que Shopify a terminé la migration de son application phare avec 586 écrans vers React Native, avec des résultats très positifs. J'espère que nous aurons bientôt un article de blog détaillé.

- 1,8 million de lignes de code supprimées
- 86 % de partage de code entre iOS et Android
- Chargement des écrans 59 % plus rapide
- Lancement de l'application 44 % plus rapide
- Webviews 63 % plus rapides
- Réduction des plantages d'application
- Amélioration des notations sur les stores d'applications
