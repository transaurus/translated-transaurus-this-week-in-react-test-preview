## React

[**Lire le code source : React**](https://alexkondov.com/readint-source-code-react/?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)

Alex lit le code source de React et commente plusieurs parties, en prenant comme point d'entrée le montage d'une application React. Détails intéressants sur l'utilisation des prototypes, du JSX et l'interaction du réconciliateur (agnostique) avec le moteur de rendu DOM.

[**Can We All Just Admit React Hooks Were a Bad Idea?**](https://medium.com/codex/can-we-all-just-admit-react-hooks-were-a-bad-idea-c48120c5188d?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)

Amy soutient que les hooks React ne respectent pas les principes d'architecture SOLID. Je ne suis pas vraiment d'accord, mais reconnais que la plupart des exemples trouvés sur internet sont simplifiés et ne mettent pas nécessairement en avant une bonne architecture.

[**Applying SOLID principles in React**](https://konstantinlebedev.com/solid-in-react/?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)

Presque une réponse directe à l'article précédent, même si l'application de SOLID est plus largement couverte ici (composants + hooks). Note : n'hésitez pas à utiliser le contexte React pour l'injection de dépendances.

## React-Native

[**Hermes as the Default**](https://reactnative.dev/blog/2022/07/08/hermes-as-the-default?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)

Le moteur JavaScript Hermes est une alternative à JSC/V8. Il peut aider votre application à démarrer plus rapidement via une pré-compilation en bytecode. Avec React-Native 0.70 (maintenant en [version candidate](https://github.com/facebook/react-native/releases/tag/v0.70.0-rc.0?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)), Hermes est activé par défaut. Michael partage divers benchmarks de l'application open-source Mattermost. Le mode de distribution d'Hermes a été modifié pour éviter les problèmes d'incompatibilité ABI. Les API Intl manquantes ont été ajoutées sur iOS. Support à venir pour BigInt et WeakRef.

[**Ce que mon expérience chez Airbnb pendant l'ère React Native m'a appris sur le web3**](https://medium.com/@devinabbott/what-working-at-airbnb-during-its-react-native-era-taught-me-about-web3-6d34d855d892?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)

Devin revient sur l'histoire de React-Native chez Airbnb et la décision très nuancée de l'abandonner. Il établit un parallèle intéressant avec le web3. À long terme, le potentiel technologique l'emporte sur les difficultés des premiers adoptants.

[**Vite 3**](https://vitejs.dev/blog/announcing-vite3.html?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)

Vite est rapidement devenu une référence dans l'écosystème frontend. La v3 apporte de nombreux changements, une nouvelle documentation, des starters pour divers frameworks (dont React), des améliorations sur les imports globaux, une nouvelle documentation...
