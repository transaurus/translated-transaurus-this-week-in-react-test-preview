## ⚛️ React {#react}

[![Next.js 15.3](/emails/issues/230/nextjs.jpg)](https://nextjs.org/blog/next-15-3)

**[Next.js 15.3](https://nextjs.org/blog/next-15-3)**

Une nouvelle version mineure de Next.js est disponible, avec les points forts :

- Turbopack pour les builds (alpha) : Des builds de production plus rapides passant 8000+ tests (99%)
- Support communautaire pour Rspack (expérimental) : Bundler alternatif compatible Webpack
- Hook d'instrumentation client : Configuration précoce de la surveillance et de l'analyse
- Hooks de navigation : Contrôle du routage avec `onNavigate` et `useLinkStatus`
- Améliorations du plugin TypeScript : Meilleure prise en charge des grandes bases de code

C'est formidable de voir Next.js s'améliorer avec un bundling plus rapide via Turbopack pour les builds, et s'ouvrir à Rspack comme alternative à Webpack. Les deux présentent des compromis et des cas d'utilisation différents.

Voir aussi ces ressources associées :

- 👀 [Next.js PR \- Experimental bfcache \- Restore state with `<Activity>`](https://github.com/vercel/next.js/pull/77992): Unsurprisingly, Next.js starts experimenting with the `<Activity>` component above, by implementing a back/forward cache feature that allows pressing the browser back button to instantly transition to the previous page while restoring its former React state. In the future, Next.js (and other React/RN frameworks too) will likely implement speculative pre-rendering: React could start rendering a target page as soon as you hover a link, even before you click it.
- 👀 [React Core PR \- Add Suspensey Images behind a flag](https://github.com/facebook/react/pull/32819): An upcoming React feature could help ensure all images have loaded before transitioning to a new page. The rollout plan is still unclear: it might come in a minor or major.

- 👀 [React Native Fantom](https://github.com/facebook/react-native/blob/main/packages/react-native-fantom/__docs__/README.md) : Un nouvel outil expérimental de tests d'intégration et de benchmarking est en développement actif. Il s'agit d'un lanceur de tests Jest exécutant les parties C++ de RN, permettant de tester des éléments comme les nœuds d'ombre, le layout, les événements, l'ordonnancement et les mises à jour d'état C++. Cet outil est principalement utile pour tester les internes de React Native, mais sa portée pourrait s'étendre au code des bibliothèques/produits à l'avenir.
