## ⚛️ React {#react}

[![useReducer](/emails/issues/131/useReducer.jpg)](https://www.builder.io/blog/use-reducer)

**[Un remède à l'enfer des useState](https://www.builder.io/blog/use-reducer)**

Propose de remplacer plusieurs `useState` par un `useReducer` avec une seule action. Il explique les avantages, comme la possibilité de garantir la cohérence de l'état, ou de migrer vers un réducteur avec plusieurs actions. Un pattern intéressant à connaître. Mon avis : on peut obtenir des bénéfices similaires avec un seul `useState` contenant un objet et un hook personnalisé. Le choix entre `useState` ou `useReducer` devrait rester un détail d'implémentation.

**[React Query en type-safe](https://tkdodo.eu/blog/type-safe-react-query)**

Avec TypeScript, mieux vaut ne pas se mentir. Peut-on faire confiance à notre backend pour retourner le bon payload ? Dominik suggère de combiner React-Query avec Zod. Grâce à l'inférence, plus besoin de forcer un type avec `as` ou des génériques. On peut utiliser tRPC ou Zodios dans un environnement fullstack quand le backend et le frontend sont dans le même monorepo.

## 📱 React-Native {#react-native}

[![useReducer](/emails/issues/131/rn-android-bt.jpg)](https://reactnative.dev/blog/2023/01/12/version-071)

**[React Native 0.71 : TypeScript par défaut, Flexbox Gap, et plus encore...](https://reactnative.dev/blog/2023/01/12/version-071)**

Une version majeure comprenant notamment :

- TypeScript par défaut
- Support du gap Flexbox en pixels (le support en pourcentage viendra plus tard)
- Nouvelles props inspirées du web (accessibilité, style, événements) qui devraient simplifier le développement multiplateforme.
- DX amélioré : inspecteur clic dans les DevTools, nouvelle version d'Hermes, nouvelle architecture...
- [Précompilés Android sur Maven Central](https://github.com/reactwg/react-native-new-architecture/discussions/105) : temps de build Android réduit de 94%, et divers autres impacts. Note : il serait possible d'[améliorer considérablement les temps de build iOS](https://twitter.com/Baconbrix/status/1611527618444120065) également.
