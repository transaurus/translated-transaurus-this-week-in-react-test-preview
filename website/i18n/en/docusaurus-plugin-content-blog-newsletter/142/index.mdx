## ⚛️ React {#react}

[![TypeScript 5.1 beta](/emails/issues/142/ts5.1.jpg)](https://devblogs.microsoft.com/typescript/announcing-typescript-5-1-beta/)

**[TypeScript 5.1 (bêta) + React](https://devblogs.microsoft.com/typescript/announcing-typescript-5-1-beta/)**

Une version importante de TypeScript qui permettra à React de déclarer ses propres types JSX valides. Il est prévu d'utiliser cette nouvelle fonctionnalité dès que possible dans les typedefs de React, qui ont déjà été [forkés](https://github.com/DefinitelyTyped/DefinitelyTyped/pull/65126) pour continuer à prendre en charge les versions antérieures (TypeScript \<= 5.0).

Il existe de bonnes raisons d'adopter TypeScript 5.1+ dès que possible. Cela permet notamment :

- d'utiliser un composant serveur React `async` dans JSX sans erreurs TypeScript ou [contournement](https://gist.github.com/cramforce/b5e3f0b103f841d2e5e429b1d5ac4ded).
- de renvoyer tous les types valides pour `ReactNode` (chaîne, tableau, booléen...) sans nécessiter de fragment ([détails](https://twitter.com/sebastienlorber/status/1648619398259769344)).
- d'autres améliorations intéressantes autour de JSX : attributs avec espace de noms et édition liée (VSCode).

[![Casser volontairement l'API de React Query](/emails/issues/142/rq.jpg)](https://tkdodo.eu/blog/breaking-react-querys-api-on-purpose)

**[Casser volontairement l'API de React Query](https://tkdodo.eu/blog/breaking-react-querys-api-on-purpose)**

React-Query supprimera les callbacks (`onSuccess`, `onError`, `onSettled`) du hook `useQuery` avec la v5. Dominik explique ce choix et propose des alternatives mieux adaptées aux cas courants, comme afficher une notification. Une décision controversée mais nécessaire car ces API étaient souvent mal utilisées. Note : les callbacks sont conservés pour `useMutation`.
