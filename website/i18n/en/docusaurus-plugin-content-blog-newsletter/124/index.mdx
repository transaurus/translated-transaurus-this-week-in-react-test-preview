## ⚛️ React {#react}

[![Inside Framer's Magic Motion](/emails/issues/124/framer-motion.png)](https://www.nan.fyi/magic-motion)

**[À l'intérieur de Magic Motion de Framer](https://www.nan.fyi/magic-motion)**

Un excellent article interactif qui explique ce qu'est une animation de mise en page. Il déconstruit brillamment la technique FLIP qui permet d'optimiser ce type de transition via les transformations CSS (position, échelle). Nous recréons pas à pas le système d'animation de mise en page de Framer Motion.

Sur le même sujet, je recommande aussi de lire [Tout sur les animations de mise en page de Framer Motion](https://blog.maximeheckel.com/posts/framer-motion-layout-animations/).

[![Timeline of a React Component With Hooks](/emails/issues/124/timeline.jpg)](https://julesblom.com/writing/react-hook-component-timeline)

**[Chronologie d'un Composant React avec Hooks](https://julesblom.com/writing/react-hook-component-timeline)**

Un diagramme interactif démontrant le cycle de vie d'un composant React, incluant les hooks, les refs, et les mises à jour/rendu du DOM (non disponible sur mobile). Comprend aussi un quiz pour tester vos connaissances et de nombreuses citations/références.

[![Next.js 13](/emails/issues/124/nextjs13ts.jpg)](https://twitter.com/delba_oliveira/status/1592209522281439232)

**[Plugin TypeScript pour Next.js 13](https://twitter.com/delba_oliveira/status/1592209522281439232)**

Shu Ding travaille sur un plugin de service de langage TypeScript ([source](https://github.com/vercel/next.js/blob/canary/packages/next/server/next-typescript.ts)) qui améliore significativement la DX de Next.js en apportant de la documentation, de nouvelles erreurs et de l'auto-complétion basées sur les conventions Next.js et Server Components. Cela compense efficacement l'[impossibilité de typer les exports d'un module ES](https://github.com/microsoft/TypeScript/issues/38511) dans TypeScript, une fonctionnalité très demandée par les auteurs de frameworks.

[![Writing Performant CSS with vanilla-extract](/emails/issues/124/vanilla-extract.png)](https://www.lekoarts.de/javascript/writing-performant-css-with-vanilla-extract)

**[Écrire du CSS performant avec vanilla-extract](https://www.lekoarts.de/javascript/writing-performant-css-with-vanilla-extract)**

Un article exhaustif sur [vanilla-extract](https://vanilla-extract.style/), la bibliothèque CSS-in-JS sans runtime basée sur TypeScript par Mark Dalgleish (créateur de CSS Modules). Mentionne également sa surcouche CSS atomique [Sprinkles](https://vanilla-extract.style/documentation/packages/sprinkles/). Exemples basés sur React : reproduction d'un composant Tailwind UI. Nombreuses comparaisons avec Tailwind et d'autres solutions CSS-in-JS. Eh bien, cela semble puissant mais pas si simple 😅 Nous devrons étudier cela plus avant.
