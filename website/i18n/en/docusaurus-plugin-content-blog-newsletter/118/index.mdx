## ⚛️ React {#react}

**[Gatsby v5 Alpha](https://www.gatsbyjs.com/blog/gatsby-v5-alpha-is-here-and-its-pretty-exciting/)**

La v5 de Gatsby est en alpha avec 2 nouvelles fonctionnalités principales :

- prise en charge de [l'hydratation partielle](https://www.gatsbyjs.com/blog/a-look-at-partial-hydration-in-gatsby-5/) basée sur les React Server Components et une directive `"client export"` en haut des composants clients
- l'API Gatsby Slices, qui permet d'améliorer significativement les temps de build lorsqu'un composant partagé (layout/navbar/footer) doit être rerendu

**[Rendre React rapide par défaut et véritablement réactif](https://legendapp.com/open-source/legend-state/)**

Article interactif illustrant les problèmes de rerendu dans React qui n'est pas optimisé par défaut. Montre comment la bibliothèque Legend-State, basée sur des observables, permet d'apporter la réactivité à React (un peu comme SolidJS). Le code est alors optimisé par défaut et évite les rerendus inutiles.

**[Ezno - Compilateur expérimental](https://kaleidawave.github.io/posts/introducing-ezno/)**

Nouveau compilateur expérimental de vérification de types pour JavaScript, compatible avec les annotations TypeScript et JSX. S'appuie sur le typage dépendant pour effectuer des analyses statiques avancées et des pré-évaluations à la compilation (rappelle [prepack](https://prepack.io/)). L'objectif semble être de pouvoir compiler une application ressemblant à React sous forme réactive, et de sauter le VDOM. Assez technique à lire.

**[Cartes de composants (paresseuses)](https://julesblom.com/writing/lazy-component-maps)**

Présente une technique utile à connaître pour tirer parti du code-splitting lors du rendu conditionnel. Particulièrement utile sur de grandes énumérations/types union, comme un composant `<Icon name="my-icon-name"/>`.

**[Whyframe](https://whyframe.dev/)**

Permet d'isoler le rendu des composants frontend dans une iframe. Prend désormais en charge [Webpack, Next.js et Docusaurus](https://twitter.com/bluwyoo/status/1576947013450539008). Utile pour les sites de documentation de bibliothèques de composants, garantissant que le CSS global du site n'affecte pas le rendu du composant affiché.

## 📱 React-Native {#react-native}

**[Reassure - Surveillance Continue des Performances d'Application Simplifiée](https://www.callstack.com/blog/app-performance-monitoring)**

Callstack estime que 80% des problèmes de performance sur React-Native proviennent de JavaScript, notamment d'une mauvaise utilisation de React. Ils ont créé Reassure, une nouvelle bibliothèque de tests de régression de performance. Elle s'intègre bien à votre CI pour éviter la dégradation des performances de vos applications au fil du temps. Le [support web](https://github.com/callstack/reassure/issues/39) n'est pas encore disponible.
