## 📱 React Native {#react-native}

[![React Native WebGPU](/emails/issues/196/rn-webgpu.jpg)](https://www.youtube.com/watch?v=i-DGY98cTjk)

**🎥 [React Native WebGPU](https://www.youtube.com/watch?v=i-DGY98cTjk)**

React Native WebGPU vient de sortir, proposant une API unifiée pour exploiter le GPU pour des graphismes avancés et des calculs dans React Native ! En réutilisant le standard Web API, nous pourrons facilement intégrer des exemples de code WebGPU existants et des bibliothèques comme ThreeJS directement dans React Native.
Au-delà des graphismes, React Native WebGPU peut être utilisé pour des tâches comme les calculs matriciels, ce qui le rend intéressant pour les algorithmes de machine learning sur appareil. Il est construit avec Nitro Modules pour une interopérabilité fluide entre TypeScript et C++. Actuellement en préversion technique, cela annonce des développements passionnants à venir !

---

**L'avenir des modules React Native**

Des temps passionnants pour les développeurs React Native ! Marc Rousavy bouleverse les choses avec NitroModules, une nouvelle façon d'écrire du code natif 📦 [qui surpasse les solutions existantes dans les tests de vitesse](https://github.com/mrousavy/NitroBenchmarks). Il a rencontré certaines limites avec JSI dans VisionCamera. Le projet est déjà stable côté C++ et est utilisé dans divers projets comme 🐦 [react-native-unistyles](https://x.com/jpudysz/status/1822541758137303233) et react-native-webgpu, avec les liaisons Swift et Kotlin encore en cours.

👀 [Parallèlement, une nouvelle proposition pour promouvoir l'utilisation de Swift pour TurboModules est apparue.](https://github.com/okwasniewski/discussions-and-proposals/blob/22edd2c7853aaf2ee654b57f4521e43d668103fb/proposals/0000-swift-turbo-modules.md) Ces changements faciliteront le développement de modules natifs en abandonnant Objective-C et C++. Avec ces mises à jour, la création de modules natifs va s'améliorer considérablement !
