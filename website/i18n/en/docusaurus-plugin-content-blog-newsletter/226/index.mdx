## ⚛️ React {#react}

[![Parcel v2.14.0](/emails/issues/226/parcel.jpg)](https://parceljs.org/blog/v2-14-0/)

[**Parcel v2.14.0**](https://parceljs.org/blog/v2-14-0/)

Parcel est intéressant : c'est un bundler robuste et complet en fonctionnalités. Bien qu'aucun framework React majeur ne l'utilise ou ne le supporte, cela ne l'empêche pas d'évoluer, avec la v2.14.0 qui ajoute plusieurs fonctionnalités majeures :

- Server Components avec rendu client, serveur et statique
- Support MDX intégré (utilisant la librairie Rust `mdx-rs`)
- Un CLI `create-parcel` pouvant remplacer `create-react-app`
- Un nouveau système d'overlay d'erreurs React

Si vous "construisez votre propre framework" ou avez un cas d'usage spécifique, tout cela rend parcel encore plus attractif.

## 📱 React-Native {#react-native}

[![React Native Worklets RFC](/emails/issues/226/worklets.jpg)](https://github.com/software-mansion/react-native-reanimated/discussions/7264)

[**React Native Worklets RFC**](https://github.com/software-mansion/react-native-reanimated/discussions/7264)

React Native Worklets est une nouvelle bibliothèque extraite de Reanimated qui permet un véritable multithreading dans React Native. Elle vous permet d'exécuter du code JavaScript sur plusieurs environnements d'exécution. Un nouveau modèle de mémoire introduit `Serializables`, `Shareables` et `Synchronizables` pour gérer les données entre les environnements d'exécution.

En découplant le multithreading des animations, Reanimated devient plus léger, ce qui est particulièrement bénéfique pour les applications web visant des tailles de bundle plus petites. Parallèlement, les développeurs peuvent désormais utiliser le multithreading pour des tâches non-UI comme le stockage asynchrone, les opérations de base de données, le traitement audio en temps réel ou les calculs alimentés par l'IA - sans dépendre d'une bibliothèque d'animation.
