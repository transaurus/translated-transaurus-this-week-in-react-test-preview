Il est rare qu'une vidéo fasse la une ici, mais celle-ci se distingue pour plusieurs raisons.

D'abord, son sujet est passionnant et superbement illustré par des animations. Le pré-rendu partiel est une [fonctionnalité expérimentale de Next.js 14](https://nextjs.org/blog/next-14#partial-prerendering-preview) qui parachève la vision du framework en combinant rendu statique et dynamique. Next.js devient ainsi un framework multi-paradigme supportant toute la diversité des techniques de rendu. Simple à adopter, basé sur Suspense et sans nouvelle API : il suffit d'activer un flag.

Deuxièmement : cette vidéo incroyablement bien produite a été créée avec [Remotion](https://www.remotion.dev/), en utilisant du code React web 🙉. Bravo Delba, tu as assuré ! Nous avons hâte de voir plus de vidéos comme celle-ci et d'en savoir plus sur votre processus créatif Remotion. Sur une note connexe, l'équipe Remotion vient de partager le sien avec un [making-of de GitHub Unwrapped 2023](https://www.youtube.com/watch?v=WW8vIZ1kwR0).

## 📱 React-Native {#react-native}

Cette section est désormais co-écrite avec [Benedikt](https://twitter.com/bndkt). N'hésitez pas à nous envoyer vos commentaires par e-mail ou sur [Twitter](https://twitter.com/bndkt) !

[![État de React Native 2023 - Résultats de l'enquête](/emails/issues/172/stateofrn.jpg)](https://results.stateofreactnative.com/)

**[État de React Native 2023 - Résultats de l'enquête](https://results.stateofreactnative.com/)**

Je suis un grand fan de toutes les enquêtes "État de...", car elles nous donnent un aperçu unique de ce qui se passe dans l'écosystème. C'est une chose de suivre le buzz généré sur X et ailleurs, mais c'en est une autre de voir les statistiques de ce que les développeurs utilisent réellement. Je suis donc très heureux de voir les résultats de l'État de React Native de l'année dernière publiés ! Et comme c'est maintenant la deuxième année que cette enquête est menée, nous pouvons également identifier certaines tendances en comparant les résultats à l'année précédente. Pour tous les détails, [jetez un œil aux résultats vous-même](https://results.stateofreactnative.com/) ! Je vais juste souligner quelques éléments que j'ai trouvés intéressants, certains m'ont vraiment surpris :

- La gestion d'état continue d'être dans une situation très intéressante (jeu de mots intentionnel), avec Redux occupant la dernière place en rétention avec seulement 44 % (contre 57 % l'an dernier), mais restant néanmoins à la deuxième place après les capacités natives de React en termes d'utilisation avec 78 % (contre 85 %).
- La récupération de données est une catégorie où les résultats correspondent à l'attention observée sur les réseaux sociaux, TanStack Query et tRPC occupant les premières places en termes d'intérêt et de rétention, tandis que Relay de Meta reste largement marginal en dehors de Meta avec seulement 5 % d'utilisation.
- L'avenir de la navigation est Expo Router, dont l'utilisation a explosé de 12 % l'an dernier à 40 % aujourd'hui.
- Au vu de l'activité sur les réseaux sociaux, le stylisme semble être la catégorie la plus discutée avec une pléthore d'options. En termes d'utilisation, les classiques (API StyleSheet, style en ligne, composants stylisés) restent dominants, tandis que NativeWind et Tamagui rattrapent leur retard. Tamagui a généré le plus d'intérêt cette année (et affiche aussi la meilleure rétention et intérêt comme bibliothèque de composants UI), suivi de près par un nouveau venu : react-native-unistyles.
- Mention spéciale pour Expo : dans la section "Outils React Native", 6 des 10 meilleurs outils ainsi que les 2 premières options de déploiement sont développés par Expo.
- Plus de 85 % des répondants estiment que React Native évolue dans la bonne direction, et la part de développeurs trouvant la création d'apps RN trop complexe a diminué d'environ 30 % à 25 % – c'est encore trop, mais nous semblons aller dans le bon sens !

Dans l'ensemble, des résultats très intéressants qui méritent d'être [consultés](https://results.stateofreactnative.com/) en détail ! Un grand merci à Bartłomiej, Kacper et Software Mansion pour avoir rendu cela possible !
