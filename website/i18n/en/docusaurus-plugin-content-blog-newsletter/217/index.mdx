## ⚛️ React {#react}

[![Revealed: React's experimental animations API](/emails/issues/217/viewtransition.jpg)](https://motion.dev/blog/reacts-experimental-view-transition-api)

**[Révélé : L'API expérimentale d'animations de React](https://motion.dev/blog/reacts-experimental-view-transition-api)**

Le créateur de Motion explique les compromis d'animation des View Transitions et pourquoi il est logique que React l'intègre directement dans son réconciliateur via un nouveau composant `<ViewTransition>`. Cet article interactif montre un aperçu de la future fonctionnalité React, incluant des détails d'implémentation actuels, et explique comment elle chevauche légèrement les animations de disposition de Motion.

Gardez à l'esprit que cette nouvelle API passionnante est toujours en développement actif, sujette à changement, avec de nouvelles PR en cours :

- 👀 [PR React Core - Prise en charge uniquement de View Transitions v2](https://github.com/facebook/react/pull/31996) : Le nouveau composant exploitera les API récentes de la spécification v2 comme `view-transition-class`, nécessite Safari 18.2+, et n'est toujours pas pris en charge dans Firefox.
- 👀 [PR React Core - Noms de classes View Transition basés sur le type d'événement](https://github.com/facebook/react/pull/32050)
- 👀 [PR Next.js - Drapeau `experimental.viewTransition`](https://github.com/vercel/next.js/pull/74659) : Vous pouvez tester la fonctionnalité dans Next.js 15.2 canary 4.

## 📱 React-Native {#react-native}

[![Aperçu d'EAS Hosting](/emails/issues/217/easdeploy.jpg)](https://expo.dev/blog/expo-announces-eas-hosting-service)

**[Aperçu d'EAS Hosting](https://expo.dev/blog/expo-announces-eas-hosting-service)**

Expo a dévoilé ce que nous attendions : la première plateforme universelle d'hébergement d'applications qui complète la vision d'apporter la DX web au monde cross-plateforme. Elle nous permet de déployer de manière transparente les fonctionnalités serveur d'Expo, notamment Expo pour le Web, les API Routes et les React Server Components universels. Elle prend en compte les subtilités des applications mobiles, comme la synchronisation des versions backend/app. Vous pouvez utiliser soit le CLI soit le CI (EAS Workflows). Sous le capot, elle utilise Cloudflare Workers. Un plan gratuit est disponible.

[![Cinq ans de React Native chez Shopify](/emails/issues/217/shopify.jpg)](https://shopifyengineering.myshopify.com/blogs/engineering/five-years-of-react-native-at-shopify)

**[Cinq ans de React Native chez Shopify](https://shopifyengineering.myshopify.com/blogs/engineering/five-years-of-react-native-at-shopify)**

Shopify a migré avec succès ses 6 applications vers React Native, et a partagé un retour nuancé qui résume bien les compromis de l'utilisation de React Native en pratique. Du côté positif : support cross-platform, performances, hot reloading, portabilité des connaissances... Des compétences natives restent nécessaires, et quelques points pourraient être améliorés, comme l'expérience de débogage et la facilité de mise à jour.
