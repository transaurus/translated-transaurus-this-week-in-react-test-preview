## ⚛️ React {#react}

[![React 19 RC1](/emails/issues/210/react19.jpg)](https://github.com/facebook/react/issues/29898#issuecomment-2477449973)

[**React 19 RC1 - Préchargement des composants frères**](https://github.com/facebook/react/issues/29898#issuecomment-2477449973)

Nous avions expliqué comment la sortie initiale de React 19 avait été annulée à cause d'un changement dans Suspense qui aurait provoqué des cascades dans les applications utilisant Suspense sans démarrer les récupérations dans les gestionnaires d'événements.  
L'équipe React a travaillé dur pour réintroduire le prérendu des composants frères avec Suspense sans provoquer de régressions de performance pour les applications qui démarrent tôt leurs récupérations. Leur solution est maintenant prête et s'appelle le [préchargement](https://react.dev/blog/2024/04/25/react-19-upgrade-guide#improvements-to-suspense). Dominik, le mainteneur de React Query qui avait alerté sur ce problème, [a confirmé sa satisfaction concernant le correctif](https://bsky.app/profile/tkdodo.eu/post/3layw6wk5bs2b).  
Comme d'habitude, les nouvelles fonctionnalités étaient déjà disponibles dans Next.JS (qui utilise React Canary). Elles incluent les composants et actions serveur, le nouveau hook `use`, des améliorations dans la gestion des formulaires (`useFormStatus` et `useOptimistic`), et un petit changement longtemps attendu : `forwardRef` n'est plus nécessaire !

## 📱 React-Native {#react-native}

[![Expo Workflows](/emails/issues/210/workflows.jpg)](https://expo.dev/blog/expo-workflows-automate-your-release-process)

[**Expo Workflows**](https://expo.dev/blog/expo-workflows-automate-your-release-process)
Expo a annoncé "Workflows", un nouvel outil CI/CD pour les apps React Native et Expo qui simplifie et accélère les publications avec des tâches préconfigurées (comme les tests E2E via Maestro), des workflows automatisés et une sélection intelligente de machines pour des performances optimales. Bien que ce ne soit pas un remplacement direct des solutions CI/CD plus générales comme GitHub Actions, il promet des améliorations passionnantes à explorer.

🎉 [La fête de lancement d'Expo](https://expo.dev/launch-party) est toujours en cours. Ne manquez pas leurs annonces !

- 📣 [Expo \- Introducing Channel Pausing for EAS Update : Pause your channels either on the web page or via the eas cli](https://expo.dev/changelog/2024/11-14-channel-pausing)
- 📜 [Universal React Server Components in Expo Router (in beta)](https://expo.dev/blog/universal-react-server-components-developer-preview): Expo Router introduces for the first time universal React Server Components, enabling server-side rendering in both native and web apps. Though not production-ready due to unavailable production exports and mainly aimed at library authors, it's an exciting development because it enables direct server-side access to resources like secrets, databases, and file systems, simplifying development and improving performance by eliminating client-server round trips.
- 📜 [The magic of Expo DOM Components](https://expo.dev/blog/the-magic-of-expo-dom-components): Expo DOM Components make it easier to integrate web features into React Native apps, combining the speed of WebViews with incremental native migration. While limited by WebView performance and compatibility with some Expo features, future updates aim to enhance native integration and optimize performance.
