## ⚛️ React {#react}

[![Next.js 15.2](/emails/issues/224/next.jpg)](https://nextjs.org/blog/next-15-2)

[**Next.js 15.2**](https://nextjs.org/blog/next-15-2)

Vercel a publié une nouvelle version mineure de Next.js avec des améliorations majeures de la DX. Les points forts incluent :

- Interface d'erreur repensée et traces de pile améliorées : Une expérience de débogage repensée exploitant la nouvelle API React [`captureOwnerStack()`](https://react.dev/reference/react/captureOwnerStack)
- Métadonnées en streaming : Les métadonnées asynchrones ne bloqueront plus le rendu des pages ni les transitions côté client
- Améliorations des performances de Turbopack : Temps de compilation plus rapides et réduction de l'utilisation mémoire
- Transitions de vue React (expérimental) : Support expérimental de la nouvelle API View Transitions de React
- Middleware Node.js (expérimental) : Support expérimental de l'utilisation du runtime Node.js dans les Middlewares

[![TanStack Form 1.0](/emails/issues/224/form.jpg)](https://tanstack.com/blog/announcing-tanstack-form-v1)

[**TanStack Form 1.0**](https://tanstack.com/blog/announcing-tanstack-form-v1)

Une nouvelle bibliothèque TanStack, des années de développement, vient d'être publiée en version stable. L'article d'annonce met en lumière un sous-ensemble intéressant de ses fonctionnalités :

- Agnostic des frameworks, headless : Supporte 5 frameworks, dont React utilisé dans les exemples de l'article
- Sécurité de typage extrême : J'avoue que `<form.Field>` est une API plutôt cool
- Validation par schéma : Support de Zod, Valibot, ArkType, Standard Schema
- Validation asynchrone, debouncing intégré, AbortSignal
- Isomorphique, multiplateforme : Partage de la logique de validation client/serveur

## 📱 React-Native {#react-native}

[![Lynx : Débloquez le natif pour plus de possibilités](/emails/issues/224/lynx.jpg)](https://lynxjs.org/blog/lynx-unlock-native-for-more)

[**Lynx : Débloquez le natif pour plus de possibilités**](https://lynxjs.org/blog/lynx-unlock-native-for-more)

ByteDance vient d'annoncer Lynx, sa technologie native multiplateforme. Elle est largement utilisée dans leurs applications de production, dont TikTok ! Avec des liaisons React, c'est un concurrent sérieux à React Native, et le support d'autres frameworks est attendu. Les choix architecturaux sont intéressants, avec un modèle à deux threads et différents runtimes. Elle prend en charge certaines API CSS et Web que React Native n'a toujours pas et utilise des éléments intrinsèques JSX intégrés pratiques comme `<view>`.

L'ensemble de fonctionnalités au lancement est impressionnant, mais il y a aussi des surprises potentiellement déplaisantes comme l'utilisation de React 17 / Preact ou des éléments personnalisés sur le web. Le temps dira si la technologie tient ses promesses et rivalise vraiment avec le leader établi. Quoi qu'il en soit, la concurrence est saine.
