## ⚛️ React {#react}

[![Remix - Fog of War](/emails/issues/192/rr.jpg)](https://remix.run/blog/fog-of-war)

**[Remix - Fog of War](https://remix.run/blog/fog-of-war)**

Nous avions déjà évoqué cela dans notre dernier numéro, mais un article de blog dédié explique désormais en détail le sujet. Une nouvelle fonctionnalité de React Router vous permet de fractionner votre manifeste de routes en morceaux. Cet article explique son intégration dans Remix 2.10 et les compromis impliqués. Un manifeste de routes peut devenir très volumineux, et si vous ne le fragmentez pas, il peut retarder l'hydratation React de votre application car il doit être téléchargé au préalable. Si vous avez un site Remix ou React Router de taille moyenne/grande, je vous recommande de vérifier immédiatement la taille de votre manifeste de routes et de réfléchir à des moyens logiques de le fragmenter, car la réduction de taille permise par cette fonctionnalité peut avoir un impact significatif sur les performances. Le manifeste initial de Shopify est passé de 10 Mo à 1,9 Ko ! Le manifeste du site Docusaurus fait aussi 200 Ko et je compte définitivement implémenter cette solution.

## 📱 React-Native {#react-native}

[![react-native-filament](/emails/issues/192/filament.jpg)](https://github.com/margelo/react-native-filament)

**[react-native-filament](https://github.com/margelo/react-native-filament)**

L'équipe Margelo a lancé un nouveau moteur de rendu 3D basé physiquement en temps réel pour React Native, basé sur Google Filament. Il est accéléré par GPU via Metal/Vulkan, rendu sur un thread séparé, fonctionne avec Reanimated, et prend déjà en charge la Nouvelle Architecture et Expo. Le readme explique bien comment il diffère des solutions 3D existantes basées sur expo-gl (expo-three et react-three-fiber RN). Avec Skia et Filament, l'avenir s'annonce prometteur pour les capacités 2D/3D de React Native.

[![Rise Tools](/emails/issues/192/rise.jpg)](https://rise.tools/)

**[Rise Tools](https://rise.tools/)**

Une toute nouvelle solution pour créer des interfaces utilisateur pilotées par serveur dans React Native, permettant de déployer de nouvelles vues côté serveur sans reconstruire/resoumettre votre application (ou même diffuser une mise à jour OTA). Elle est philosophiquement très proche du modèle des React Server Components, mais avec des compromis légèrement différents : au prix d'un câblage manuel supplémentaire, cette nouvelle solution est agnostique en termes de framework et de langage grâce à un protocole définissant la sérialisation des vues. C'est encore le début, mais cela semble prometteur.
