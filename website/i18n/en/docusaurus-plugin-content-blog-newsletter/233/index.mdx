## 📱 React-Native {#react-native}

[![Expo SDK 53](/emails/issues/233/expo53.jpg)](https://expo.dev/changelog/sdk-53)

**[Expo SDK 53](https://expo.dev/changelog/sdk-53)**

C'est une version d'Expo assez impressionnante, avec comme points forts :

- Mises à niveau vers React Native 0.79, React 19 et RNW 0.20
- Activation par défaut de la Nouvelle Architecture
- Mode Edge-to-edge activé par défaut sur Android
- Nouveau [Expo BackgroundTask](https://expo.dev/blog/goodbye-background-fetch-hello-expo-background-task), une méthode moderne, fiable, économe en énergie et pérenne pour exécuter des tâches en arrière-plan utilisant les API système `WorkManager` d'Android et `BGTaskScheduler` d'iOS
- Nouveau [Expo Maps](https://expo.dev/blog/introducing-expo-maps-a-modern-maps-api-for-expo-developers) (alpha) construit sur l'API Expo Modules, utilisant SwiftUI et Jetpack Compose, exposant les composants par défaut Apple Maps et Google Maps
- Résolution `package.json#exports` de Metro activée par défaut
- Déploiement des builds de développement sur TestFlight avec `npx testflight`
- Réduction des temps de build Android jusqu'à 25% avec les Expo Modules précompilés
- Et bien plus encore !

Voir aussi la version associée [Expo Router v5](https://expo.dev/blog/expo-router-v5) avec ses propres points forts :

- Nécessite React 19 / Expo 53
- Améliore la DX (expérience développeur), l'écran d'erreur et la journalisation, exploite les piles propriétaires de React
- Déploiement de routes API versionnées en production avec EAS Hosting
- React Server Functions en version bêta
- Routes protégées

[![Game On - Apport de WebGPU à React Native](/emails/issues/233/skia.jpg)](https://www.youtube.com/watch?v=t9t-VXwIc4I)

**[🎥 Game On \- Apport de WebGPU à React Native](https://www.youtube.com/watch?v=t9t-VXwIc4I)**

Dans une vidéo de 37 minutes remplie d'annonces passionnantes, William Candillon et ses invités nous ont offert un aperçu du futur du dessin et de l'animation. Soutenu par Shopify, il apporte la prise en charge moderne de WebGPU (Vulkan et Metal) à React Native.

En s'alignant sur les API web, il devient possible de copier-coller des exemples web existants (y compris Three.js) et de les faire fonctionner sur toutes les plateformes. Cela permet également une utilisation polyvalente du GPU (IA, TensorFlow.js), réalisable plus facilement grâce à une bibliothèque bas niveau pratique TypeGPU.

Ces travaux étaient nécessaires pour créer un nouveau backend expérimental Skia Graphite basé sur WebGPU. Il permet de composer et d'entrelacer des expériences 2D/3D de manière performante, comme sur le web, mais sur toutes les plateformes, avec une intégration complète de Reanimated. Skia n'introduit aucune nouvelle fonctionnalité, mais améliore significativement les performances, réduit les plantages, et prend en charge 3 nouvelles plateformes : tvOS, macOS et Node. Continue de masser cette pieuvre William !
