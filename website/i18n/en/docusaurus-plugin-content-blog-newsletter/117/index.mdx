## ⚛️ React {#react}

**[React I Love You, But You're Bringing Me Down](https://marmelab.com/blog/2022/09/20/react-i-love-you.html)**

L'auteur de React-Admin donne une critique équitable de React sur de nombreux aspects (formulaires, useContextSelector, refs, tableau de dépendances...). Ne manquez pas la [réponse de Dan Abramov](https://twitter.com/dan_abramov/status/1572590212647391235).

**[Next.js Layouts RFC in 5 minutes](https://vercel.com/blog/next-js-layouts-rfc-in-5-minutes)**

Le routeur Next.js basé sur le système de fichiers sera considérablement amélioré dans la prochaine version majeure. Il inclut le support des mises en page imbriquées (comme dans Remix). Cela permet d'éviter le démontage/remontage inutile d'une mise en page lors de la navigation, mais aussi de gérer le chargement des données et les erreurs. Cet article donne un aperçu rapide. Voir aussi le [fil de Delba](https://twitter.com/delba_oliveira/status/1568291558776274945).

**[React Router 6.4](https://remix.run/blog/react-router-v6.4)**

Grosse mise à jour : comme prévu, de nombreuses API de Remix (chargement de données, mutation...) ont finalement été déplacées vers React-Router. Regardez la [vidéo de 2min](https://www.youtube.com/watch?v=aX74DbFT5nI).

**[What’s New In Gatsby 2022?](https://www.smashingmagazine.com/2022/09/new-gatsby-2022/)**

Paul a mis à jour son blog vers la dernière version Gatsby v4 et partage avec nous comment il a utilisé les dernières fonctionnalités. La Génération Statique Différée peut réduire considérablement le temps de build. Voir aussi la [Discussion Umbrella Gatsby 5](https://github.com/gatsbyjs/gatsby/discussions/36609) et la vidéo [Gatsby v5 Alpha Showcase](https://www.youtube.com/watch?v=C-WrnfUm33k) : l'hydratation partielle arrive !

## 📱 React-Native {#react-native}

**[RFC : Routage natif basé sur le système de fichiers avec Expo et React Native](https://blog.expo.dev/rfc-file-system-based-routing-in-react-native-7a35474722a)**

Evan Bacon travaille sur un nouveau routeur Expo basé sur des conventions de système de fichiers, similaire à Next.js ou Remix. Il nous explique tout ce qu'il faut savoir sur cette fonctionnalité très attendue qui apporte la DX (Developer Experience) du web au mobile. Cela devrait considérablement réduire le code récurrent de nos applications, mais aussi améliorer la découvrabilité, le deep linking, l'indexation, le chargement des données, la gestion des erreurs... Voir aussi ce [thread](https://twitter.com/Baconbrix/status/1574837248029331495) et ce [podcast](https://podrocket.logrocket.com/expo-router).
