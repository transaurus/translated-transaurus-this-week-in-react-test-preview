## ⚛️ React {#react}

[![StyleX](/emails/issues/165/stylex.jpg)](https://github.com/facebook/stylex)

[**StyleX**](https://github.com/facebook/stylex)

La solution CSS-in-JS de Meta est enfin open-source après une longue attente. Utilisée sur des applications majeures comme Facebook, Instagram et WhatsApp, elle n'est pas strictement couplée à React. Le CSS atomique est extrait au build, avec un runtime léger pour la concaténation des classes. Le support TypeScript est bien conçu. L'API minimaliste repose principalement sur 2 fonctions : `create` et `props`. Nous verrons comment cette nouvelle solution se positionne dans un écosystème déjà concurrentiel avec Tailwind, Panda et d'autres.

Voir aussi :

- 🎥 [Jack Herrington - StyleX : La solution de Meta pour le CSS à grande échelle](https://www.youtube.com/watch?v=6ZDiGtg1jN4)
- 🎥 [Theo T3 - Facebook a essayé Tailwind, puis a construit ça à la place](https://www.youtube.com/watch?v=PHV94q7BivI)

[![CSS Hooks](/emails/issues/165/csshooks.jpg)](https://css-hooks.com/)

Bibliothèque CSS avec bindings React, proposant un véritable retour aux bases via les styles inline. Dans [Du chaos CSS aux CSS Hooks](https://nsaunders.dev/posts/css-madness-to-hooks), l'auteur explique comment il utilise une astuce CSS variable pour supporter les fonctionnalités manquantes comme les media-queries, pseudo-sélecteurs etc.

Personnellement, j'ai toujours préféré la DX des styles inline, mais en pratique c'est trop limité et pas toujours performant. Reste à voir si cette innovation tiendra. Mark Dalgleish (🐦 [tweet](https://twitter.com/markdalgleish/status/1729399475494608923)) s'en réjouit. Notez qu'Adam Wathan n'aurait probablement pas créé Tailwind si les styles inline avaient un meilleur support (🐦 [tweet](https://twitter.com/adamwathan/status/1732209400641724785)).
