[![RBreaking Up with SVG-in-JS in 2023](/emails/issues/151/svginjs.jpg)](https://kurtextrem.de/posts/svg-in-js)

**[Rompre avec SVG-in-JS en 2023](https://kurtextrem.de/posts/svg-in-js)**

Article très complet sur les différentes façons d'utiliser les SVG dans une application React, et leur impact sur les performances.
Déconseille le "SVG-in-JS" : vous devriez éviter de rendre le balisage SVG via des composants React et d'avoir le code SVG présent dans le bundle JS, y compris via des solutions comme le loader SVGR. À quelques exceptions près, il est préférable d'éviter l'inline des SVG dans le HTML : les React Server Components ne sont pas nécessairement une solution idéale non plus. Les sprites SVG sont une bonne solution.

[![Réinventer le traitement de la caméra - TensorFlow Lite GPU directement depuis JavaScript](/emails/issues/151/visioncamera.jpg)](https://mrousavy.com/blog/Reinventing-Camera-Processing)

**[Réinventer le traitement de la caméra - TensorFlow Lite GPU directement depuis JavaScript](https://mrousavy.com/blog/Reinventing-Camera-Processing)**

Marc Rousavy explique comment il parvient à exécuter des modèles TensorFlow sur React-Native Vision Camera v3 ([démo](https://twitter.com/mrousavy/status/1674791891823648770)). Son plugin vient d'être open-sourcé dans [VisionCamera v3 rc.3](https://twitter.com/mrousavy/status/1675883155822202880). L'article est assez technique, expliquant son fonctionnement interne pour fonctionner à 60 FPS ou plus. TLDR : il est désormais très facile d'intégrer un modèle TensorFlow dans Vision Camera, et d'afficher des retours en temps réel via React-Native-Skia. Il n'est pas nécessaire d'écrire du code natif, et toute cette logique peut même être déployée via des mises à jour Over-The-Air !
