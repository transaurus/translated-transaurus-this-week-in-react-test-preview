## 📱 React-Native {#react-native}

[![React Native 0.77](/emails/issues/218/react-native-0.77.jpg)](https://reactnative.dev/blog/2025/01/21/version-0.77)

[**React Native 0.77**](https://reactnative.dev/blog/2025/01/21/version-0.77)

Comme prévu, la nouvelle version de React Native vient de sortir. Elle inclut notamment :

- Améliorations du support CSS : `display: contents`, `boxSizing`, `mixBlendMode`, et `outline`
- Support d'Android 15 edge-to-edge et de la taille de page mémoire 16KB (amélioration des performances)
- Template d'initialisation iOS avec Swift AppDelegate

Bonne nouvelle pour les utilisateurs d'Expo : [le SDK 52 supporte déjà la v0.77](https://expo.dev/changelog/2025/01-21-react-native-0.77), ainsi que de nombreux packages d'infrastructure (Screens, RNGH, Reanimated). Pour le support de React 19, il faudra attendre la v0.78, mais cela pourrait arriver plus tôt que prévu !

[![Reanimated 4.0 beta](/emails/issues/218/reanimated4.jpg)](https://blog.swmansion.com/reanimated-4-is-new-but-also-very-familiar-b926dd59aa40)

[**Reanimated 4.0 beta**](https://blog.swmansion.com/reanimated-4-is-new-but-also-very-familiar-b926dd59aa40)

Reanimated 4 est en version bêta, et c'est une mise à jour majeure ! 🔥 Le cœur d'animation a été repensé. Vous pouvez désormais animer directement avec du CSS standard, sur le thread natif, grâce au nouveau support de :

- Transitions de propriétés CSS
- Animations CSS par keyframes

C'est particulièrement excitant pour les micro-interactions React Native avec une réduction significative du boilerplate, et la capacité à déployer des animations multiplateformes. Les API worklet existantes restent rétrocompatibles et pertinentes pour les besoins plus avancés. Cependant, la Nouvelle Architecture est désormais requise.

Voir aussi [Reactiive migre une animation de Reanimated v2 vers v4](https://reactiive.io/articles/css-animations-react-native) ce qui simplifie grandement les choses. Et nous ne parlons même pas de migrer l'ancien code Reanimated v1 🙈.

Note annexe : avez-vous vu le [futur composant React `<ViewTransition>`](https://motion.dev/blog/reacts-experimental-view-transition-api) en cours de développement actif ? Il n'est pas dans React-DOM, et il y a probablement une raison excitante à cela 👀.
