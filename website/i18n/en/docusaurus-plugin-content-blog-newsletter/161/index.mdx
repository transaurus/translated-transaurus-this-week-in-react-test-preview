## ⚛️ React {#react}

[![The Uphill Battle of Memoization](/emails/issues/161/memo.jpg)](https://tkdodo.eu/blog/the-uphill-battle-of-memoization)

[**The Uphill Battle of Memoization**](https://tkdodo.eu/blog/the-uphill-battle-of-memoization)

Nous voulons créer des applications hautes performances, et pour cela nous utilisons `React.memo`, qui évite les rendus inutiles. Mais ces optimisations se cassent trop facilement. Dominik partage avec nous plusieurs scénarios qui conduisent à une dégradation des performances, notamment l'utilisation d'objets/fonctions de rappel en ligne, ou l'utilisation de slots JSX avec une prop comme `children`.

J'en ai profité pour donner [mon avis sur Twitter](https://twitter.com/sebastienlorber/status/1709258817844752794) : les performances d'un composant React devraient être mieux encapsulées. En attendant le compilateur React-Forget, il existe quelques pistes moins connues à explorer, comme l'utilisation de plugins Babel.

[![Accessible, Typesafe, Progressively Enhanced Modern Web Forms](/emails/issues/161/epicweb.jpg)](https://www.epicweb.dev/accessible-typesafe-progressively-enhanced-modern-web-forms)

[**Formulaires Web modernes accessibles, typés et à amélioration progressive**](https://www.epicweb.dev/accessible-typesafe-progressively-enhanced-modern-web-forms)

Kent nous montre quelques exemples de code Remix et à quel point il est facile de mettre en place une interface utilisateur en attente (pending UI) et une interface utilisateur optimiste (optimistic UI). Il recommande d'utiliser un schéma Zod partagé pour valider les formulaires côté frontend, et de l'étendre côté backend pour éventuellement ajouter des règles supplémentaires. [conform](https://conform.guide/) facilite l'utilisation d'un schéma Zod pour valider un `FormData`, et semble être une solution sous-estimée pour gérer un formulaire React, de l'état local jusqu'au serveur.

[![Une alternative non-cloud à Google Forms qui a tout pour plaire](/emails/issues/155/surveyjs.jpg)](https://surveyjs.io/?utm_source=thisweekinreact&utm_medium=email)

**[Une alternative non-cloud à Google Forms qui a tout pour plaire](https://surveyjs.io/?utm_source=thisweekinreact&utm_medium=email)**

Si vous cherchez à mettre en place **un système robuste de gestion de formulaires entièrement intégré à votre infrastructure informatique** vous permettant de **conserver un contrôle total sur le flux de données**, sans vouloir y consacrer des mois ni des milliers de dollars, alors SurveyJS est la solution idéale !

SurveyJS est une suite de quatre bibliothèques JavaScript open-source permettant de **créer et modifier** plusieurs **formulaires dynamiques basés sur JSON** via un constructeur par glisser-déposer, de les afficher dans votre application React et de **stocker les données sensibles d'enquêtes dans votre infrastructure interne**. Vous pouvez ensuite **afficher vos enquêtes personnalisées sous forme de fichiers PDF modifiables** dans un navigateur et visualiser les résultats via **des graphiques et tableaux interactifs**. [Découvrez-en plus dès maintenant](https://surveyjs.io/?utm_source=thisweekinreact&utm_medium=email).
