[![Vercel réintègre tout le rendu edge vers Node.js](/emails/issues/183/r183.jpg)](https://x.com/leeerob/status/1780705942734331983)

🐦 [Vercel réintègre tout le rendu edge vers Node.js](https://x.com/leeerob/status/1780705942734331983)

Le VP Produit de Vercel, Lee Rob, a expliqué sur X que Vercel "réintègre tout le rendu edge vers Node.js". Si vous utilisez Vercel (et probablement même si ce n'est pas le cas), vous savez que Vercel a poussé le concept "edge" depuis longtemps. D'un côté, edge signifie que le calcul s'effectue près de l'utilisateur accédant à un site hébergé sur Vercel. D'un autre côté, cela signifie aussi que ce calcul s'exécute dans un "runtime edge", basé sur V8 et non sur Node.js. Bien que l'idée d'exécuter des calculs près des utilisateurs finaux semble séduisante, les avantages disparaissent rapidement si les données doivent être chargées depuis un emplacement central (ce qui est très souvent le cas). Parallèlement, les équipes de développement peinent souvent à adapter leur charge de travail au sous-ensemble limité d'API disponibles sur le runtime edge. Il semble que Vercel ait conclu que globalement, les défis pour les développeurs l'emportent sur les bénéfices limités, et revient donc aux runtimes Node.js centralisés. J'apprécie leur transparence sur ce sujet ! Si vous voulez plus de détails, Theo a réalisé une [vidéo](https://youtu.be/lAGE-k1Zfrg?si=5SP4hyoFkixTsh-y) à ce sujet.

**[NLUX ― Nouvelle bibliothèque React pour simplifier l'ajout de fonctionnalités IA](https://nlux.dev/?c-twr2-h)**

[NLUX](https://nlux.dev?c-twr2-p1) est une bibliothèque React [open-source](https://github.com/nluxai/nlux) qui simplifie énormément l'intégration de modèles de langage avancés (LLM) comme ChatGPT dans vos applications web ou sites. Avec seulement quelques lignes de code, vous pouvez ajouter des capacités conversationnelles d'IA.

Principales fonctionnalités 🌟 :

- **Créez des interfaces de chat IA en minutes** - Interfaces conversationnelles IA de haute qualité avec peu de code.
- **Composants React et Hooks** - `<AiChat />` pour l'UI et le hook `useChatAdapter` pour une intégration facile.
- **Adaptateurs flexibles** - Pour `ChatGPT`, `LangChain` et `HuggingFace`, avec une API pour **créer votre propre adaptateur** pour n'importe quel backend.
- **Synchronisez l'état de l'UI avec les backends IA** de manière optimale.
- **Streaming Markdown** - Affichez le markdown généré par les LLM en temps réel.
- **Zéro dépendance** - Base de code légère.

Plus d'informations, d'exemples et la documentation sur [nlux.dev](https://nlux.dev?c-twr2-p1)

## 📱 React-Native {#react-native}

[![React Native 0.74 publiée, l'IDE React Native est désormais en bêta](/emails/issues/183/rn183.jpg)](https://reactnative.dev/blog/2024/04/22/release-0.74)

📦 [React Native 0.74 publiée](https://reactnative.dev/blog/2024/04/22/release-0.74)

Cette semaine a vu la sortie de React Native 0.74, une étape majeure pour la communauté ! Cette dernière version rend le "mode sans pont" (bridgeless mode) par défaut pour les nouveaux projets utilisant la Nouvelle Architecture (avec la prochaine version 0.75, la Nouvelle Arch deviendra également la valeur par défaut). Les deux peuvent toujours être désactivés si nécessaire, mais ils représentent clairement l'avenir de React Native. Ces derniers mois, en particulier, BEAUCOUP de bibliothèques ont ajouté leur prise en charge. Mais il y a plus : RN 0.74 inclut Yoga 3.0. Il y a très peu de changements cassants et de dépréciations, mais il vaut tout de même la peine de les consulter dans les [notes de version](https://reactnative.dev/blog/2024/04/22/release-0.74).

📦 [L'IDE React Native est désormais en bêta](https://ide.swmansion.com/)

Autre nouvelle passionnante cette semaine : la bêta fermée de l'IDE React Native, développé par l'équipe de Software Mansion. Vous pouvez désormais vous inscrire pour obtenir un accès anticipé. Ils ont également mis en ligne une page de présentation, et la [documentation](https://ide.swmansion.com/docs/getting-started) est publiquement accessible pour les curieux.
