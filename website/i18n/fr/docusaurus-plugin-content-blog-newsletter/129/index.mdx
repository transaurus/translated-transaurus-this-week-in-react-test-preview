## ⚛️ React {#react}

[![Storybook 7.0 bêta](/emails/issues/129/storybook7.png)](https://storybook.js.org/blog/7-0-beta/)

**[Storybook 7.0 bêta](https://storybook.js.org/blog/7-0-beta/)**

Storybook annonce officiellement la sortie de la version 7 en bêta. La version finale sera publiée dans quelques mois. C'est une mise à jour majeure avec un plan de migration et même des outils pour nous aider à mettre à jour. Un sous-ensemble des changements principaux :

- Améliorations du design et des performances
- Tests : tests d'interaction, Playwright, rapports de couverture
- Documentation en MDX 2
- Component Story Format 3.0
- Frameworks d'API : Vite, Next.js, SvelteKit

[![OpenNEXT - Open source Next.js serverless adapter](/emails/issues/129/opennext.png)](https://github.com/serverless-stack/open-next)

**[OpenNEXT - Open source Next.js serverless adapter](https://github.com/serverless-stack/open-next)**

Un projet visant à améliorer la portabilité de Next.js vers des plateformes serverless, incluant **toutes** les fonctionnalités (SSR, mais aussi ISR, middleware, optimisation d'images). Pour l'instant, OpenNEXT ne permet de déployer que sur AWS.

Ryan Florence a déclaré que [Next.js n'est pas vraiment open-source](https://twitter.com/ryanflorence/status/1600526114010845184), ce qui a suscité des débats intéressants. La vidéo de Theo Browne résume bien la situation : [NextJS est-il un piège ? Rant sur le verrouillage fournisseur](https://www.youtube.com/watch?v=mhdO8f_45CA). Vercel ne semble pas faire de verrouillage fournisseur, mais il est normal que leur plateforme reste la meilleure pour déployer des applications Next.js.

[![React Email](/emails/issues/129/reactemail.png)](https://react.email/)

**[React Email](https://react.email/)**

Nouvelle solution pour créer des e-mails basée sur React et TypeScript. En pratique, il s'agit d'un moteur de rendu et d'un ensemble de composants headless compatibles avec les clients e-mail populaires, ainsi que d'une interface pour développer des e-mails avec une bonne expérience développeur (DX). La [démo](https://demo.react.email/) reproduit certains e-mails de Vercel, Airbnb, Stripe... Cela ressemble à une version moderne de [React-MJML](https://github.com/wix-incubator/mjml-react), incluant une interface de prévisualisation.

## 📱 React-Native {#react-native}

[![Pointer Events in React Native](/emails/issues/129/pointer.png)](https://reactnative.dev/blog/2022/12/13/pointer-events-in-react-native)

**[Les événements de pointeur dans React Native](https://reactnative.dev/blog/2022/12/13/pointer-events-in-react-native)**

L'API web [Pointer Events](https://www.w3.org/TR/pointerevents3/) est en cours de portage vers React Native. C'est une excellente nouvelle pour le cross-platform. Cela devrait également faciliter l'utilisation de fonctionnalités comme le survol et le clic droit avec des éléments JSX React Native, qui n'étaient initialement pas conçus pour supporter ces fonctionnalités.

---

[![Expo-MDX](/emails/issues/129/expo-mdx.png)](https://twitter.com/Baconbrix/status/1603053674548367360)

**[Expo-MDX](https://twitter.com/Baconbrix/status/1603053674548367360)**

Evan Bacon publie un package pour faciliter l'utilisation de MDX sur React-Native. Metro compile MDX en composants React, et un fournisseur substitue les éléments DOM (cible par défaut de MDX) par des éléments React-Native que vous pouvez facilement styler. On peut s'attendre à une utilisation future dans Expo-Router, comme la possibilité de créer un nouvel écran React-Navigation en créant un simple fichier `.mdx` ? 👌

Le support de MDX sur React-Native est un sujet que j'apprécie particulièrement. Voir aussi ma conférence [React Native EU 2020 - Cross-platform: ReactNative, Jamstack, And Beyond](https://www.youtube.com/watch?v=ScgFQojbAAc) ou mon article [Using Expo in Gatsby](https://sebastienlorber.com/using-expo-in-gatsby).
