## ⚛️ React {#react}

[![Introducing storage on Vercel](/emails/issues/144/vercel.jpg)](https://vercel.com/blog/vercel-storage)

**[Présentation du stockage chez Vercel](https://vercel.com/blog/vercel-storage)**

Vercel vient de lancer son offre de stockage orientée edge, particulièrement adaptée aux React Server Components et à Next.js :

- Vercel KV : stockage clé-valeur Redis, basé sur Upstash.
- Vercel Postgres : base de données PostgreSQL, basée sur Neon.
- Vercel Blob : stockage de fichiers, basé sur Cloudflare R2.

C'est la [semaine Vercel Ship](https://vercel.com/ship) : d'autres nouvelles devraient être annoncées dans les prochains jours. L'image choisie laisse penser qu'il y aura une version de Next.js demain, introduisant une nouvelle fonctionnalité **Next.js Server Actions**. Cela révèle aussi la volonté d'améliorer l'[amélioration progressive](https://twitter.com/sebastienlorber/status/1653701538336915456) dans les applications React, d'une manière similaire à Remix : ce formulaire peut aussi fonctionner sans JavaScript ni hydratation.

Note : cet exemple n'est pas vulnérable aux injections SQL car il utilise des littéraux de modèle.

[![Million.js - Virtual DOM: Back in Block](/emails/issues/144/million.jpg)](https://millionjs.org/blog/virtual-dom)

**[Million.js - Virtual DOM : Retour en Bloc](https://millionjs.org/blog/virtual-dom)**

Article interactif présentant Million.js et son concept de "Block Virtual DOM". Son approche de diffing est différente : non récursive, basée sur l'analyse statique, la compilation et le dirty checking, assez similaire à Svelte.

Million.js ne remplace pas React mais s'intègre à celui-ci pour améliorer les performances. Il est recommandé de l'utiliser uniquement dans des cas très spécifiques : beaucoup de contenu statique et des arbres d'interface utilisateur avec une structure stable.

## 📱 React-Native {#react-native}

[![Présentation de React Native macOS 0.71](/emails/issues/144/rnmacos.jpg)](https://devblogs.microsoft.com/react-native/2023-04-27-announcing-macos-71/)

**[Présentation de React Native macOS 0.71](https://devblogs.microsoft.com/react-native/2023-04-27-announcing-macos-71/)**

Microsoft a sauté les versions v0.69 et v0.70 pour passer directement à la v0.71. L'objectif est désormais de rester à jour avec les autres plateformes : les travaux sur la v0.72 sont en cours. Cette harmonisation des versions de plateforme semble très utile : selon les retours, il n'est pas facile de mélanger différentes versions de React-Native dans un seul monorepo.

La nouvelle version v0.71 apporte un support expérimental pour Fabric, ainsi que toutes les nouvelles fonctionnalités récemment publiées sur les autres plateformes : TypeScript, Flexbox... Microsoft et Meta collaborent et l'utilisation de cette plateforme semble croître au sein des deux entreprises : Meta utilise par exemple React-Native macOS sur Messenger Desktop.
