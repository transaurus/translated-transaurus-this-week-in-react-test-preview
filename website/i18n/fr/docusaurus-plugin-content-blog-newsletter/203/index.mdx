## ⚛️ React {#react}

[![Les Server Actions ont été renommées en Server Functions](/emails/issues/203/serverfunctions.jpg)](https://19.react.dev/reference/rsc/server-functions)

**[Les Server Actions ont été renommées en Server Functions](https://19.react.dev/reference/rsc/server-functions)**

Ce n'est pas un changement de code, mais une modification terminologique importante désormais officiellement publiée sur le site de documentation de React v19. Les Server Actions sont désormais considérées comme un sous-ensemble de la fonctionnalité plus large des Server Functions. Elles ne deviennent une action que lorsqu'elles sont utilisées dans :

- `<form action>`
- `useActionState()`
- `startTransition()`

Ce changement clarifie que "use server" concerne l'exposition d'un point de terminaison serveur, pas nécessairement limité aux mutations. Récupérer des données et du JSX avec des Server Functions pourrait finalement ne pas être un anti-modèle.

**À propos des Server Actions de Next.js** : elles utilisent des requêtes `POST` et s'exécutent séquentiellement, mais il s'agit d'un détail d'implémentation de Next.js et React ne vous empêche pas d'exécuter des actions en parallèle. L'équipe Next.js a confirmé son intention d'introduire de nouvelles API pour appeler les Server Functions avec `GET`.

**[React DevTools 6.0](https://github.com/facebook/react/blob/main/packages/react-devtools/CHANGELOG.md#600)**

Les React DevTools ont récemment fait l'objet de nombreux travaux, notamment la capacité à présenter un arbre React unifié et facile à comprendre pour les composants Serveur et Client. Les extensions de navigateur n'ont pas encore été publiées, mais le journal des modifications de la v6 est disponible avec de nombreuses PR et captures d'écran à consulter. D'après ce que je constate, l'équipe a accompli un travail remarquable pour offrir une excellente expérience développeur (DX) autour de l'inspection d'éléments, du profilage, de la journalisation, des traces de pile et des sourcemaps.
