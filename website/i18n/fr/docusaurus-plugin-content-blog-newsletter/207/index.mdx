## ⚛️ React {#react}

[![Récapitulatif Next.js Conf 2024](/emails/issues/207/nextjsconf.jpg)](https://vercel.com/blog/recap-next-js-conf-2024)

[**Récapitulatif Next.js Conf 2024**](https://vercel.com/blog/recap-next-js-conf-2024)

La semaine dernière, la communauté s'est réunie à la conférence Next.js. Ce récapitulatif présente les points forts, notamment :

- Next.js 15 stable
- Turbopack Dev stable
- Mise en cache simplifiée, complexité réduite
- Améliorations et modèles d'auto-hébergement

L'article complémentaire [Notre parcours avec la mise en cache](https://nextjs.org/blog/our-journey-with-caching) est particulièrement intéressant, introduisant de nouvelles sémantiques et API de mise en cache expérimentales comme `use cache`, `cacheTag` et `cacheLife`. C'est un système plus flexible, et particulièrement intéressant que vous n'ayez même pas besoin de fournir de clé de cache.

## 📱 React-Native {#react-native}

[![Nouvelle Architecture par défaut, DevTools et plus](/emails/issues/207/newarch.jpg)](https://reactnative.dev/blog/2024/10/23/release-0.76-new-architecture)

[**React Native 0.76 - Nouvelle Architecture par défaut, DevTools et plus**](https://reactnative.dev/blog/2024/10/23/release-0.76-new-architecture)

React Native 0.76 est disponible avec des mises à jour majeures ! La Nouvelle Architecture est maintenant activée par défaut, un nouvel outil React Native DevTools améliore le débogage, et le support CSS pour `boxShadow` et `filter` permet des effets UI avancés (uniquement avec la nouvelle architecture). Attention aux changements cassants : les SDK minimum requis sont maintenant iOS 15.1 et Android SDK 24 (Android 7). C'est le moment de mettre à niveau et d'essayer !

La v0.76 est cool, mais ce qui est encore mieux, c'est que la [Nouvelle Architecture est là](https://reactnative.dev/blog/2024/10/23/the-new-architecture-is-here) et officiellement déclarée prête pour la production ! C'est un effort de plusieurs années et une énorme étape pour l'écosystème React Native ! À ce stade, vous avez probablement déjà entendu parler de JSI, Fabric, Turbomodules, de la communication bridgeless synchrone à faible latence, du mode interop, et plus encore. Il ne s'agit pas seulement de performances, mais aussi de résoudre d'importants problèmes UX/DX comme mesurer des vues de façon synchrone pour positionner une infobulle, ou faire fonctionner `useLayoutEffect` comme sur le web. Ce qui m'enthousiasme le plus est probablement le plus sous-estimé : React Native peut enfin utiliser les fonctionnalités concurrentes de React 18+, notamment les transitions, Suspense et ce qui vient ensuite.
