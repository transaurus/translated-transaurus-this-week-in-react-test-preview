[![useSignal() est le futur des frameworks web](/emails/issues/135/signals.jpg)](https://www.builder.io/blog/usesignal-is-the-future-of-web-frameworks)

**[`useSignal()` est le futur des frameworks web](https://www.builder.io/blog/usesignal-is-the-future-of-web-frameworks)**

Misko est le créateur de Qwik et AngularJS. Il met en avant l'intérêt des signaux, et fait une comparaison avec le fonctionnement de React. React utilise directement la valeur de l'état, tandis que les signaux utilisent un accesseur (getter), et créent des abonnements lorsque cet accesseur est appelé.

Les signaux permettent une optimisation automatique des re-rendus avec une granularité fine, sans nécessiter de mémoïsation. Les composants intermédiaires qui ne font parfois que propager les props n'ont pas besoin de se re-rendre.

Cet article a été publié le lendemain de [l'annonce d'Angular](https://twitter.com/angular/status/1625939902046117890) qui prévoit d'utiliser les signaux (une [démo](https://twitter.com/sarah_edo/status/1628065696247857152) est maintenant disponible).

Andrew Clark a [commenté](https://twitter.com/acdlite/status/1626590880126889984) l'introduction potentielle des signaux dans React. Il préfère pour le moment parier sur le compilateur [React Forget](https://www.youtube.com/watch?v=lGEMwh32soc), dont nous devrions avoir des nouvelles prochainement via un article React Labs. Je pense que c'est une bonne chose que React aille à contre-courant et continue [d'explorer la direction actuelle](https://twitter.com/acdlite/status/1625599973420154881).

[![Announcing Sandpack 2.0 and a Node.js runtime for any browser](/emails/issues/135/sandpack2.jpg)](https://codesandbox.io/blog/announcing-sandpack-2)

**[Annonce de Sandpack 2.0 et d'un runtime Node.js pour n'importe quel navigateur](https://codesandbox.io/blog/announcing-sandpack-2)**

CodeSandbox a annoncé la nouvelle version de Sandpack, un outil permettant de créer des expériences de codage interactif en direct dans le navigateur (éditeur de code + aperçu). Sandpack s'intègre parfaitement avec React grâce à un package dédié. La v1 était déjà utilisée sur le nouveau site de documentation beta de React, par Josh Comeau et bien d'autres.

Cette v2 est basée sur la nouvelle technologie NodeBox qui permet d'exécuter Node.js dans le navigateur. Il est possible d'exécuter Next.js, Vite, Astro et d'autres stacks.

Cela rappelle la technologie de StackBlitz, qui vient d'[annoncer l'API WebContainers](https://blog.stackblitz.com/posts/webcontainer-api-is-here/). L'article de Sandpack explique les différences entre les technologies NodeBox et WebContainers.

**[Bibliothèques JavaScript extensibles pour enquêtes et formulaires](https://surveyjs.io/?utm_source=thisweekinreact&utm_medium=referral)**

[SurveyJS](https://github.com/surveyjs) est un ensemble de bibliothèques JavaScript open-source pour formulaires que vous pouvez facilement intégrer à votre application web. Il permet de **créer et exécuter plusieurs formulaires**, en conservant toutes les données sensibles sur vos serveurs, ou d'avoir **votre propre éditeur de formulaires WYSIWYG auto-hébergé** que les utilisateurs non techniques peuvent utiliser directement.

Vous pouvez également **analyser les résultats d'enquêtes** dans des tableaux de bord personnalisés, sauvegarder vos formulaires en PDF, ou simplement **les convertir en fichiers PDF modifiables**. Il offre un support natif pour le **rendu React et Angular** et une totale liberté de choix concernant le backend, car **toute combinaison serveur + base de données est entièrement compatible**.
