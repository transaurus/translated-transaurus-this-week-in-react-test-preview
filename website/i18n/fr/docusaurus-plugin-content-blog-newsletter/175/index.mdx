**[Paramètres de recherche instantanée avec les React Server Components](https://buildui.com/posts/instant-search-params-with-react-server-components)**

Un article interactif génial montrant qu'il est étonnamment difficile de persister l'état de l'UI dans la chaîne de requête dans un monde de React Server Components tout en fournissant un retour utilisateur immédiat. La mise à jour de l'URL déclenche un aller-retour serveur. Le nouveau hook `useOptimistic` résout ce problème élégamment. Cela montre aussi un bon cas d'utilisation pour le CSS `:has()`, pour éviter de remonter l'état d'attente de `startTransition`.

## 📱 React Native {#react-native}

Cette section est désormais co-écrite avec [Benedikt](https://twitter.com/bndkt). N'hésitez pas à nous envoyer vos commentaires par email ou sur [Twitter](https://twitter.com/bndkt) !

Cette semaine, nous avons tous bénéficié d'une brève accalmie concernant les grandes nouvelles de React Native (ou ai-je manqué quelque chose, dans ce cas [@moi](https://twitter.com/bndkt)). C'est peut-être un changement bienvenu, surtout après la tourmente des événements comme Expo 50, visionOS, l'État de React Native, react-strict-dom, et plus encore. Cependant, cela ne signifie pas que l'écosystème React Native s'est arrêté. De nombreuses petites annonces nous donnent un aperçu du futur proche de React Native. Les premières versions candidates pour [RN 0.74](https://github.com/facebook/react-native/blob/ee2bc59dfe4579cad3b7f0b5de667a89fcac8710/CHANGELOG.md#0740-rc0) ont été publiées. Elles contiennent de nombreuses améliorations ainsi que des dépréciations et des changements cassants, mais à première vue rien de majeur qui m'inquiète quant à l'effort de mise à niveau. Plus notablement, le mode bridgeless devient le nouveau comportement par défaut avec la 0.74 (mais uniquement lorsque la New Arch est activée). Le mode bridgeless est le troisième pilier de la New Arch, après les TurboModules déjà introduits et le nouveau moteur de rendu (Fabric). En parlant de New Arch et des valeurs par défaut, une [PR](https://github.com/facebook/react-native/pull/43135) a été fusionnée qui fera de la New Arch le comportement par défaut à partir de RN 0.75 (il sera toujours possible de s'en désengager). De plus, la RFC définissant les [React Native Frameworks](https://github.com/react-native-community/discussions-and-proposals/blob/main/proposals/0759-react-native-frameworks.md) a été fusionnée, détaillant les responsabilités de RN par rapport aux frameworks construits sur RN.
