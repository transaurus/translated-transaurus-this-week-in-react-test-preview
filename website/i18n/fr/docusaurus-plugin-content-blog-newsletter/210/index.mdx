## ⚛️ React {#react}

[![React 19 RC1](/emails/issues/210/react19.jpg)](https://github.com/facebook/react/issues/29898#issuecomment-2477449973)

[**React 19 RC1 - Préchargement des composants frères**](https://github.com/facebook/react/issues/29898#issuecomment-2477449973)

Nous avions expliqué comment la sortie initiale de React 19 avait été annulée à cause d'un changement dans Suspense qui aurait provoqué des cascades de chargement dans les applications utilisant Suspense sans démarrer les récupérations de données dans les gestionnaires d'événements.  
L'équipe React a travaillé dur pour réintroduire le prérendu des composants frères avec Suspense sans causer de régressions de performances pour les applications qui démarrent les récupérations tôt. Leur solution est maintenant prête et s'appelle le [préchargement (pre-warming)](https://react.dev/blog/2024/04/25/react-19-upgrade-guide#improvements-to-suspense). Dominik, le mainteneur de React Query qui avait alerté sur ce problème, [a confirmé être satisfait du correctif](https://bsky.app/profile/tkdodo.eu/post/3layw6wk5bs2b).  
Comme d'habitude, les nouvelles fonctionnalités étaient déjà disponibles dans Next.JS (qui utilise la version canary de React). Elles incluent les composants serveur et les actions, le nouveau hook `use`, des améliorations pour la gestion des formulaires (`useFormStatus` et `useOptimistic`), et un petit changement longtemps attendu : `forwardRef` n'est plus nécessaire !

## 📱 React-Native {#react-native}

[![Expo Workflows](/emails/issues/210/workflows.jpg)](https://expo.dev/blog/expo-workflows-automate-your-release-process)

[**Expo Workflows**](https://expo.dev/blog/expo-workflows-automate-your-release-process)
Expo a annoncé "Workflows", un nouvel outil CI/CD pour les applications React Native et Expo qui simplifie et accélère les publications d'apps avec des jobs pré-packagés (comme des tests E2E utilisant Maestro), des workflows automatisés et une sélection intelligente de machines pour des performances optimales. Bien que ce ne soit pas un remplacement direct des solutions CI/CD plus générales comme GitHub Actions, il promet des améliorations passionnantes à explorer.

🎉 La [Expo Launch Party](https://expo.dev/launch-party) est toujours en cours. Ne manquez pas leurs annonces !

- 📣 [Expo - Présentation de la mise en pause des canaux pour EAS Update : Mettez vos canaux en pause via la page web ou la CLI eas](https://expo.dev/changelog/2024/11-14-channel-pausing)
- 📜 [Composants serveur React universels dans Expo Router (en bêta)](https://expo.dev/blog/universal-react-server-components-developer-preview) : Expo Router introduit pour la première fois des composants serveur React universels, permettant le rendu côté serveur dans les applications natives et web. Bien que non prêt pour la production en raison de l'indisponibilité des exports de production et principalement destiné aux auteurs de bibliothèques, c'est un développement passionnant car il permet un accès direct côté serveur à des ressources comme les secrets, les bases de données et les systèmes de fichiers, simplifiant le développement et améliorant les performances en éliminant les allers-retours client-serveur.
- 📜 [La magie des composants DOM d'Expo](https://expo.dev/blog/the-magic-of-expo-dom-components) : Les composants DOM d'Expo facilitent l'intégration de fonctionnalités web dans les applications React Native, en combinant la rapidité des WebViews avec une migration native progressive. Bien que limités par les performances des WebViews et la compatibilité avec certaines fonctionnalités d'Expo, les futures mises à jour visent à améliorer l'intégration native et à optimiser les performances.
