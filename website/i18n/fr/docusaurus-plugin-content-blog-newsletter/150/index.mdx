L'équipe Next.js partage un contexte sur le nouveau routeur d'application. Elle a récemment reçu beaucoup de critiques et est bien consciente des bogues et autres problèmes avec l'expérience de développement actuelle. Elle détaille ce sur quoi elle travaille en priorité :

[![](/emails/issues/150/skew-protection.jpg)](https://vercel.com/blog/version-skew-protection)

**[Vercel - Présentation de Skew Protection](https://vercel.com/blog/version-skew-protection)**

Nouvelle fonctionnalité de Vercel qui atténue la désynchronisation front/back pouvant survenir lors d'un nouveau déploiement : les requêtes des clients dans un certain état peuvent maintenant être acheminées vers le déploiement serveur immuable approprié. Particulièrement utile pour ceux qui font du déploiement continu. Disponible expérimentalement dans Next.js 13.4.7, qui appliquera automatiquement les en-têtes HTTP appropriés.
