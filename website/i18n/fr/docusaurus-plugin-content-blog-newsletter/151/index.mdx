- Remotion Studio : Édition visuelle des props basée sur Zod, bouton de rendu...

- Moteur SVG 3D
- Intégration Tailwind et Rive

Il y a aussi un [lancement sur ProductHunt](https://www.producthunt.com/posts/remotion-4-0) aujourd'hui, alors n'hésitez pas à voter ;)

[![RBreaking Up with SVG-in-JS in 2023](/emails/issues/151/svginjs.jpg)](https://kurtextrem.de/posts/svg-in-js)

**[Rupture avec SVG-in-JS en 2023](https://kurtextrem.de/posts/svg-in-js)**

Article très complet sur les différentes façons d'utiliser les SVG dans une application React et leur impact sur les performances.
Déconseille le "SVG-in-JS" : vous devriez éviter de rendre le balisage SVG via des composants React et d'avoir le code SVG présent dans le bundle JS, y compris via des solutions comme le loader SVGR. Avec quelques exceptions, il est préférable d'éviter d'intégrer les SVG en ligne dans le HTML : les React Server Components ne sont pas non plus une solution idéale. Les sprites SVG sont une bonne solution.

[![Réinventer le traitement de la caméra - TensorFlow Lite GPU directement depuis JavaScript](/emails/issues/151/visioncamera.jpg)](https://mrousavy.com/blog/Reinventing-Camera-Processing)

**[Réinventer le traitement de la caméra - TensorFlow Lite GPU directement depuis JavaScript](https://mrousavy.com/blog/Reinventing-Camera-Processing)**

Marc Rousavy explique comment il parvient à exécuter des modèles TensorFlow sur React-Native Vision Camera v3 ([démo](https://twitter.com/mrousavy/status/1674791891823648770)). Son plugin vient d'être open-source dans [VisionCamera v3 rc.3](https://twitter.com/mrousavy/status/1675883155822202880). L'article est assez technique, expliquant son fonctionnement interne pour tourner à 60+ FPS. TLDR : il est désormais très facile d'intégrer un modèle TensorFlow dans Vision Camera et d'afficher un retour en temps réel via React-Native-Skia. Il n'est pas nécessaire d'écrire du code natif, et toute cette logique peut même être déployée via des mises à jour Over-The-Air !
