## ⚛️ React {#react}

[![Les composants serveur React désormais dans RedwoodJS](/emails/issues/179/redwood.jpg)](https://redwoodjs.com/blog/rsc-now-in-redwoodjs)

[**Les composants serveur React désormais dans RedwoodJS**](https://redwoodjs.com/blog/rsc-now-in-redwoodjs)

Bighorn, la prochaine "époque" de Redwood démarrant en v8.0, misera entièrement sur les composants serveur React (RSC). Cet article explique pourquoi Redwood préfère désormais les RSC à GraphQL, qui n'offrait pas la meilleure expérience développeur (DX) initiale pour la plupart des développeurs et projets. GraphQL restera pris en charge et pourra être utilisé conjointement aux RSC dans la même application. Les Cellules Redwood permettent désormais d'exporter une nouvelle fonction "data" pour charger les données de la cellule.

Vous pouvez essayer Redwood Bighorn aujourd'hui en version canari v8, mais il subsiste encore de nombreuses imperfections mentionnées dans l'article de blog, qu'ils corrigeront avant la sortie officielle majeure. Il y a un autre article intéressant à lire [comparant les conventions de routage de Redwood à Next.js et Remix](https://redwoodjs.com/blog/whats-different-comparing-the-router-in-nextjs-app-api-nextjs-pages-api-remix-and-redwoodjs).

## 📱 React Native {#react-native}

Cette section est désormais co-écrite avec [Benedikt](https://twitter.com/bndkt). N'hésitez pas à nous envoyer vos commentaires par e-mail ou sur [Twitter](https://twitter.com/bndkt)!

[![React Navigation - Présentation de l'API Statique](/emails/issues/179/rnav.jpg)](https://reactnavigation.org/blog/2024/03/25/introducing-static-api)

[**React Navigation - Présentation de l'API Statique**](https://reactnavigation.org/blog/2024/03/25/introducing-static-api)

Sans doute l'une des bibliothèques React Native les plus utilisées et les plus importantes est React Navigation. La version majeure actuelle (6) a été introduite il y a environ trois ans, et il semble maintenant que nous nous rapprochions de la prochaine version majeure. Avec la v7 (actuellement en alpha), React Navigation réintroduit une API statique, qui vous permet de définir les écrans à partir d'une configuration objet. Cela permet l'inférence de types (si vous avez déjà créé des types pour un arbre React Navigation dynamique, vous savez à quoi cela vous soulage) et rend également la configuration des liens profonds plus facile (voire partiellement automatique).

Assurez-vous également de lire le [guide de mise à niveau v7](https://reactnavigation.org/docs/7.x/upgrading-from-6.x/) si vous voulez connaître les autres changements à venir !
