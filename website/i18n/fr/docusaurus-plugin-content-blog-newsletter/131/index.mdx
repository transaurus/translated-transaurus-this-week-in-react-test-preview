## ⚛️ React {#react}

[![useReducer](/emails/issues/131/useReducer.jpg)](https://www.builder.io/blog/use-reducer)

**[Un remède à l'enfer de useState](https://www.builder.io/blog/use-reducer)**

Propose de remplacer plusieurs `useState` par un `useReducer` avec une seule action. Il explique les avantages, comme la possibilité de garantir la cohérence de l'état ou de migrer ultérieurement vers un réducteur avec plusieurs actions. Un pattern intéressant à connaître. Mon avis : on peut obtenir des avantages similaires en utilisant un seul `useState` avec un objet unique et un hook personnalisé. `useState` ou `useReducer` devrait être un détail d'implémentation.

[![useReducer](/emails/issues/131/react-query-ts.jpg)](https://tkdodo.eu/blog/type-safe-react-query)

**[React Query type-safe](https://tkdodo.eu/blog/type-safe-react-query)**

Avec TypeScript, vous préféreriez ne pas vous tromper vous-même. Peut-on faire confiance à notre backend pour renvoyer le bon payload ? Dominik suggère de combiner React-Query avec Zod. Grâce à l'inférence, il n'est plus nécessaire de forcer un type avec `as` ou des génériques. On peut utiliser tRPC ou Zodios dans un environnement fullstack lorsque le backend et le frontend sont dans le même monorepo.

## 📱 React-Native {#react-native}

[![useReducer](/emails/issues/131/rn-android-bt.jpg)](https://reactnative.dev/blog/2023/01/12/version-071)

**[React Native 0.71 : TypeScript par défaut, Flexbox Gap, et plus encore...](https://reactnative.dev/blog/2023/01/12/version-071)**

Une version majeure comprenant notamment :

- TypeScript par défaut
- Prise en charge des écarts Flex en pixels (le support des pourcentages arrivera plus tard)
- Nouvelles props inspirées du web (accessibilité, style, événements) qui devraient simplifier le développement multiplateforme.
- Amélioration de l'expérience développeur : inspection au clic dans DevTools, nouvelle version d'Hermes, nouvelle architecture...
- [Précompilations Android sur Maven Central](https://github.com/reactwg/react-native-new-architecture/discussions/105) : temps de build Android réduit de 94%, et divers autres impacts. Note : il serait possible [d'améliorer considérablement les temps de build iOS](https://twitter.com/Baconbrix/status/1611527618444120065).
