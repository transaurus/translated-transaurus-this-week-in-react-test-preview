## ⚛️ React {#react}

[![Panda CSS](/emails/issues/149/panda.jpg)](https://panda-css.com/)

**[Panda CSS](https://panda-css.com/)**

Segun Adebayo (créateur de Chakra UI) dévoile sa bibliothèque CSS-in-JS tant attendue ([bande-annonce](https://twitter.com/thesegunadebayo/status/1669732623994834946)). Elle offre une excellente DX, s'appuie sur des fonctionnalités CSS modernes (couches en cascade), est performante (analyse statique à la compilation, pas de runtime, CSS atomique)... Bref, elle coche beaucoup de cases et suscite un vif intérêt. Elle s'intègre particulièrement bien avec React (props de style JSX, compatible avec les React Server Components, etc.), mais peut aussi être utilisée avec d'autres frameworks.

C'est une bibliothèque à surveiller de près qui pourrait bien rivaliser avec Tailwind : elle s'intègre mieux avec les frameworks cibles et ne vous force pas à utiliser un `className` interminable.

## 📱 React-Native {#react-native}

[![RN 0.72](/emails/issues/149/rn.jpg)](https://reactnative.dev/blog/2023/06/21/0.72-metro-package-exports-symlinks)

**[React Native 0.72 - Prise en charge des liens symboliques, de meilleures erreurs et plus](https://reactnative.dev/blog/2023/06/21/0.72-metro-package-exports-symlinks)**

Après plusieurs Golden RC, la dernière version de React-Native est enfin sortie et sera bientôt utilisée dans le SDK Expo 49. Plusieurs améliorations sont mentionnées (DX, Hermes, performance...) ainsi que des changements cassants. J'apprécie particulièrement ces 2 nouvelles fonctionnalités dans le bundler Metro :

- Liens symboliques (bêta) : fonctionnalité longtemps attendue pour simplifier la prise en charge des monorepos et de pnpm
- [Package Exports](https://reactnative.dev/blog/2023/06/21/package-exports-support) (bêta) : meilleure encapsulation des bibliothèques, compatibilité avec le reste de l'écosystème frontend et capacité à exposer un point d'entrée React-Native spécifique.
