## ⚛️ React {#react}

[![Sathya Gunasekaran - Quelle est la suite pour le compilateur React ?](/emails/issues/208/compiler.jpg)](https://www.youtube.com/watch?v=qd5yk2gxbtg)

🎥 [**Sathya Gunasekaran \- Quelle est la suite pour le compilateur React ?**](https://www.youtube.com/watch?v=qd5yk2gxbtg)

J'ai enfin pris le temps de regarder cette excellente conférence qui annonce la [version bêta du compilateur React](https://react.dev/blog/2024/10/21/react-compiler-beta-release), mais dévoile également d'autres aspects intéressants. Le compilateur ne se limite pas à la mémoïsation automatique, mais constitue une plateforme pour développer davantage d'optimisations. Trois fonctionnalités à venir ont été présentées :

- Configuration des types : Vous pouvez aider le compilateur à mieux fonctionner en fournissant des informations de type supplémentaires.
- JSX Inlining : Le compilateur peut supprimer les appels JSX inutiles et les remplacer par des objets JSX préconstruits. `jsx('div')` devient `{type: 'div'}`.
- JSX Outlining : Le compilateur peut extraire automatiquement les sous-composants, qui peuvent ensuite être optimisés indépendamment. Cela est particulièrement utile pour les éléments de liste lors de l'utilisation de `array.map` et de JSX intégré ; le compilateur peut créer un composant `<Item>` (voir la capture d'écran de la présentation, sinon [cette PR](https://github.com/facebook/react/pull/30956) l'explique bien).

## 💸 Sponsor {#sponsor2}

[![Introducing @clerk/nextjs v6 - Next.js 15 compatible](/emails/issues/208/clerk.jpg)](https://go.clerk.com/763LGC2)

**[Présentation de @clerk/nextjs v6 - Compatible avec Next.js 15](https://go.clerk.com/763LGC2)**

L'équipe Next.js a annoncé la sortie stable de Next.js 15, et Clerk poursuit la tradition d'un support (presque) le jour même pour les nouvelles versions majeures de Next.js avec la sortie de `@clerk/nextjs` v6. Avec la v6, l'assistant `auth()` est désormais asynchrone, `<ClerkProvider>` utilise par défaut le rendu statique au lieu du rendu dynamique, et la prise en charge du pré-rendu partiel a été introduite.

## 📱 React-Native {#react-native}

[![React Navigation 7.0](/emails/issues/208/reactnavigation7.jpg)](https://reactnavigation.org/blog/2024/11/06/react-navigation-7.0/)

[**React Navigation 7.0**](https://reactnavigation.org/blog/2024/11/06/react-navigation-7.0/)

React Navigation v7 est désormais disponible et devrait alimenter le prochain Expo Router v4 livré avec Expo SDK 52, tous deux en version bêta. Mes points forts préférés sont :

- Préchargement des écrans : Vous pouvez préparer les écrans à l'avance avant d'y naviguer et améliorer les performances perçues. Celui-ci est vraiment excitant et pourrait utiliser les fonctionnalités Concurrent React à l'avenir.
- API Statique : Simplifie la configuration du navigateur et facilite l'utilisation avec TypeScript et le deep linking.

[**Nous venons de terminer la migration de l'application mobile Shopify vers React Native**](https://threadreaderapp.com/thread/1853619638141071573.html)

Mustafa (directeur de l'ingénierie) a annoncé sur X que Shopify a terminé la migration de son application phare avec 586 écrans vers React Native, avec des résultats très positifs. J'espère que nous aurons bientôt un article de blog détaillé.

- 1,8 million de lignes de code supprimées
- 86 % de partage de code entre iOS et Android
- 59 % de chargement d'écran plus rapide
- 44 % de lancement d'application plus rapide
- 63 % de webviews plus rapides
- Réduction des plantages d'application
- Amélioration des notes des applications dans les stores
