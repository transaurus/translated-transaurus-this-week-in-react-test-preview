## ⚛️ React {#react}

[![Révélé : L'API expérimentale d'animations de React](/emails/issues/217/viewtransition.jpg)](https://motion.dev/blog/reacts-experimental-view-transition-api)

**[Révélé : L'API expérimentale d'animations de React](https://motion.dev/blog/reacts-experimental-view-transition-api)**

Le créateur de Motion explique les compromis d'animation liés à l'utilisation des View Transitions et pourquoi il est logique pour React de l'intégrer directement dans son réconciliateur via un nouveau composant `<ViewTransition>`. Cet article interactif montre un aperçu précoce de la future fonctionnalité de React, incluant certains détails d'implémentation actuels, et explique comment elle chevauche légèrement les animations de disposition de Motion.

Gardez à l'esprit que cette nouvelle API passionnante est encore en développement actif, sujette à modification, et de nouvelles PR continuent d'arriver, notamment :

- 👀 [PR React Core - Prise en charge uniquement de View Transitions v2](https://github.com/facebook/react/pull/31996) : Le nouveau composant exploitera les API récentes de la spécification v2 telles que `view-transition-class`, nécessite Safari 18.2+ et n'est toujours pas pris en charge dans Firefox.
- 👀 [PR React Core - Noms de classes View Transition basés sur le type d'événement](https://github.com/facebook/react/pull/32050)
- 👀 [PR Next.js - Drapeau `experimental.viewTransition`](https://github.com/vercel/next.js/pull/74659) : Vous pouvez tester la fonctionnalité dans Next.js 15.2 canary 4.

## 📱 React-Native {#react-native}

[![Aperçu de l'hébergement EAS](/emails/issues/217/easdeploy.jpg)](https://expo.dev/blog/expo-announces-eas-hosting-service)

**[Aperçu de l'hébergement EAS](https://expo.dev/blog/expo-announces-eas-hosting-service)**

Expo a dévoilé ce que nous attendions, la première plateforme universelle d'hébergement d'applications qui concrétise la vision d'apporter la DX web au monde multiplateforme. Elle nous permet de déployer de manière transparente les fonctionnalités liées au serveur d'Expo, notamment Expo pour le Web, les routes d'API et les composants serveur React universels. Elle prend en compte les subtilités des applications mobiles, comme la synchronisation des versions backend/app. Vous pouvez utiliser soit la CLI, soit l'intégration continue (EAS Workflows). Sous le capot, elle utilise Cloudflare Workers. Il existe un plan gratuit.
