## ⚛️ React {#react}

**[Génération d'images OG par Vercel : des cartes sociales rapides et dynamiques à la périphérie](https://vercel.com/blog/introducing-vercel-og-image-generation-fast-dynamic-social-card-images)**

Vercel présente une solution très efficace pour générer des cartes sociales dynamiques dans les Edge Functions de Vercel. Pratique : utilisation possible de JSX, de composants React (sans état) et de Tailwind sans configuration supplémentaire. Je recommande [ce fil](https://twitter.com/shuding_/status/1579607964549513217) expliquant le fonctionnement, très intéressant ! Ils utilisent même Yoga, le moteur de layout de React-Native 🤭

**[L'avenir du rendu dans React](https://prateeksurana.me/blog/future-of-rendering-in-react/)**

Article bien illustré présentant les modèles de rendu de React 18 et au-delà. Vous y trouverez des informations sur le fonctionnement du streaming SSR, Suspense, React lazy, l'implémentation des feature flags avec les Server Components...

**[Surmonter les problèmes courants des projets React](https://dinosaurs-with-jetpacks.com/posts/react-problems)**

Encourage à découpler nos applications JavaScript du framework (React ou autre). Prône un retour à plus de simplicité. Certaines prises de position ne feront pas l'unanimité.

**[Material UI dans Storybook](https://storybook.js.org/blog/material-ui-in-storybook/)**

Tutoriel avec une section particulièrement intéressante expliquant comment automatiser la création de contrôles Storybook en réutilisant les types TypeScript de la bibliothèque UI sous-jacente. Très utile pour le transfert de props. Prend même en charge les helpers comme `Pick` et `Omit`.

- 👤 Sathya Gunasekaran (ex V8) [rejoint l'équipe cœur de React](https://twitter.com/_gsathya/status/1578816177371250688), très probablement pour [travailler sur le compilateur React-Forget](https://twitter.com/Huxpro/status/1578865666010263552)
- 👥 [Intervenants de la Next.js Conf 2022](https://twitter.com/vercel/status/1578401243243741185)
- 📖 Documentation bêta : [React.Children](https://twitter.com/dan_abramov/status/1578457525628047360) + [useId](https://twitter.com/dan_abramov/status/1577283961763467265) + [createElement](https://twitter.com/dan_abramov/status/1577463721193897985) + [useImperativeHandle](https://twitter.com/dan_abramov/status/1577717226269966336)
- 📖 [Améliorations de la documentation Next.js sur Vercel](https://twitter.com/leeerob/status/1577788730349436928)
- 🌶 [React et Next.js sont MORTS — Quelque chose de nouveau les remplace (enfin) (Pour de bon)](https://javascript.plainenglish.io/react-and-next-js-is-dead-something-new-is-finally-replacing-it-for-good-c792c48806f6) : explique comment le framework Qwik présente un modèle supérieur à React. Titre putaclic, mais article très intéressant.
- 📜 [Construire un moteur de rendu personnalisé React - Partie 1](https://www.markcodes.dev/posts/build-react-custom-renderer-part-1) : nouvelle série montrant comment créer un moteur de rendu React. Pour l'instant facile à suivre, mais ne prend pas encore en charge les mises à jour.
- 📜 [Remix, React et l'état](https://blog.jim-nielsen.com/2022/remix-react-and-state/) : propose un modèle mental pour Remix qui réconcilie l'état React de bout en bout / fullstack.
- 📜 [Comment écrire votre propre bibliothèque de gestion d'état](https://judehunter.dev/blog/how-to-write-your-own-state-management-library) : utile pour comprendre le fonctionnement d'une bibliothèque de gestion d'état et implémenter un système de sélecteurs.
- 📜 [Comment récupérer des données dans React en tenant compte des performances](https://www.developerway.com/posts/how-to-fetch-data-in-react) : explique pourquoi la récupération de données dans React n'est pas aussi simple qu'il n'y paraît. Les performances sont liées à l'UX/perception. Détaille les cascades, limites de requêtes navigateur...
- 📜 [Éviter les états d'interface impossibles avec React, Typescript et xState](https://whereisthemouse.com/avoid-impossible-ui-states-with-react-typescript-and-xstate) : les types union discriminés de TypeScript évitent les états impossibles, et XState évite les transitions d'état impossibles.
- 📜 [J'ai changé d'avis sur l'écriture de nouveaux frameworks JavaScript](https://whitep4nth3r.com/blog/write-a-new-javascript-framework/) : seuls 4 frameworks supportent SSG + SSR : Eleventy, Next.js, Gatsby, Redwood.
- 📜 [Comment utiliser Sandpack pour des démonstrations de code](https://www.ctnicholas.dev/articles/how-to-use-sandpack-for-code-demos) : article [mis à jour avec les dernières fonctionnalités Sandpack](https://twitter.com/ctnicholasdev/status/1577683748878884864), cet outil Codesandbox pour créer des environnements de code live. Utilisé par la doc bêta React, Josh Comeau...
- 📦 [Tremor 1.0](https://www.tremor.so/) : bibliothèque de composants UI React pour créer des tableaux de bord analytiques.
- 🐦 [W3C Trusted Types + React + XSS](https://twitter.com/sebastienlorber/status/1579485631260286977) : une nouvelle proposition HTML pourrait fournir une solution native contre les vulnérabilités XSS, et peut-être supprimer le besoin de `dangerouslySetInnerHTML` ?
