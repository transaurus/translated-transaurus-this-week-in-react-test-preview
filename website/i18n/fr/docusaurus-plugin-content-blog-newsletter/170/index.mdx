Bonjour à tous !

Cette semaine, nous avons eu de très bons articles sur React, mais aussi des versions intéressantes de Waku et Remix. La communauté a partagé des retours mitigés mais intéressants sur les React Server Components, concernant la complexité et la propriété.

Côté React Native, c'est calme. Nous attendons impatiemment l'Expo SDK 50 qui cette fois est publié bien plus tard que la bêta. Nous attendons également le lancement officiel de VisionOS, une nouvelle plateforme que React Native prendra en charge.

Je viens de [fêter mes 10 ans d'expérience avec React](https://twitter.com/sebastienlorber/status/1747307299356447148) 🙉 Il est difficile de croire à quel point ma carrière a changé grâce à React, alors que je n'aimais vraiment pas JavaScript auparavant.

## ⚛️ React {#react}

[![Waku 0.19](/emails/issues/170/waku.jpg)](https://waku.gg/blog/introducing-create-pages)

**[Waku 0.19](https://waku.gg/blog/introducing-create-pages)**

Cette version introduit une nouvelle API `createPage` pour créer des mises en page et des pages de manière programmatique. Cela marque notamment le passage de Waku d'une expérience d'apprentissage des React Server Components à un véritable framework React prêt pour la production. Un routage de page basé sur des fichiers léger est prévu pour la v0.20.

**[Composants Donut Délicieux](https://frontendatscale.com/blog/donut-components/)**

Cet article interactif joyeux et génial utilise l'analogie du donut (composant client avec un trou « children ») pour expliquer comment les composants client/serveur peuvent être entrelacés. Il aborde également des sujets de composition RSC plus avancés. Aujourd'hui, j'ai appris cette astuce élégante de render prop avec les actions serveur !

- 📜 [Un peu agacé par React](https://blog.cassidoo.co/post/annoyed-at-react/) : Cassidy partage un point de vue raisonnable sur l'état actuel de React qui semble trouver un écho dans la communauté, notamment 🐦 [Tanner Linsley utilisant React avec un ressentiment croissant](https://twitter.com/tannerlinsley/status/1746970043836158330).
- 📜 [React, où vas-tu ?](https://dev.to/matfrana/react-where-are-you-going-5284) : Matteo s'appuie sur l'analyse de Cassidy et explique sa frustration concernant la propriété de React et la complexité croissante. Il plaide pour un framework financièrement durable et piloté par la communauté. Ce qui m'a frappé, c'est que l'ajout du support RSC à leur bibliothèque React-Bricks a diminué la DX (confirmé par plusieurs autres, 🐦 [tweet](https://twitter.com/sebastienlorber/status/1747643069460021405)).
- 📜 [Flux et composants serveur React](https://minimalistweb.dev/p/streams-and-react-server-components) : Même en dehors des RSC, c'est une excellente plongée dans les API de flux par un contributeur de Waku.
- 📜 [Rendu côté serveur de dates locales sans FOUC](https://blog.6nok.org/server-side-rendering-local-dates-without-fouc/) : Il est courant d'obtenir des erreurs d'hydratation React en essayant d'afficher l'heure dans le fuseau horaire utilisateur. Cette approche suggère d'injecter un script pour modifier l'heure rendue en SSR juste avant l'hydratation de React pour éviter l'erreur.
- 📜 [Comment configurer le relais client-serveur dans Next.js](https://ericbureltech.hashnode.dev/how-to-setup-client-server-relaying-in-nextjs) : Explique comment afficher une valeur dynamique côté serveur, puis laisser SWR prendre le relais côté client.

- 📦 [Remix 2.5 - Mode SPA](https://github.com/remix-run/remix/blob/main/CHANGELOG.md#v250) : Déployez des applications Remix sans serveur (par exemple sur GitHub Pages) en utilisant le mode SPA, générant un seul fichier `index.html`.
