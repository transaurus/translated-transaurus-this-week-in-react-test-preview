## ⚛️ React {#react}

[![Devtools pour les React Server Components](/emails/issues/155/rscdevtools.jpg)](https://www.alvar.dev/blog/creating-devtools-for-react-server-components)

[**Devtools pour les React Server Components**](https://www.alvar.dev/blog/creating-devtools-for-react-server-components)

Suite à la sortie des React Server Components, Alvar a remarqué que la communication réseau entre client et serveur utilisait un format difficile à déchiffrer. Il a donc commencé à travailler sur un site [RSC Parser](https://rsc-parser.vercel.app/) permettant d'inspecter plus facilement les payloads. Cela fonctionnait, mais l'expérience utilisateur n'était pas idéale. Depuis, il développe une véritable extension navigateur fonctionnant comme les DevTools Réseau de Chrome, avec une chronologie montrant les payloads dans le temps et prenant même en charge le streaming avec Suspense. Cet outil lui a déjà permis de détecter des payloads trop volumineux sur son blog et de les optimiser.

C'est un outil prometteur, encore en développement et à surveiller. L'extension n'est pas encore disponible dans les boutiques de navigateurs, mais le sera bientôt. Quelques démos vidéo dans ce [fil Twitter](https://twitter.com/sebastienlorber/status/1689203357368365056).

[![Mise en cache dans Next.js](/emails/issues/155/cachinginnextjs.jpg)](https://nextjs.org/docs/app/building-your-application/caching)

[**Mise en cache dans Next.js**](https://nextjs.org/docs/app/building-your-application/caching)

Une nouvelle page de documentation Next.js explique en détail le fonctionnement des 4 niveaux de cache, leur raison d'être et comment les contourner :

- Mémorisation des requêtes
- Cache de données
- Cache de route complet
- Cache du routeur

Il n'est pas possible de contourner le cache du routeur : un problème mis en évidence dans [cette vidéo](https://www.youtube.com/watch?v=_yhSh4g0NSk).

**[Une alternative non-cloud à Google Forms qui a tout](https://surveyjs.io/?utm_source=thisweekinreact&utm_medium=email)**

Si vous cherchez à mettre en place un système robuste de gestion de formulaires entièrement intégré à votre infrastructure informatique, vous permettant de conserver un contrôle total sur le flux de données, sans passer des mois ni dépenser des milliers de dollars pour le déployer, alors SurveyJS est la solution idéale !

SurveyJS est une suite produit de quatre bibliothèques JavaScript open-source permettant de créer et modifier plusieurs formulaires dynamiques basés sur JSON via un constructeur drag-and-drop, de les afficher dans votre application React et de stocker les données sensibles d'enquêtes au sein de votre infrastructure interne. Vous pouvez ensuite générer vos enquêtes personnalisées sous forme de PDF modifiables dans un navigateur et visualiser les résultats via des graphiques et tableaux interactifs.
