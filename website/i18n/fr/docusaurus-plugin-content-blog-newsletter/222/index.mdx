## ⚛️ React {#react}

[![Mise en sommeil de Create React App](/emails/issues/222/cra.jpg)](https://react.dev/blog/2025/02/14/sunsetting-create-react-app)

**[Mise en sommeil de Create React App](https://react.dev/blog/2025/02/14/sunsetting-create-react-app)**

Attendue depuis longtemps, CRA a reçu les dernières corrections de React 19 et est désormais officiellement déprécié. Cet article expose les limites de CRA et des outils de build concernant le routage, les cascades, le découpage de code et autres. L'équipe React nous encourage à adopter un framework comme Next.js, React Router ou Expo pour résoudre ces problèmes. Cependant, vous pouvez toujours [construire une application React à partir de zéro](https://react.dev/learn/build-a-react-app-from-scratch) avec Vite, Parcel, Rsbuild ou d'autres outils de build modernes.

L'annonce initiale a suscité des débats animés sur X et GitHub, principalement parce qu'elle mettait excessivement l'accent sur les frameworks au détriment des outils de build comme Vite. Elle a été [rapidement mise à jour](https://github.com/reactjs/react.dev/pull/7624) pour répondre à ces préoccupations. Cela a également conduit à des opportunités de collaboration potentielles entre [Evan You (Vite) et l'équipe React](https://github.com/facebook/react/pull/31768#issuecomment-2666856223) pour travailler ensemble sur une approche "officielle" et opinée d'intégration de Vite avec les React Server Components, avec un engagement à maintenir cette intégration dans le temps dans le dépôt React.

Actualités connexes :

- 📜 [Utiliser un framework pour créer des applications React Native](https://reactnative.dev/blog/2024/06/25/use-a-framework-to-build-react-native-apps) : Il y a quelques mois, l'équipe React Native a fait une annonce similaire, affirmant que « soit vous utilisez un framework, soit vous construisez votre propre framework ».
- 📦 [Create TanStack Router App](https://github.com/TanStack/create-tsrouter-app) : La première contribution de Jack Herrington au TanStack est une alternative à Create-React-App, également présentée dans une [vidéo](https://www.youtube.com/watch?v=10J6RyMOxN0)
- 📦 [CRA to Parcel](https://github.com/parcel-bundler/cra-to-parcel) : Un script pour migrer une application Create React App non éjectée vers Parcel

## 📱 React-Native {#react-native}

[![React Native 0.78 - React 19 et plus](/emails/issues/222/rn.jpg)](https://reactnative.dev/blog/2025/02/19/react-native-0.78)

**[React Native 0.78 \- React 19 et plus](https://reactnative.dev/blog/2025/02/19/react-native-0.78)**

Je parie que vous ne vous attendiez pas à une version de React Native si tôt, mais l'équipe a récemment décidé de publier des versions plus petites plus fréquemment. Voici les points forts :

- Prise en charge de React 19 !
- Processus simplifié pour activer le React Compiler.
- Le streaming des logs Metro est de retour en 0.78 et 0.77.1, mais en option avec un nouveau flag `--client-logs`, et toujours déprécié.
- `<Image>` prend en charge les ressources XML Android, débloquant des améliorations potentielles de performances.
- `ReactNativeFactory` sur iOS pour faciliter l'utilisation brownfield dans n'importe quel ViewController

React 19 lui-même débloque de nombreuses fonctionnalités intéressantes :

- Actions (transitions asynchrones), `useActionState()`, `useOptimistic()`, tous utiles pour soumettre des formulaires et gérer les états de chargement, d'erreur et optimiste
- Utiliser `ref` comme prop régulière au lieu de `forwardRef()`, prenant également en charge une fonction de nettoyage de ref
- Utiliser `<Context>` au lieu de `<Context.Provider>`
- Lire les contextes et promesses avec `use()`
