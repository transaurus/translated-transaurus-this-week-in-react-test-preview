## ⚛️ React {#react}

[![Next.js 15.2](/emails/issues/224/next.jpg)](https://nextjs.org/blog/next-15-2)

[**Next.js 15.2**](https://nextjs.org/blog/next-15-2)

Vercel a publié une nouvelle version mineure de Next.js, avec des améliorations majeures de l'expérience développeur (DX). Les points forts incluent :

- Interface d'erreur repensée et traces de pile améliorées : Une expérience de débogage repensée exploitant la nouvelle API React [`captureOwnerStack()`](https://react.dev/reference/react/captureOwnerStack)
- Métadonnées en streaming : Les métadonnées asynchrones ne bloqueront plus le rendu des pages ou les transitions de page côté client
- Améliorations des performances de Turbopack : Temps de compilation plus rapides et réduction de l'utilisation de la mémoire
- Transitions de vue React (expérimental) : Support expérimental de la nouvelle API de transitions de vue de React
- Middleware Node.js (expérimental) : Support expérimental de l'utilisation du runtime Node.js dans le middleware

[![TanStack Form 1.0](/emails/issues/224/form.jpg)](https://tanstack.com/blog/announcing-tanstack-form-v1)

[**TanStack Form 1.0**](https://tanstack.com/blog/announcing-tanstack-form-v1)

Une nouvelle bibliothèque TanStack, des années de développement, vient d'être publiée en version stable. L'article d'annonce met en lumière un sous-ensemble intéressant de ses fonctionnalités :

- Indépendant du framework, headless : Prend en charge 5 frameworks, dont React utilisé dans les exemples de l'article
- Sécurité de type extrême : Je dois admettre que `<form.Field>` est une API plutôt cool
- Validation par schéma : Prise en charge de Zod, Valibot, ArkType, Standard Schema
- Validation asynchrone, debouncing intégré, AbortSignal
- Isomorphe, multi-plateforme : partagez la logique de validation entre client et serveur

## 📱 React-Native {#react-native}

[![Lynx: Unlock Native for More](/emails/issues/224/lynx.jpg)](https://lynxjs.org/blog/lynx-unlock-native-for-more)

[**Lynx : Libérez le potentiel natif pour aller plus loin**](https://lynxjs.org/blog/lynx-unlock-native-for-more)

ByteDance vient d'annoncer Lynx, sa technologie native multiplateforme. Elle est largement utilisée dans leurs applications de production, dont TikTok ! Avec des liaisons React, c'est un concurrent sérieux à React Native, et le support d'autres frameworks est attendu. Les choix architecturaux sont intéressants, avec un modèle à double thread et différents environnements d'exécution. Elle prend en charge certaines API CSS et Web que même React Native n'a toujours pas, et utilise des éléments intrinsèques JSX intégrés pratiques comme `<view>`.

L'ensemble des fonctionnalités au lancement est impressionnant, mais il y a aussi quelques éléments surprenants que vous pourriez ne pas apprécier, comme l'utilisation de React 17 / Preact, ou des éléments personnalisés sur le web. Le temps dira si la technologie est à la hauteur de ses promesses et peut vraiment rivaliser avec le joueur établi. Dans tous les cas, la concurrence est saine.
