## 📱 React-Native {#react-native}

[![React Native WebGPU](/emails/issues/196/rn-webgpu.jpg)](https://www.youtube.com/watch?v=i-DGY98cTjk)

**🎥 [React Native WebGPU](https://www.youtube.com/watch?v=i-DGY98cTjk)**

React Native WebGPU vient de sortir, offrant une API unifiée pour exploiter le GPU dans React Native pour des graphiques et calculs avancés ! En réutilisant le standard des API Web, nous pourrons intégrer facilement des exemples de code et bibliothèques WebGPU existants comme ThreeJS directement dans React Native.
Au-delà des graphismes, React Native WebGPU peut être utilisé pour des calculs matriciels, le rendant intéressant pour les algorithmes de machine learning sur appareil. Construit avec Nitro Modules pour une interopérabilité fluide TypeScript/C++. Actuellement en préversion technique, annonçant des développements passionnants !

---

**L'avenir des modules React Native**

Des moments passionnants pour les développeurs React Native ! Marc Rousavy révolutionne avec NitroModules, une nouvelle façon d'écrire du code natif 📦 [qui surpasse les solutions existantes en tests de vitesse](https://github.com/mrousavy/NitroBenchmarks). Il a rencontré des limites avec JSI dans VisionCamera. Le projet est déjà stable côté C++ et utilisé dans 🐦 [react-native-unistyles](https://x.com/jpudysz/status/1822541758137303233) et react-native-webgpu, avec les liaisons Swift/Kotlin en cours.

👀 [Parallèlement, une nouvelle proposition pour pousser Swift dans les TurboModules est apparue.](https://github.com/okwasniewski/discussions-and-proposals/blob/22edd2c7853aaf2ee654b57f4521e43d668103fb/proposals/0000-swift-turbo-modules.md) Ces changements faciliteront le développement des modules natifs en abandonnant Objective-C/C++. Ces mises à jour amélioreront considérablement la création de modules natifs !
