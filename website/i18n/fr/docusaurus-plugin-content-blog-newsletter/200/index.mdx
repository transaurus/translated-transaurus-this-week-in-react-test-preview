## ⚛️ React {#react}

[![Josh Comeau - L'utilité indéniable de CSS `:has`](/emails/issues/200/css-has.jpg)](https://www.joshwcomeau.com/css/has/)

**CSS `:has` pour les développeurs React**

Cette semaine, nous avons deux excellents articles sur le nouveau sélecteur CSS `:has`, et comment les développeurs React peuvent l'exploiter pour simplifier/supprimer leur code JS tout en améliorant les performances. Cette nouvelle fonctionnalité CSS bénéficie d'une bonne prise en charge (92%) mais ne peut être polyfillée, donc pour l'instant il vaut mieux l'utiliser pour des améliorations subtiles et progressives.

Lire dans cet ordre :

- 📜 [Josh Comeau \- The Undeniable Utility Of CSS `:has`](https://www.joshwcomeau.com/css/has/) : Superbe article d'introduction interactif, incluant également des commentaires et des exemples pertinents pour les développeurs React, comme le blocage du défilement de la page globale lors de l'ouverture d'une modale avec un attribut de données au lieu d'un `useEffect`.
- 📜 [Nadia Makarevich \- Replacing React code with CSS `:has` selector](https://www.developerway.com/posts/replacing-react-with-css) : Donne quelques exemples intéressants supplémentaires basés sur React, comme la gestion du focus, la mise en évidence des lignes de tableau ou des groupes de champs de formulaire en fonction de l'état de saisie.

Un grand pouvoir implique de grandes responsabilités. N'oubliez pas de garder vos composants React encapsulés et d'éviter autant que possible les styles qui fuient en dehors des limites des composants.

## 📱 React-Native {#react-native}

La semaine dernière, Wrocław a été le théâtre de React Universe, où des annonces majeures ont enthousiasmé la communauté React Native. L'un des moments marquants a été le dévoilement de React Native 0.76, dont le lancement est prévu pour fin octobre. Cette version apporte des avancées significatives, notamment la Nouvelle Architecture activée par défaut et l'introduction de React Native DevTools entièrement repensés. Elle est déjà disponible en tant que [candidat à la version 0.76](https://github.com/facebook/react-native/releases/tag/v0.76.0-rc.0) !

[![React Native DevTools](/emails/issues/200/react-native-devtools.jpg)](https://www.youtube.com/watch?v=b48Lax2-jOQ&t=4350s)

- 🎥 [Alex Hunt - The New React Native Debugger - Codename "Fusebox"](https://www.youtube.com/watch?v=b48Lax2-jOQ&t=4350s)
  Les outils de débogage pour React Native ont longtemps été fragmentés, mais les nouveaux "React Native DevTools" introduisent une expérience unifiée avec un nouveau backend alimenté par le Chrome DevTools Protocol (CDP). Cette architecture, supportée par les IDE et Chrome DevTools, promet des points d'arrêt plus fiables et des rechargements stables. Elle prend en charge à la fois les nouvelles et les anciennes architectures, se positionnant comme la solution de débogage incontournable pour les applications React Native. Le débogage direct d'Hermes via Chrome sera progressivement abandonné, bien que les fonctionnalités d'inspection des performances et du réseau soient encore en cours de développement.

[![Hermes Performance Chart](/emails/issues/200/hermes-perf.jpg)](https://www.youtube.com/watch?v=pfeTPSai6vc&t=8702s)

- 🎥 [Tzvetan Mikov - Hermes: Better performance with runtime bytecode translation](https://www.youtube.com/watch?v=pfeTPSai6vc&t=8702s)  
  Static Hermes, présenté lors de React Native Europe l’année dernière, a suscité un intérêt significatif en remettant en question les standards existants des moteurs JavaScript. Bien qu’encore en développement, Static Hermes propose une approche novatrice en compilant JavaScript en code machine natif ou en bytecode typé via l’analyse des annotations de type, avec un potentiel de performances comparables au C++. Cependant, ce flux de travail présente des limites : les mises à jour over-the-air (OTA) ne fonctionneront plus, les outils de build JavaScript actuels comme Babel ne conservent pas les annotations de type, et une quantité considérable de code non typé reste utilisée. Pour relever ces défis, l’équipe Hermes a introduit la traduction de bytecode à l’exécution, qui promet des gains de performances jusqu’à 12 fois plus rapides pour la compilation native. Ces innovations seront disponibles dans Hermes v2, après des tests internes approfondis chez Meta.
