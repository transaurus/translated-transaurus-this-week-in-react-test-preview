## ⚛️ React {#react}

📜 **[Modèles de récupération de données dans les applications monopages](https://martinfowler.com/articles/data-fetch-spa.html)**

Une lecture longue (~10k mots), mais un excellent article ! Le titre ne rend pas justice au contenu, car il couvre bien plus que la simple récupération de données : concepts de base de React, Suspense, gestion d'état asynchrone, découpage de code, pré-récupération et récupération parallèle pour éviter les cascades. Alors que les performances des applications web font débat sur X (Rails vs. React, décision MS Edge...), c'est une excellente ressource pour rafraîchir les bases et créer une expérience utilisateur optimale !

- 💸 [Meilisearch - Build search as you type for React](https://www.meilisearch.com/with/react?utm_campaign=sponsoring&utm_source=thisweekinreact)
- 👀 [React Router PR - defineRoute API](https://github.com/remix-run/react-router/pull/11596)
- 🐦 [shadcn/ui is working on adding charts](https://x.com/shadcn/status/1795920488797188525)
- 🐦 [Remix working inside a Cloudflare Durable Object 🤯](https://x.com/threepointone/status/1796223745243119890)
- 🐦 [React 19.0.0-rc.0](https://x.com/acdlite/status/1797668537349328923): Andrew Clark says “This is the exact build we'll release as 19.0, unless an issue is reported that requires a breaking change.”
- 🐦 [Upcoming Spline (3D rendering) and NextJS RSC integration](https://x.com/rauchg/status/1798021886657650711)
- 📜 [I tried React Compiler today, and guess what …](https://www.developerway.com/posts/i-tried-react-compiler): A super interesting and detailed article about React Compiler and real-world use cases. Seems like we’re not at the point yet where we can just ignore everything about memoization because the Compiler takes care of it, and apart from this conclusion the article is also a great reminder of how memoization works exactly.
- 📜 [Do you still need Framer Motion?](https://motion.dev/blog/do-you-still-need-framer-motion): The creator of the popular React animation library explains you might not need his library anymore thanks to 5 great new/upcoming CSS features.
- 📜 [Sneaky React Memory Leaks II: Closures Vs. React Query](https://schiener.io/2024-05-29/react-query-leaks): This follows up on last week's great article. This time it shows how React Query might be affected as well by memory leaks, and how to fix them. Using custom hooks looks like a good general solution to this problem.
- 📜 [An even faster Microsoft Edge](https://blogs.windows.com/msedgedev/2024/05/28/an-even-faster-microsoft-edge/): Microsoft was using React for some elements of the UI in its Edge browser and is now abandoning this approach. See the video linked below for more context.
- 📜 [Speeding up the JavaScript ecosystem - Server Side JSX](https://marvinh.dev/blog/speeding-up-javascript-ecosystem-part-9/): The JSX runtime of React can lead to many object allocations. Marvin (Preact/Deno) shows it's possible to precompile JSX to improve server rendering of JSX and reduce GC. Maybe React could use that as well?
- 📜 [Type-safe module mocking in Storybook](https://storybook.js.org/blog/type-safe-module-mocking/): Very elegant way to leverage subpath imports for module mocking.
- 📜 [Multiple action handlers with Zod in Remix](https://www.jacobparis.com/content/zod-actions-remix): If you target one action endpoint from multiple components, this article shows you how to combine the different Zod schemas via discriminated union.
- 📜 [Embracing Slow Networks: Improving User Experience](https://www.epicweb.dev/embracing-slow-networks-improving-user-experience)
- 📜 [Cleaner components with useSuspenseQuery](https://swizec.com/blog/cleaner-components-with-use-suspense-query/) & [Why useSuspenseQuery works](https://swizec.com/blog/why-use-suspense-query-works/)
- 📜 [Explore new hooks coming up in React 19](https://blog.saeloun.com/2024/06/05/new-hooks-in-react-19/)
- 📦 [React 19.0.0-rc.0](https://www.npmjs.com/package/react/v/19.0.0-rc.0): Andrew Clark said "This is the exact build we'll release as 19.0, unless an issue is reported that requires a breaking change."
- 📦 [Docusaurus 3.4](https://docusaurus.io/blog/releases/3.4): A new way to centrally manage tags (great for consistency), an experimental hash router (great for offline browsing without a web server), and an option to use namespaces for local storage (useful if you’re deploying multiple instances of Docusaurus on the same domain).

## 📱 React-Native {#react-native}

![Simon Grimm: Construire React Native avec Expo](/emails/issues/188/rn188.jpg)

🎥 **[Toutes les façons de construire votre application React Native avec Expo](https://www.youtube.com/watch?v=cs-zgHjt5RQ)**

Tous les chemins mènent à Rome, et il semble qu'il y ait presque autant de façons de construire une application Expo. Cela nous donne aux développeurs beaucoup de puissance et de liberté, mais peut aussi parfois mener à la confusion - pas seulement pour les débutants ! Heureusement, Simon a réalisé une excellente vidéo comparant les différentes approches via Expo Go, Expo Prebuild, Xcode, Android Studio et EAS. C'est aussi une belle occasion de féliciter Expo pour avoir atteint [30 000 étoiles sur GitHub !](https://x.com/expo/status/1796262781584654492)

- 💸 [React Native Mastery - Le seul cours dont vous avez besoin pour maîtriser React Native & Expo](https://notjust.dev/react-native-mastery?utm_source=twir&utm_medium=email)
- 🐦 [La démo d'application IA la plus puissante, avec RSC via Expo Router](https://x.com/Baconbrix/status/1795804445738963283) : Nous avons déjà parlé de la démo RSC d'Evan Bacon. Ce thread sur X résume bien les promesses et inclut une comparaison vidéo montrant à quel point l'UI mobile peut être performante, notamment dans le contexte des applications de chat LLM.
- 📜 [Guide de sécurité React Native](https://ospfranco.com/react-native-security-guide/) : Oscar aborde un sujet trop peu discuté. Une bonne introduction couvrant les secrets (TLDR : Ne les stockez pas sur l'appareil) et le chiffrement des données utilisateur (y compris l'utilisation de la biométrie).
- 📜 [Transitions fluides : Du natif à React Native](https://commerce.nearform.com/blog/2024/seamless-transitions-from-native-to-react-native) : Des pistes pour le développement brownfield : Comme RN crée de véritables applications natives, l'introduction de RN dans une application existante ou la migration vers RN peut être transparente pour les utilisateurs finaux.
- 📦 [React Native 0.74.2](https://github.com/facebook/react-native/releases/tag/v0.74.2) : Version corrective, mettant à jour certaines dépendances (typescript-eslint, CLI) et corrigeant des bugs.
- 📦 [React Native Reanimated 3.12.0](https://github.com/software-mansion/react-native-reanimated/releases/tag/3.12.0) : Introduit un nouveau hook useComposedEventHandler qui simplifie la fusion de plusieurs gestionnaires d'événements.
- 🎙️ [RNR 298 - Create Expo Stack, NativeWind UI, & Plugin Pro avec Dan Stepanov](https://reactnativeradio.com/episodes/rnr-298-create-expo-stack-nativewind-ul-plugin-pro-with-dan-stepanov)
- 🎥 [Rocket Ship 42 - React Native est vivant ! Retour sur App.js avec Áron Berezkin](https://www.youtube.com/watch?v=GeUfrojkoo0)
