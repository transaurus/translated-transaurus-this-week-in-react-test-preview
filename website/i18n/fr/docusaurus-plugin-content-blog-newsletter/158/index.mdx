## ⚛️ React {#react}

[![Astro 3.0](/emails/issues/158/astro3.jpg)](https://astro.build/blog/astro-3/)

[**Astro 3.0**](https://astro.build/blog/astro-3/)

Cette nouvelle version d'Astro innove notamment grâce à l'utilisation de la nouvelle View Transition API, permettant une navigation fluide et animée entre les pages tout en préservant l'état des composants affichés. Cela réduit le besoin d'une SPA dans de nombreux cas d'usage.

Autres améliorations notables :

- Optimisation d'images stable
- Performances de rendu accrues : les composants Astro rendent 30 à 75% plus vite
- Améliorations SSR pour Serverless : nouvelles méthodes de connexion à votre plateforme d'hébergement
- Améliorations HMR pour JSX : prise en charge de Fast Refresh pour React et Preact
- Sortie de build optimisée : HTML plus propre et performant

Si vous cherchez à mettre en place un système robuste de gestion de formulaires entièrement intégré à votre infrastructure informatique, vous permettant de conserver un contrôle total sur le flux de données, sans passer des mois ni dépenser des milliers de dollars pour le déployer, alors SurveyJS est la solution idéale !

SurveyJS est une suite de quatre bibliothèques JavaScript open-source qui permettent de créer et modifier plusieurs formulaires dynamiques basés sur JSON via un constructeur par glisser-déposer, de les afficher dans votre application React et de stocker les données sensibles des enquêtes au sein de votre propre infrastructure. Vous pouvez ensuite afficher vos enquêtes et formulaires personnalisés sous forme de fichiers PDF modifiables dans un navigateur et visualiser les résultats avec des graphiques et tableaux interactifs. ## 📱 React-Native {#react-native}

[![VisionCamera 3.0](/emails/issues/158/visioncamera3.jpg)](https://github.com/mrousavy/react-native-vision-camera/releases/tag/v3.0.0)

[**VisionCamera 3.0**](https://github.com/mrousavy/react-native-vision-camera/releases/tag/v3.0.0)

La nouvelle version de VisionCamera vient de sortir. Points forts :

- Réécriture complète d'Android Camera2
- Pipeline vidéo OpenGL GPU personnalisé
- Nouvelles fonctionnalités de caméra
- Nouveau moteur de traitement d'images (react-native-worklets-core)
- Nouvelle API de plugin de traitement d'images

Marc a pris la décision de finalement [supprimer l'intégration avec Skia](https://github.com/mrousavy/react-native-vision-camera/pull/1740) pour maintenir une taille de code et une complexité raisonnables dans VisionCamera.

Dans [**Détection de poses avec VisionCamera V3, TFLite et Skia**](https://mrousavy.com/blog/VisionCamera-Pose-Detection-TFLite), il explique ce qu'il est possible de réaliser avec une telle intégration. Les résultats sont très impressionnants.

[![Détection de poses avec VisionCamera V3, TFLite et Skia](/emails/issues/158/frame-processing-skia.jpg)](https://mrousavy.com/blog/VisionCamera-Pose-Detection-TFLite)
