## ⚛️ React {#react}

[![Astro 5.0](/emails/issues/212/astro.jpg)](https://astro.build/blog/astro-5/)

**[Astro 5.0](https://astro.build/blog/astro-5/)**

Une nouvelle version majeure d'Astro vient de sortir. Bien que techniquement pas un framework React pur, il reste largement utilisé dans la communauté React, intègre facilement des composants React interactifs et repose sur MDX. Les points forts sont :

- Content Layer pour centraliser tout votre contenu dans un magasin de données convivial et typé. Incluant les CMS, API et autres sources.
- Server Islands pour servir une réponse de page initiale aussi vite que possible, tout en intégrant du contenu dynamique dans une mise en page majoritairement statique. Une alternative intéressante et plus simple aux React Server Components et au Partial Pre-Rendering de Next.js.
- Pré-rendu simplifié, supprimant l'option "hybride".
- Variables d'environnement typées.
- Vite 6 avec la nouvelle Environment API.
- Fonctionnalités expérimentales : rognage d'images, mises en page responsives d'images, composants SVG.

## 📱 React-Native {#react-native}

[![Radon IDE 1.0](/emails/issues/212/radon.jpg)](https://ide.swmansion.com/)

**[Radon IDE 1.0](https://ide.swmansion.com/)**

Software Mansion a créé une extension VSCode (supportant également Cursor) qui transforme votre éditeur en un IDE complet pour React Native et Expo. Elle est désormais officiellement lancée en version stable. C'est un produit payant, mais compte tenu du prix abordable et de l'amélioration de la DX, c'est une évidence qui soutient également les formidables efforts open source de l'entreprise. Consultez la [vidéo de lancement de Radon IDE](https://www.youtube.com/watch?v=07Un9EfE8D4) pour un aperçu rapide des fonctionnalités : panneau des simulateurs et paramètres, débogage, inspection par clic, navigation pilotée par URL, intégration de Storybook, et plus encore.
