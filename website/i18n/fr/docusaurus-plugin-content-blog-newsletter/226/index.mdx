## ⚛️ React {#react}

[![Parcel v2.14.0](/emails/issues/226/parcel.jpg)](https://parceljs.org/blog/v2-14-0/)

[**Parcel v2.14.0**](https://parceljs.org/blog/v2-14-0/)

Parcel est un outil intéressant : c'est un bundler très robuste et complet en fonctionnalités. Bien qu'il ne soit pas utilisé ou supporté par aucun framework React majeur, cela ne l'empêche pas d'avancer, avec la version 2.14.0 qui ajoute quelques grandes fonctionnalités :

- Server Components avec rendu client, serveur et statique
- Support intégré de MDX (utilisant la bibliothèque Rust `mdx-rs`)
- Une CLI `create-parcel` qui peut remplacer `create-react-app`
- Un nouvel overlay d'erreur React

Si vous êtes en train de « construire votre propre framework » ou avez un cas d'utilisation de niche, tout cela rend Parcel encore plus attractif.

## 📱 React-Native {#react-native}

[![React Native Worklets RFC](/emails/issues/226/worklets.jpg)](https://github.com/software-mansion/react-native-reanimated/discussions/7264)

[**React Native Worklets RFC**](https://github.com/software-mansion/react-native-reanimated/discussions/7264)

React Native Worklets est une nouvelle bibliothèque extraite de Reanimated qui permet un véritable multithreading dans React Native. Elle vous permet d'exécuter du code JavaScript sur plusieurs runtimes. Un nouveau modèle de mémoire introduit les `Serializables`, `Shareables` et `Synchronizables` pour gérer les données entre les runtimes.

En découplant le multithreading des animations, Reanimated devient plus léger, ce qui est particulièrement bénéfique pour les applications web visant des tailles de bundle plus petites. Parallèlement, les développeurs peuvent désormais utiliser le multithreading pour des tâches non-UI comme le stockage asynchrone, les opérations de base de données, le traitement audio en temps réel ou les calculs alimentés par l'IA - sans dépendre d'une bibliothèque d'animation.
