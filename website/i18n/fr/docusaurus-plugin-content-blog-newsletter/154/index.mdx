## ⚛️ React {#react}

[![Comprendre les React Server Components](/emails/issues/154/understandingrsc.jpg)](https://vercel.com/blog/understanding-react-server-components)

[**Comprendre les React Server Components**](https://vercel.com/blog/understanding-react-server-components)

Nouvel article de Vercel qui donne une bonne vue d'ensemble des composants serveur React, leur implémentation dans Next.js, pourquoi ils existent, quand les utiliser et leurs limites. Aborde également divers sujets connexes : Server Actions, code-splitting, mise en cache, suspense, hydratation sélective...

[![Tout ce que j'aurais aimé savoir avant de migrer 50 000 lignes de code vers les React Server Components](/emails/issues/154/everythingrsc.jpg)](https://www.mux.com/blog/what-are-react-server-components)

[**Tout ce que j'aurais aimé savoir avant de migrer 50 000 lignes de code vers les React Server Components**](https://www.mux.com/blog/what-are-react-server-components)

Super article sur les RSCs ! Retour d'expérience qui détaille les 3 principaux défis rencontrés lors de l'adoption chez Mux : pas de CSS-in-JS, pas de React Context, et une complexité supplémentaire. Fournit également des réponses à de nombreuses questions et une stratégie pour adopter les RSCs de manière incrémentale.

- 📜 [Accélération de la documentation dbt™ par 20x avec les composants serveur React](https://dagster.io/blog/dbt-docs-on-react) : retour d'expérience sur une migration POC d'une application Angular 1 vers Next.js App Router et les composants serveur React. Des gains très significatifs sont rapportés en termes de temps de chargement, consommation mémoire et blocage CPU.
- 📜 [La plateforme de Shopify est la plateforme Web](https://shopify.engineering/shopifys-platform-is-the-web-platform) : Shopify recommande désormais Remix pour créer des "Admin Apps", qui s'exécutent dans une iframe et communiquent avec l'application parente via un "App Bridge" basé sur `postMessage()`. Une architecture frontend/React intéressante pour ceux qui construisent une plateforme nécessitant un découplage.
- 📜 [Composants partiellement contrôlés : un modèle de conception déclaratif dans React](https://www.jameskerr.blog/posts/partially-controlled-react-components/) : introduit un nouveau terme/concept pour décrire des composants capables d'offrir à la fois un état contrôlé/non contrôlé à la demande, avec une granularité applicable à l'état entier ou à une partie de celui-ci.
- 📜 [Utiliser des icônes SVG en sprite dans React](https://www.jacobparis.com/content/svg-icons) : explique comment utiliser des sprites SVG (plus optimisés que les composants SVG React) dans un contexte Remix avec une étape de génération de code.
- 📜 [Typer fortement useRef avec ElementRef](https://www.totaltypescript.com/strongly-type-useref-with-elementref) : Aujourd'hui, j'ai appris (TIL) le type TypeScript `ElementRef`, qui est utilisé pour obtenir le type d'une référence React de n'importe quel composant, de manière similaire à `ComponentProps`.
- 📜 [10 techniques essentielles d'optimisation des performances React pour des applications Web plus rapides](https://techreactlearning.blogspot.com/2023/07/10-essential-react-performance.html)
- 📜 [Les bases de la gestion des erreurs dans Remix](https://otiv.dev/blog/essential-remix-error-handling)
- 📦 [Lucia v2.0](https://lucia-auth.com/blog/lucia-2) : Lucia est une bibliothèque d'authentification TypeScript qui gagne du terrain récemment. Inclut des intégrations avec Next.js et Remix.
- 📦 [Mode automatique de Million.js en bêta](https://million.dev/docs/automatic) : analyse désormais votre code React et l'optimise automatiquement avec Million.js !
- 📦 [Redwood v6](https://github.com/redwoodjs/redwood/releases/tag/v6.0.0) : nouvelle version stable avec Vite, Prisma 5, Storybook 7, Suspense Router... Prépare le terrain pour l'introduction du streaming et des composants serveur React.
