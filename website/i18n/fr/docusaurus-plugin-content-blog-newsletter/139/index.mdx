## ⚛️ React {#react}

[![Présentation de react.dev](/emails/issues/139/reactdev.jpg)](https://react.dev/blog/2023/03/16/introducing-react-dev)

**[Présentation de react.dev](https://react.dev/blog/2023/03/16/introducing-react-dev)**

Le nouveau site React est enfin en ligne ! Un nouvel article accompagne ce lancement. Projet colossal (technique et contenu) ayant requis plusieurs années d'efforts et de nombreux contributeurs.

Ce que j'apprécie particulièrement :

- L'accent sur les hooks et composants fonctionnels
- Les guides avec diagrammes, illustrations et défis interactifs
- La présentation de React comme solution multiplateforme (pas uniquement web)
- Les pages API clairement séparées (référence/usage, actuel/déprécié, client/serveur)
- La cohérence globale de l'expérience

Certains choix ont suscité des débats cette semaine, notamment la recommandation exclusive de quelques frameworks pour démarrer (Next.js, Remix, Gatsby, Expo) au détriment d'autres (Vite, Create-React-App, SPA côté client...).

- 👀 [React PR - Suspendre le commit sans bloquer le rendu](https://github.com/facebook/react/pull/26398) : une fonctionnalité qui sera d'abord utilisée en interne, notamment pour les ressources (CSS, polices, images). Lors de la navigation avec startTransition, React pourra attendre que les images de la page suivante soient chargées avant de valider la transition ([détails](https://twitter.com/acdlite/status/1637102880850059264)).
- 👀 [React PR - Supprimer l'avertissement sur les effets de mise en page côté serveur](https://github.com/facebook/react/pull/26395) : l'avertissement SSR pour useLayoutEffect sera supprimé. Dan Abramov commente sur [useIsomorphicLayoutEffect](https://twitter.com/dan_abramov/status/1636176793592905729).
- 🧵 [Tim Neutkens sur l'amélioration de `next export`](https://twitter.com/timneutkens/status/1636693508223270912) : compatible avec App Router, Server Components...
- 📜 [Future Proofing Your Remix App](https://remix.run/blog/future-flags) : Remix explique sa stratégie pour déployer des changements cassants dans les versions majeures. Ils prévoient d'introduire des "Future flags" permettant d'opter pour les changements planifiés à l'avance. Si vous activez tous les flags en v1, la migration vers v2 sera simplifiée.
- 📜 [Internationalisation dans Next.js 13 avec les React Server Components](https://www.smashingmagazine.com/2023/03/internationalization-nextjs-13-react-server-components/) : Les Server Components auront un impact majeur sur la traduction des apps React, notamment sur les performances, le code splitting et la taille des bundles. Cet article introduit bien le sujet. J'apporte des détails complémentaires dans ce [thread](https://twitter.com/sebastienlorber/status/1638528549056311297).
- 📜 [Hooks que vous n'avez probablement pas besoin](https://reacttraining.com/blog/hooks-you-probably-dont-need) : présentation des hooks React méconnus, classés par probabilité d'utilisation (bonjour `useInsertionEffect` 😅). La dernière section présente un anti-pattern courant et explique comment `useLayoutEffect` le prévient. Je trouve que [useSyncExternalStore est excellent](https://twitter.com/sebastienlorber/status/1615329010761842689) pour lire les APIs navigateur comme `localStorage`.
- 📜 [Front-Commerce passe à Remix](https://developers.front-commerce.com/changelog/front-commerce-is-getting-a-remix) : après Shopify, Front-Commerce choisit de s'appuyer sur Remix. Cela offre de nombreux avantages : amélioration progressive, mode MPA et moins de couplage avec React.
- 📜 [Astro : à quel point ce nouveau framework JavaScript est-il bon ?](https://cult.honeypot.io/reads/astro-the-new-javascript-framework/)
- 📜 [Réduire de 97% le JavaScript du site Open UI en migrant vers Astro](https://blog.andri.co/023-migrating-open-ui-site-from-gatsby-to-astro/)
- 📦 [React-Error-Boundary v4](https://twitter.com/sebastienlorber/status/1638465247886426113) : nouvelle API `useErrorBoundary` pour déclencher les limites d'erreur pour les erreurs non-rendu, et réinitialiser facilement la limite depuis n'importe où.
- 📦 [Wakuwork](https://github.com/dai-shi/wakuwork) : Daishi Kato développe un framework React minimaliste avec support des Server Components.
- 📦 [Framer-Motion 10.5](https://twitter.com/mattgperry/status/1636714500731592706) : `useAnimate()`
- 📦 [Remix-Sitemap](https://github.com/fedeya/remix-sitemap)
- 🎥 [Le nouveau cache de React va-t-il corriger son hook "use" ?](https://www.youtube.com/watch?v=T3m-MZkuadU)
