## ⚛️ React {#react}

[![Comment React 18 améliore les performances des applications](/emails/issues/152/react18perf.jpg)](https://vercel.com/blog/how-react-18-improves-application-performance)

**[Comment React 18 améliore les performances des applications](https://vercel.com/blog/how-react-18-improves-application-performance)**

Article complet et bien illustré sur React 18 et son impact sur les performances. Explique le fonctionnement du Concurrent React et son influence sur des métriques comme le Total Blocking Time et l'Interaction to Next Paint. Présente également d'autres nouvelles fonctionnalités comme l'API Transition, les React Server Components, Suspense et le nouveau cache React.

[![Kuma UI](/emails/issues/152/kuma.jpg)](https://www.kuma-ui.com/)

**[Kuma UI](https://www.kuma-ui.com/)**

Nouvelle bibliothèque CSS-in-JS inspirée par la DX offerte par Chakra UI, et compatible avec les React Server Components. La bibliothèque repose sur une méthode hybride unique combinant le meilleur des solutions CSS-in-JS sans runtime et des solutions runtime traditionnelles. Nous devrons étudier tout cela et voir comment cette bibliothèque se compare à [Panda CSS](https://panda-css.com/) de l'équipe Chakra.

**[Expo SDK 49](https://blog.expo.dev/expo-sdk-49-c6d398cdf740)**

Le nouveau SDK 49 vient d'être publié. De nombreuses fonctionnalités ont été annoncées, dont mes préférées :

- React Native 0.72.3, React Native Web 0.19.6
- Expo Router v2.0
- Débogage réseau, débogage VS Code
- React devtools intégrés
- Variables d'environnement
- Modules Expo locaux
- EAS Insights
