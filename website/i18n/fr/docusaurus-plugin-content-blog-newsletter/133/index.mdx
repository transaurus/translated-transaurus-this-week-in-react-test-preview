## ⚛️ React {#react}

[![Astro rendering v2](/emails/issues/133/astro-rendering.jpg)](https://astro.build/blog/hybrid-rendering/)

**[Astro - Débloquez de nouvelles possibilités avec le rendu hybride](https://astro.build/blog/hybrid-rendering/)**

Avec Astro 1, vous deviez choisir globalement pour tout le site : 100% statique (SSG) ou 100% serveur (SSR). Avec Astro 2.0, il est désormais possible de choisir par page. L'article détaille le fonctionnement interne de la build Astro et ses adaptations pour cette nouvelle fonctionnalité. Ils utilisent une analyse statique pour séparer les routes statiques des routes SSR.

[![Why React isn't dying](/emails/issues/133/reactisntdying.jpg)](https://tkdodo.eu/blog/why-react-isnt-dying)

**[Pourquoi React ne meurt pas](https://tkdodo.eu/blog/why-react-isnt-dying)**

Dominik explique pourquoi React n'est pas près de disparaître. React a popularisé JSX et les concepts de composant et d'UI déclarative. Les alternatives reprennent ses bons aspects et tentent de résoudre ses problèmes, mais il sera difficile de le concurrencer vu la taille de son écosystème et sa communauté qui se renforce avec le temps. Il faudrait une innovation majeure. Les bibliothèques agnostiques (TanStack) jouent un rôle clé en permettant de transférer les connaissances entre écosystèmes.

[![Resumable React: How To Use React Inside Qwik](/emails/issues/133/qwik.jpg)](https://www.builder.io/blog/resumable-react-how-to-use-react-inside-qwik)

**[React Résumable : Comment utiliser React dans Qwik](https://www.builder.io/blog/resumable-react-how-to-use-react-inside-qwik)**

Cet article met en lumière l'intégration entre Qwik et React. Qwik est un framework innovant en version bêta, basé sur la reprise plutôt que sur l'hydratation. Il expose une méthode `qwikify$()` qui permet d'adapter un composant React existant pour être utilisé dans une application Qwik, avec plusieurs modes d'hydratation possibles. Pour réduire autant que possible la taille du bundle, il est possible d'éviter d'hydrater le composant React et d'utiliser directement les événements DOM natifs.

## 💸 Sponsor {#sponsor2}

[![surveyjs.jpg](/emails/issues/133/surveyjs.jpg)](https://surveyjs.io/?utm_source=thisweekinreact&utm_medium=referral)

**[Bibliothèques JavaScript extensibles pour les enquêtes et formulaires](https://surveyjs.io/?utm_source=thisweekinreact&utm_medium=referral)**

[SurveyJS](https://github.com/surveyjs) est un ensemble de bibliothèques JavaScript open-source pour formulaires que vous pouvez facilement intégrer à votre application web. Il vous permet de **créer et exécuter plusieurs formulaires**, en conservant toutes les données sensibles sur vos serveurs, ou d'avoir **votre propre éditeur de formulaires WYSIWYG auto-hébergé** que les utilisateurs non techniques peuvent utiliser directement.

Vous pouvez également **analyser les résultats des enquêtes** dans des tableaux de bord personnalisés, enregistrer vos formulaires au format PDF ou simplement **les convertir en fichiers PDF modifiables**. Il offre une prise en charge native pour le **rendu React et Angular** et une totale liberté de choix pour le backend, car **toute combinaison serveur + base de données est entièrement compatible**.
