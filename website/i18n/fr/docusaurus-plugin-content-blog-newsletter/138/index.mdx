## ⚛️ React {#react}

[![Chargement paresseux des routes dans React Router 6.4+](/emails/issues/138/reactrouterlazy.jpg)](https://remix.run/blog/lazy-loading-routes)

**[Chargement paresseux des routes dans React Router 6.4+](https://remix.run/blog/lazy-loading-routes)**

[React-Router 6.9](https://github.com/remix-run/react-router/releases/tag/react-router%406.9.0) vient de sortir avec une nouvelle fonctionnalité de routes paresseuses qui permet d'éviter les cascades de chargement et d'optimiser le temps de chargement de vos applications. C'est ainsi que fonctionne Remix, mais vous pouvez désormais utiliser cette technique sur n'importe quelle application basée sur React-Router. Personnellement, je prévois de l'utiliser sur Docusaurus (basé sur React-Router).

## 💸 Sponsor {#sponsor2}

[![Surveyjs - Bibliothèques JavaScript extensibles pour les enquêtes et formulaires](/emails/issues/136/surveyjs.jpg)](https://surveyjs.io/?utm_source=thisweekinreact&utm_medium=referral)

**[Bibliothèques JavaScript extensibles pour les enquêtes et formulaires](https://surveyjs.io/?utm_source=thisweekinreact&utm_medium=referral)**

[SurveyJS](https://surveyjs.io/form-library?utm_source=thisweekinreact&utm_medium=referral) est un ensemble de bibliothèques JavaScript open source pour les formulaires que vous pouvez facilement intégrer à votre application web. Il vous permet de **créer et exécuter plusieurs formulaires**, en conservant toutes les données sensibles sur vos serveurs, ou d'avoir **votre propre constructeur de formulaires WYSIWYG auto-hébergé** que les utilisateurs non techniques peuvent utiliser directement.

Vous pouvez également **analyser les résultats des enquêtes** dans des tableaux de bord personnalisés, sauvegarder vos formulaires au format PDF, ou simplement **les convertir en fichiers PDF modifiables**. Il offre une prise en charge native pour le **rendu React et Angular** et une liberté totale de choix quant au backend, car **toute combinaison serveur + base de données est entièrement compatible**.

## 🔀 Divers {#other}

[![Rspack](/emails/issues/138/rspack.jpg)](https://www.rspack.dev/)

**[Rspack](https://www.rspack.dev/)**

Annonce officielle du bundler Rust de ByteDance, compatible avec Webpack mais bien plus rapide. Il sera difficile de choisir entre Rspack et Turbopack, qui a également annoncé [une prochaine prise en charge des loaders Webpack](https://twitter.com/jaredpalmer/status/1619071988181651456). Notez qu'[Unplugin 1.3](https://twitter.com/antfu7/status/1634517321392726017) prend déjà en charge Rspack et permet de créer des plugins de bundler agnostiques. Aucune prise en charge Turbopack pour Unplugin n'existe encore, mais j'espère qu'elle est prévue !

Lire aussi [Rspack : Le bundler web basé sur Rust qui allie haute performance et interopérabilité](https://valor-software.com/press-release/rs-pack-rust-based-web-bundler)
