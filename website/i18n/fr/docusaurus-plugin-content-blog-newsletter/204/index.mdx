## ⚛️ React {#react}

[![React à Meta Connect 2024](/emails/issues/204/meta.jpg)](https://engineering.fb.com/2024/10/02/android/react-at-meta-connect-2024/)

[**React à Meta Connect 2024**](https://engineering.fb.com/2024/10/02/android/react-at-meta-connect-2024/)

Lors d'un récent événement, Meta a annoncé :

- De nouvelles applications de réalité mixte Instagram et Facebook pour le casque Meta Quest. Ces applications utilisent React Native et fonctionnent sur Static Hermes. L'équipe a pu réutiliser le code et l'infrastructure web existants de Facebook grâce à des technologies comme StyleX et React Strict DOM.
- Une nouvelle application Meta Horizon RN qui performe au même niveau que leur application Facebook native principale. Ils mentionnent le modèle render-as-you-fetch comme l'amélioration de performance la plus impactante qu'ils ont implémentée.
- Le Meta Horizon Store pour web/mobile/HorizonOS construit avec React Compiler, StyleX et React Strict DOM à la pointe.
- Le Meta Spatial Editor construit sur React Native Windows/macOS leur permet d'intégrer nativement leur moteur de rendu 3D personnalisé. L'équipe C++/Qt était initialement sceptique mais a fini par adorer le Fast Refresh.

De nombreux projets innovants sont testés en interne chez Meta sur de véritables applications de production, améliorant le framework et l'écosystème pour nous tous, sur toutes les plateformes. Avec tout cela, nous pouvons affirmer avec confiance que Meta continuera de miser sur React comme une excellente solution multiplateforme pour les années à venir.

Voir aussi la 🎥 [vidéo commentée de Theo](https://www.youtube.com/watch?v=GjBfD9PHff4).

## 📱 React-Native {#react-native}

[![One - Nouveau framework React multiplateforme](/emails/issues/204/one.jpg)](https://onestack.dev/)

[**One - Nouveau framework React multiplateforme**](https://onestack.dev/)

Nate, le créateur de Tamagui, a publié un nouveau framework universel prenant en charge le web et le mobile. Basé sur Vite plutôt que Metro, il dispose d'un routage universel typé basé sur les fichiers, des loaders de type Remix et différents modes de rendu (SSG/SSR/SPA). Il aide également à créer des applications local-first avec une intégration optionnelle du moteur de synchronisation granulaire Zero.
