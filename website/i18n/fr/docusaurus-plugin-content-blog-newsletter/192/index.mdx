## ⚛️ React {#react}

[![Remix - Fog of War](/emails/issues/192/rr.jpg)](https://remix.run/blog/fog-of-war)

**[Remix - Fog of War](https://remix.run/blog/fog-of-war)**

Déjà évoqué dans notre dernier numéro, un article dédié explique désormais ce sujet en détail. Une nouvelle fonctionnalité de React Router permet de fragmenter le manifeste des routes. L'article détaille son intégration dans Remix 2.10 et les compromis associés. Un manifeste de routes peut devenir volumineux, retardant l'hydratation React s'il est téléchargé intégralement. Pour les sites Remix/React Router de taille moyenne/importante, vérifiez immédiatement la taille de votre manifeste et envisagez un découpage logique : la réduction de taille peut être significative (10Mo → 1.9Ko chez Shopify !). Le manifeste de Docusaurus (200Ko) sera aussi optimisé.

## 📱 React-Native {#react-native}

[![react-native-filament](/emails/issues/192/filament.jpg)](https://github.com/margelo/react-native-filament)

**[react-native-filament](https://github.com/margelo/react-native-filament)**

L'équipe Margelo a publié un impressionnant nouveau moteur de rendu 3D en temps réel basé sur la physique pour React Native, fondé sur Google Filament. Accéléré par GPU via Metal/Vulkan, rendu sur un thread séparé, compatible avec Reanimated, et prenant déjà en charge la Nouvelle Architecture et Expo. Le readme explique bien comment il diffère des solutions 3D existantes basées sur expo-gl (expo-three et react-three-fiber RN). Avec Skia et Filament, l'avenir s'annonce prometteur pour les capacités 2D/3D de React Native.

[![Rise Tools](/emails/issues/192/rise.jpg)](https://rise.tools/)

**[Rise Tools](https://rise.tools/)**

Une toute nouvelle solution pour créer des interfaces utilisateur pilotées par serveur dans React Native, permettant de déployer de nouvelles vues côté serveur sans reconstruire/resoumettre votre application (ni même livrer une mise à jour OTA). Philosophiquement très proche du modèle React Server Components, mais avec des compromis légèrement différents : au prix d'un câblage manuel supplémentaire, cette solution est agnostique vis-à-vis du framework et du langage grâce à un protocole définissant la sérialisation des vues. Encore précoce, mais prometteur.
