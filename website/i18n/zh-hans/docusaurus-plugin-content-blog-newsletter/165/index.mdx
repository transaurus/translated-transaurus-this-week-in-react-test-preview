## ⚛️ React {#react}

[![StyleX](/emails/issues/165/stylex.jpg)](https://github.com/facebook/stylex)

[**StyleX**](https://github.com/facebook/stylex)

经过漫长等待，Meta的CSS-in-JS解决方案终于开源。该方案已在Facebook、Instagram和WhatsApp等主要应用中投入使用。它并非严格与React耦合，原子化CSS将在构建时提取，并配有轻量级运行时用于类名拼接。TypeScript支持经过精心设计，API极简主义，主要依赖`create`和`props`两个函数。我们将观察这个新方案如何在已存在Tailwind、Panda等竞争方案的生态中定位。

另请参阅：

- 🎥 [Jack Herrington - StyleX：Meta的大规模CSS解决方案](https://www.youtube.com/watch?v=6ZDiGtg1jN4)
- 🎥 [Theo T3 - Facebook尝试Tailwind后，转而开发了这个](https://www.youtube.com/watch?v=PHV94q7BivI)

[![CSS Hooks](/emails/issues/165/csshooks.jpg)](https://css-hooks.com/)

带有React绑定的CSS库，通过内联样式实现真正的回归基础。作者在[从CSS混乱到CSS Hooks](https://nsaunders.dev/posts/css-madness-to-hooks)中解释了如何利用CSS变量技巧支持媒体查询、伪选择器等缺失功能。

个人而言，我一直偏爱内联样式的开发体验，但实践中其功能有限且性能表现不稳定。这项创新能否经得起实践考验尚待观察。Mark Dalgleish（🐦[推文](https://twitter.com/markdalgleish/status/1729399475494608923)）对此表示兴奋。注意：Adam Wathan曾表示若内联样式支持更完善，他可能就不会创建Tailwind（🐦[推文](https://twitter.com/adamwathan/status/1732209400641724785)）。
