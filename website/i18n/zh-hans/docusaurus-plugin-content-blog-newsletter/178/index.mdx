[![Remix 以 Vite 为首选](/emails/issues/178/remix_heart_vite.jpg)](https://twitter.com/markdalgleish/status/1770224011831890036)

🐦 [**Remix 现已采用 Vite 优先架构**](https://twitter.com/markdalgleish/status/1770224011831890036)

在经历短暂的过渡期（Remix 曾推出标记为不稳定的 Vite 集成）后，该方案现已标记为稳定版本并立即成为默认配置。我个人非常欣赏这种快速转型，认为 Remix 与 Vite 的组合极具潜力——让 Vite 作为开发环境和编译器，使 Remix 团队能缩减维护范围，专注于核心框架能力优化。现在新建的 Remix 项目仅需一个引用 Remix 插件的 Vite 配置文件，堪称精简设计的典范！
