## 📱 React-Native {#react-native}

本版块现由 [Benedikt](https://twitter.com/bndkt) 联合撰写。欢迎通过邮件或 [Twitter](https://twitter.com/bndkt) 向我们反馈意见！

过去一周，我们暂时没有重大 React Native 新闻（或者我错过了什么？有请 [@me](https://twitter.com/bndkt)）。在经历了 Expo 50、visionOS、React Native 现状报告、react-strict-dom 等事件旋风后，这或许是个令人欣慰的喘息。但这并不意味着 React Native 世界停滞不前。许多小型公告让我们得以窥见 React Native 的近未来。[RN 0.74](https://github.com/facebook/react-native/blob/ee2bc59dfe4579cad3b7f0b5de667a89fcac8710/CHANGELOG.md#0740-rc0) 的首批候选版本已发布，包含大量改进以及部分弃用和破坏性变更，但初步看来没有令人担忧的重大升级负担。最值得注意的是，桥接模式（bridgeless）将在 0.74 版本中成为新默认项（但仅在启用新架构时生效）。作为新架构的第三大支柱，桥接模式紧随已推出的 TurboModules 和新渲染器（Fabric）之后。关于新架构和默认设置，一个[PR](https://github.com/facebook/react-native/pull/43135) 已合并，将使新架构从 RN 0.75 起成为默认选项（但仍可选择退出）。此外，定义 [React Native 框架](https://github.com/react-native-community/discussions-and-proposals/blob/main/proposals/0759-react-native-frameworks.md) 的 RFC 也已合并，明确了 RN 与上层框架的职责边界。
