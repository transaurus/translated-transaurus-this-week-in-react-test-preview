## ⚛️ React {#react}

[![Sathya Gunasekaran - React Compiler的未来规划](/emails/issues/208/compiler.jpg)](https://www.youtube.com/watch?v=qd5yk2gxbtg)

🎥 [**Sathya Gunasekaran \- React Compiler的未来规划**](https://www.youtube.com/watch?v=qd5yk2gxbtg)

我最近观看了这个精彩演讲，它不仅宣布了[React Compiler测试版](https://react.dev/blog/2024/10/21/react-compiler-beta-release)，还揭示了几项创新功能。该编译器不仅是自动记忆化工具，更是构建更多优化功能的平台。演讲介绍了3个即将推出的特性：

- 类型配置：通过提供额外的类型信息，可以帮助编译器更好地工作。
- JSX 内联：编译器可以移除无用的 JSX 运行时调用，将其替换为预构建的 JSX 对象。`jsx('div')` 会变成 `{type: 'div'}`。
- JSX 外联：编译器可以自动提取子组件，这些组件随后可被独立优化。这在处理列表元素时尤其有用，当使用 `array.map` 和内联 JSX 时，编译器可以创建 `<Item>` 组件（参见演讲截图，或参阅[此 PR](https://github.com/facebook/react/pull/30956) 了解详情）。

## 📱 React-Native {#react-native}

[![React Navigation 7.0](/emails/issues/208/reactnavigation7.jpg)](https://reactnavigation.org/blog/2024/11/06/react-navigation-7.0/)

[**React Navigation 7.0**](https://reactnavigation.org/blog/2024/11/06/react-navigation-7.0/)

React Navigation v7 现已发布，将为即将随 Expo SDK 52 推出的 Expo Router v4 提供支持（两者均处于测试阶段）。最值得关注的亮点包括：

- 屏幕预加载：可在实际导航前提前准备屏幕，从而提升感知性能。该功能非常令人期待，未来可能支持并发 React 特性。
- 静态 API：简化了导航器配置，更便于与 TypeScript 和深度链接结合使用。

[**我们刚刚完成 Shopify 移动应用向 React Native 的迁移**](https://threadreaderapp.com/thread/1853619638141071573.html)

工程总监 Mustafa 在 X 上宣布，Shopify 已完成将其旗舰应用（包含 586 个屏幕）迁移至 React Native，并取得显著成效。期待后续的详细技术博文。

- 删除 180 万行代码
- iOS 与 Android 间代码共享率达 86%
- 屏幕加载速度提升 59%
- 应用启动速度提升 44%
- WebView 速度提升 63%
- 减少应用崩溃
- 提升应用商店评分
