## ⚛️ React {#react}

[![正式弃用 Create React App](/emails/issues/222/cra.jpg)](https://react.dev/blog/2025/02/14/sunsetting-create-react-app)

**[正式弃用 Create React App](https://react.dev/blog/2025/02/14/sunsetting-create-react-app)**

姗姗来迟，CRA 已获得最新的 React 19 修复并正式宣告弃用。本文阐述了 CRA 及构建工具在路由、瀑布流、代码拆分等方面的局限性。React 团队鼓励我们采用 Next.js、React Router 或 Expo 等框架来解决这些问题。不过您仍可使用 Vite、Parcel、Rsbuild 等现代构建工具[从零构建 React 应用](https://react.dev/learn/build-a-react-app-from-scratch)。

最初的公告在X和GitHub上引发了激烈争论，主要是因为它过度强调框架而忽视了Vite等构建工具。该公告[很快进行了更新](https://github.com/reactjs/react.dev/pull/7624)以解决这些担忧。这一切还促成了[尤雨溪（Vite）与React团队](https://github.com/facebook/react/pull/31768#issuecomment-2666856223)之间的潜在合作机会，双方将共同制定一种"官方"推荐方案，将Vite与React Server Components集成，并承诺在React代码库中长期维护该集成方案。

## 📱 React-Native {#react-native}

[![React Native 0.78 - 支持React 19及其他更新](/emails/issues/222/rn.jpg)](https://reactnative.dev/blog/2025/02/19/react-native-0.78)

**[React Native 0.78 \- 支持React 19及其他更新](https://reactnative.dev/blog/2025/02/19/react-native-0.78)**

您可能没想到React Native会这么快发布新版本，但团队最近决定更频繁地发布小型更新。以下是重点内容：

- 支持React 19！
- 简化了启用React编译器的流程
- Metro日志流功能在0.78和0.77.1版本回归，但需通过新的`--client-logs`标志启用，且仍处于弃用状态
- `<Image>`组件支持Android XML资源，可解锁潜在性能优化
- iOS端新增`ReactNativeFactory`，简化任意ViewController中的遗留代码集成

React 19本身带来了多项重要能力：

- Actions（异步转换）、`useActionState()`、`useOptimistic()`，均用于提交表单并管理加载状态、错误及乐观状态
- 将`ref`作为常规属性使用（取代`forwardRef()`），同时支持ref清理函数
- 使用`<Context>`替代`<Context.Provider>`
- 通过`use()`读取上下文和Promise
