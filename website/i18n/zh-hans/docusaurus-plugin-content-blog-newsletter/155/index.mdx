## ⚛️ React {#react}

[![React Server Components开发工具](/emails/issues/155/rscdevtools.jpg)](https://www.alvar.dev/blog/creating-devtools-for-react-server-components)

[**React Server Components开发工具**](https://www.alvar.dev/blog/creating-devtools-for-react-server-components)

React Server Components发布后，Alvar注意到客户端与服务端间的网络通信采用了一种难以解读的格式。因此他开发了网站[RSC解析器](https://rsc-parser.vercel.app/)，可更便捷地检查有效载荷。虽然有效，但用户体验欠佳。此后他转向开发浏览器扩展，其功能类似Chrome网络开发者工具，能通过时间轴展示有效载荷变化，甚至支持Suspense流式传输。该工具已帮助他在博客中检测并优化了过大的有效载荷。

这个前景广阔的工具仍在开发中，值得持续关注。扩展程序尚未上架浏览器商店，但即将推出。相关视频演示详见[Twitter帖子](https://twitter.com/sebastienlorber/status/1689203357368365056)。

[![Next.js中的缓存](/emails/issues/155/cachinginnextjs.jpg)](https://nextjs.org/docs/app/building-your-application/caching)

[**Next.js中的缓存**](https://nextjs.org/docs/app/building-your-application/caching)

新的Next.js文档页面全面解释了4级缓存的工作原理、存在原因以及如何绕过它们：

- 请求记忆化
- 数据缓存
- 完整路由缓存
- 路由器缓存

无法绕过路由器缓存：这个问题在[此视频](https://www.youtube.com/watch?v=_yhSh4g0NSk)中有详细说明。
