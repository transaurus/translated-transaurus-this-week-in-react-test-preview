## React

[**源码解析：React**](https://alexkondov.com/readint-source-code-react/?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)

Alex通过解析React挂载应用的入口点，深入解读源码并剖析多个技术细节。涉及原型链应用、JSX实现原理，以及（通用）协调器与DOM渲染器的交互机制等精彩内容。

[**我们能否承认 React Hooks 是个糟糕的主意？**](https://medium.com/codex/can-we-all-just-admit-react-hooks-were-a-bad-idea-c48120c5188d?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)

Amy 认为 React Hooks 违背了 SOLID 架构原则。我虽不完全认同，但承认互联网上大多数示例都过于简化，未能体现良好架构实践。

[**在 React 中应用 SOLID 原则**](https://konstantinlebedev.com/solid-in-react/?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)

这几乎是对上文观点的直接回应，尽管本文更全面地覆盖了 SOLID 原则的应用（组件 + Hooks）。注意：可善用 React context 实现依赖注入。

## React-Native

[**Hermes 成为默认引擎**](https://reactnative.dev/blog/2022/07/08/hermes-as-the-default?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)

Hermes JS 引擎是 JSC/V8 的替代方案，通过字节码预编译可加速应用启动。随着 React-Native 0.70（现已发布[候选版本](https://github.com/facebook/react-native/releases/tag/v0.70.0-rc.0?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)）的推出，Hermes 已默认启用。Michael 分享了开源应用 Mattermost 的多项基准测试。Hermes 分发模式已变更以避免 ABI 兼容性问题，iOS 平台新增了缺失的 Intl API，即将支持 BigInt 和 WeakRef。

[**在Airbnb React Native时代的工作经历教会了我关于web3的知识**](https://medium.com/@devinabbott/what-working-at-airbnb-during-its-react-native-era-taught-me-about-web3-6d34d855d892?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)

Devin回顾了React Native在Airbnb的发展历程以及最终停止使用的微妙决策。他提出了一个有趣的类比：从长远来看，技术潜力终将超越早期采用者所经历的阵痛。
