## React

[**Reassure**](https://github.com/callstack/reassure?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)

Reassure 是一个新的 React 性能测试库。之前已经介绍过，现在它已被 Callstack 开源。它可以在检测到性能退化时，在拉取请求中发布报告。例如：一个组件重新渲染变慢，或者比以前更频繁。目前只支持 React-Native，但很快就会支持 web。

[**如何实现多人光标的动画效果**](https://liveblocks.io/blog/how-to-animate-multiplayer-cursors?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)

一篇很棒的互动文章，提出了在多个浏览器上实时实现鼠标光标动画。这似乎是一个比看起来更复杂的问题。每种方法都通过 React 实现进行说明，并呈现了不同的权衡：CSS 过渡、弹簧、样条曲线...

[**如何使用 Next.js 中间件**](https://www.ctnicholas.dev/articles/how-to-use-nextjs-middleware?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)

来自同一位作者的互动文章，介绍了许多中间件的使用场景。之前已经在这里发布过，现在刚刚更新，以跟进 Next.js 12.2 的官方发布，其中包含一些破坏性变更。

[**Meteor 十年**](https://meteor10.sachagreif.com/?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)

Sacha Grief（《Discover Meteor》合著者，同时也是 State of JS、CSS 等开发者调查的发起人）回顾了他与 Meteor 框架的十年渊源。尽管 Meteor 热度有所下降，但仍在被使用且对前端生态产生了深远影响。

[**React 元素、子组件、父组件与重渲染的奥秘**](https://www.developerway.com/posts/react-elements-children-parents?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)

Nadia 揭示了 React 中某些令人惊讶甚至反直觉的行为，并详细解释了在哪些边缘场景下父组件重渲染会触发子组件重渲染。

[**Storybook 社区展示 #2**](https://storybook.js.org/blog/community-showcase-2/?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)

Storybook 社区最新动态摘要：组件百科全书、Figma 插件、Story 浏览器，以及多种插件：变体管理、CSS 变量、Recoil 状态管理...

[**React 状态管理的新浪潮**](https://frontendmastery.com/posts/the-new-wave-of-react-state-management/?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)

本文解析了状态管理库需要解决的不同问题并进行了横向对比，包含 Redux 演进历程的回顾。内容全面但篇幅较长：阅读体验较有挑战性。

## React-Native

[**FlashList - 高性能React Native列表组件**](https://shopify.github.io/flash-list/?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)

Shopify刚发布FlashList，作为FlatList的替代方案以解决滚动性能问题和空白单元格显示问题。其目标是在低端安卓设备上保持60fps帧率，且不增加实现复杂度。该库复用已渲染单元格：您需调整React `key`的使用方式以适配其机制。

[**React-Native-Compat-Table**](https://retyui.github.io/react-native-compat-table/?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)

以全面方式列出了各 React-Native 运行时（JSC、V8、Android 等）的 API 支持情况。作者还解释了[为什么 React Native 推荐 TS 配置库使用 "es2017"](https://dev.to/retyui/why-react-native-typescript-config-recommends-to-use-a-five-years-old-standard-lib-es2017-3bjk?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)
