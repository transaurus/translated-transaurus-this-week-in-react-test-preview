## ⚛️ React {#react}

[![Astro渲染v2](/emails/issues/133/astro-rendering.jpg)](https://astro.build/blog/hybrid-rendering/)

**[Astro - 通过混合渲染解锁新可能](https://astro.build/blog/hybrid-rendering/)**

在Astro 1中，你需要为整个站点做全局选择：100%静态(SSG)或100%服务端(SSR)。Astro 2.0现在支持逐页选择。文章详述了Astro构建的内部工作原理，以及它如何演进以支持此新特性。他们使用静态分析来分离静态路由和SSR路由。

[![为什么React不会消亡](/emails/issues/133/reactisntdying.jpg)](https://tkdodo.eu/blog/why-react-isnt-dying)

**[为什么React不会消亡](https://tkdodo.eu/blog/why-react-isnt-dying)**

Dominik解释了React为何不会消亡。React普及了JSX以及组件化和声明式UI的概念。替代方案取其精华并试图解决问题，但鉴于React生态的规模和持续壮大的社区，挑战它将非常困难。这需要重大突破性创新。框架无关库(TanStack)发挥着作用，允许知识在不同生态间迁移。

[![可恢复的React：如何在Qwik中使用React](/emails/issues/133/qwik.jpg)](https://www.builder.io/blog/resumable-react-how-to-use-react-inside-qwik)

**[可恢复的React：如何在Qwik中使用React](https://www.builder.io/blog/resumable-react-how-to-use-react-inside-qwik)**

本文重点介绍了 Qwik 和 React 之间的集成。Qwik 是一个处于测试阶段的创新框架，基于可恢复性（resumability）而非水合（hydration）。它公开了一个方法 `qwikify$()`，允许将现有的 React 组件适配到 Qwik 应用中使用，并提供多种水合模式。为了尽可能减小包体积，可以避免对 React 组件进行水合，直接使用原生 DOM 事件。

**[可扩展的 JavaScript 调查问卷和表单库](https://surveyjs.io/?utm_source=thisweekinreact&utm_medium=referral)**

[SurveyJS](https://github.com/surveyjs) 是一套开源的 JavaScript 表单库，你可以轻松地将其集成到你的 Web 应用中。它让你能够**创建并运行多个表单**，将所有敏感数据保留在你的服务器上，或者拥有**你自己的自托管所见即所得表单构建器**，非技术用户可以直接使用。

你还可以在自定义构建的仪表板中**分析调查结果**，将表单保存为 PDF，或者简单地将它们**转换为可编辑的 PDF 文件**。它原生支持 **React 和 Angular 渲染**，并且在后端选择上完全自由，因为**任何服务器+数据库组合都是完全兼容的**。
