## ⚛️ React {#react}

**[React将走向何方？](https://www.infoxicator.com/is-react-going-anywhere)**

Ruben在接触某电商CTO（决定弃用React转向原生JS）后的深度思考：应用性能问题是否应归咎React？改用替代方案是否合理？未来5-10年React能否保持领先地位？

**[将React交互性能提升4倍](https://www.causal.app/blog/react-perf)**

Causal的云端电子表格应用曾遇性能瓶颈。文章通过Chrome性能分析工具详细解析问题根源：部分源自AG Grid（已打补丁修复），其余主要为自身代码的memoization问题。尝试使用Web Workers时，序列化开销过大导致方案不可行。

- 📜 [逐步增强 Remix 中的 useFetcher 钩子](https://sergiodxa.com/articles/progressively-enhance-the-usefetcher-hook-in-remix)：小技巧，用于在 Remix 的 useFetcher API 上支持启用/禁用 JavaScript 的用户：强制重定向未启用 JavaScript 的用户。同样地，也可以处理[文件上传的渐进式增强](https://twitter.com/ebey_jacob/status/1596223682246234113)。这些功能对许多应用可能并非必需，但了解其可行性很有价值。
- 📜 [Remix Deferred 概述](https://gist.github.com/jacob-ebey/9bde9546c1aafaa6bc8c242054b1be26)：解释 [defer](https://github.com/remix-run/remix/issues/4486) 在 Remix 中的工作原理，它允许将 promise 从后端“传送”到前端。适合希望理解 Remix 内部机制的用户。
- 📜 [如何在 React 中更新兄弟组件的状态？](https://alexsidorenko.com/blog/react-siblings/)：短文，清晰阐释了“状态提升”的概念。
- 📜 [React 极简 TypeScript 速成课程](https://profy.dev/article/react-typescript)：帮助开发者快速入门 TypeScript 和 React。
- 📜 [前端 JavaScript 实战中的数据结构 - 附 React 代码示例](https://profy.dev/article/javascript-data-structures)：在 React 环境中熟悉数据结构的实用指南。
- 📜 [ArConnect 将其 Chrome 扩展迁移至 Plasmo 框架](https://blog.plasmo.com/p/migrating-chrome-extension-to-plasmo)：关于使用 [Plasmo](https://github.com/PlasmoHQ/plasmo) 的实践反馈：用于创建 Web 扩展的开源 React 框架。
- 📜 [面向全栈开发者的 Storybook](https://storybook.js.org/blog/storybook-for-full-stack-developers/)：全面概述 Storybook 生态系统能为开发者提供的功能。
- 📜 [Solid Final Form - 概念验证](https://erikras.com/blog/solid-final-form-poc)：Erik 说明其库 [Final Form](https://final-form.org/react)（几年前流行）已与 React 解耦，并演示了在 SolidJS 环境中的应用。
- 📜 [Ant Design 5.0 - 组件级 CSS-in-JS](https://ant.design/docs/blog/css-in-js)：React 设计系统发布新主版本，详解其 CSS-in-JS 运行时的优化方案。
- 🎥 [论 React 与单页面应用的优劣](https://www.youtube.com/watch?v=ChfjZV-aA_Y)：Alex Russell 与 Theo Browne 的 2 小时辩论。尚未观看。Alex 以批评 React 和 SPA 闻名，Theo 应是 React 生态的有力辩护者。
- 🎥 [React 开发者必须了解的 @container！](https://www.youtube.com/watch?v=ypN-Uwshc5M)：在原生 CSS + Tailwind 的 React 环境中，介绍查询容器新 CSS 特性。Firefox 暂不支持，但有 polyfill。
- 🎥 [RTK Query 基础：查询端点、数据流与 TypeScript](https://egghead.io/courses/rtk-query-basics-query-endpoints-data-flow-and-typescript-57ea3c43)：RTK Query 创建者 Lenz 在 Egghead 推出的免费新课程。
- 🧑‍💻 [Remix PR - 添加“跳转到定义”支持](https://github.com/remix-run/remix/pull/4689)：tRPC 贡献者将其 TypeScript 经验移植到 Remix 以提升开发者体验（DX）。
- 📦 [react-three-editor](https://github.com/pmndrs/react-three-editor)：正在开发中的 React-Three-Fiber 场景编辑器。
- 📦 [Gatsby 5.1](https://www.gatsbyjs.com/docs/reference/release-notes/v5.1/)

## 📱 React-Native {#react-native}

**[React Native 核心贡献者峰会 2022](https://reactnative.dev/blog/2022/11/22/react-native-core-contributor-summit-2022)**

9月初，大部分 React Native 贡献者齐聚 React Native EU 会议。他们通过系列研讨会形成报告，深入讨论了新架构、Metro 打包器和发布流程等议题。

**[Shopify - 我们测量 React Native 渲染时间的解决方案](https://shopify.engineering/measuring-react-native-rendering-times)**

[react-native-performance](https://github.com/Shopify/react-native-performance) 包可生成 JSON 格式的[现场/真实用户监控](https://web.dev/lab-and-field-data-differences/)性能报告。该工具与 React-Navigation 无缝集成，支持将报告发送至自选分析面板，帮助定位生产环境中存在性能问题的具体屏幕。其设计理念类似于现有 Web 指标：Core Web Vitals、TTI、INP 及 Vercel/Netlify/Gatsby 性能仪表板...

- 📜 [Shopify - 开源 FlashList 的经验总结](https://shopify.engineering/what-we-learned-from-open-sourcing-flashlist)：Shopify 成功开源项目的复盘，强调营销与规划的重要性。另可参考[Docusaurus 2.0 发布经验](https://www.indiehackers.com/post/1-on-product-hunt-thanks-to-newsletters-and-good-timing-2c7d8809d9)
- 📜 [如何用 Nitro 5 倍速构建 React Native 应用](https://blog.nitro.build/how-to-build-a-react-native-app-5x-faster-with-nitro-d7e512b18d90)：Nitro（付费预览工具）在仅修改 JS 代码时可大幅缩短 RN CI 构建时间
- 📜 [2022 年及未来的 React Native](https://semaphoreci.com/blog/react-native)
- 📜 [React Native 如何通过新架构实现原生级性能](https://medium.com/ocp-digital-factory/how-react-native-became-performant-as-native-with-the-new-architecture-6e678c3178aa)
