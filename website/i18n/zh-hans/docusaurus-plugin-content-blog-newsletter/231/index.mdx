## 📣 来自React团队 {#react-core-team}

[![React实验室：视图过渡、活动状态及其他](/emails/issues/231/labs.jpg)](https://react.dev/blog/2025/04/23/react-labs-view-transitions-activity-and-more)

**[React实验室：视图过渡、活动状态及其他](https://react.dev/blog/2025/04/23/react-labs-view-transitions-activity-and-more)**

关于React团队只关心服务端的传言被严重夸大了。

在今天的[React实验室文章](https://react.dev/blog/2025/04/23/react-labs-view-transitions-activity-and-more)中，我们分享了[视图过渡](https://react.dev/reference/react/ViewTransition)和[活动状态](https://react.dev/reference/react/Activity)功能现已在实验频道开放试用，并附有相关文档。这些新功能使运行在客户端的React能够实现UI过渡动画，并在用户进行客户端导航时动态显示/隐藏UI部分。它们可与服务端功能协同工作，但主要解决客户端问题。

我们还[分享了更新](https://react.dev/blog/2025/04/23/react-labs-view-transitions-activity-and-more#features-in-development)，涉及影响客户端渲染应用的重点领域：包括更优的性能分析工具、自动编译effect依赖项、潜在的IDE扩展、片段引用(refs)，甚至包括并发存储方案。这些都是我们长期希望解决的问题，而React编译器和`use`API等新功能使我们终于能够着手处理。

这些功能仍处于早期阶段，在正式发布前可能会有重大变更。您可以在我们构建过程中持续关注，本通讯是获取最新进展的理想渠道。但无需急于学习所有内容——请先给我们研发时间。

当我们更确定某个功能将发布时，会像本次视图过渡和活动状态功能一样，在博客宣布并发布文档。

尝试这些新的实验性功能，并在[博客文章](https://react.dev/blog/2025/04/23/react-labs-view-transitions-activity-and-more)中查看我们正在研究的领域。

## 📱 React-Native {#react-native}

[![Legend List 1.0](/emails/issues/231/legend-list.png)](https://bsky.app/profile/jayz.us/post/3lni5ofbuvs2s)

**[Legend List 1.0](https://bsky.app/profile/jayz.us/post/3lni5ofbuvs2s)**

优化长列表性能是个难题，React Native 的旧架构因其异步布局计算机制反而增加了复杂度。我们曾见证 Margelo 放弃他们的 `WishList` 项目。
Legend List 现已推出 1.0 版本，不仅承诺比 `FlashList` 更快的性能，还能处理"复杂场景"：双向无限列表、动态尺寸项……同时兼容旧架构（尽管性能稍低且可能存在显示瑕疵）。

正如 Seb 上周提到的，[`FlashList` v2 版本](https://github.com/Shopify/flash-list/tree/new-rlv-prototype)也在开发中，使用会更简便（例如无需指定 estimatedItemSize）。

值得庆幸的是，这些库都沿用了 `FlatList` 的 API 设计，切换起来非常方便 🙂。
