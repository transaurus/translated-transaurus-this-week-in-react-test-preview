[**React Aria 发布于5月1日**](https://react-spectrum.adobe.com/releases/2024-05-01.html)

Adobe 工程师近期为其无样式、无障碍的 React 组件库发布了多项改进，主要亮点包括：

- 子菜单功能，详见[专题博客](https://react-spectrum.adobe.com/blog/creating-a-pointer-friendly-submenu-experience.html)

- TreeView alpha 版
- 拖放区 + 文件触发器
- 类型化客户端路由集成
- onAction/isDisabled 项API
- 更优的 tree shaking（体积缩小约 4 倍）

[![Expo SDK 51](/emails/issues/184/expo51.jpg)](https://expo.dev/changelog/2024/05-07-sdk-51)

[**Expo SDK 51**](https://expo.dev/changelog/2024/05-07-sdk-51)

紧接 React Native 0.74 发布后，Expo SDK 51 正式推出。此版本主要推进内部架构升级，并将部分实验性包提升为默认配置。快速回顾本通讯已涵盖内容：大多数 Expo 模块支持新架构（仍存在[已知问题](https://docs.expo.dev/guides/new-architecture/#known-issues-in-expo-sdk-libraries)）、无桥接支持、Apple 隐私清单支持、相机和 SQLite 包的"next"版本现为默认选项、Fingerprint 结束实验阶段。此外新增 expo-symbols 包（集成 Apple SF Symbols 库，仅限 iOS）、全新 expo-video 库、支持 URL 中 # 分段的 Expo Router 3.5 版本，以及 EAS Update 改进（如逐步推送更新的 Web UI）。预计即将召开的 React Conf 将有关于 Expo Router（RSC 👀）的重大公告，敬请关注！另一重要变化：从 51 版开始，Expo Go 仅支持单一 Expo SDK 版本。此决策使 Expo Go 维护更简便，且多数场景本应使用 Expo Dev Client。
