## ⚛️ React {#react}

[![Josh Comeau - CSS `:has` 的不可否认实用性](/emails/issues/200/css-has.jpg)](https://www.joshwcomeau.com/css/has/)

**面向 React 开发者的 CSS `:has`**

本周有两篇关于 CSS `:has` 选择器的精彩文章，探讨 React 开发者如何利用它简化/删除 JS 代码并提升性能。该 CSS 新特性支持率已达92%，但无法通过 polyfill 实现，目前建议用于细节优化和渐进增强。

请按以下顺序阅读：

- 📜 [Josh Comeau - CSS `:has` 选择器的不可否认实用性](https://www.joshwcomeau.com/css/has/)：一篇优秀的交互式入门文章，包含与React开发者相关的评论和示例，例如使用数据属性（而非`useEffect`）在模态框打开时阻止全局页面滚动。
- 📜 [Nadia Makarevich - 用CSS `:has`选择器替代React代码](https://www.developerway.com/posts/replacing-react-with-css)：提供更多基于React的有趣示例，例如处理焦点状态、根据输入状态高亮表格行或表单域。

能力越大责任越大。请保持React组件的封装性，尽可能避免样式泄漏到组件边界之外。

## 📱 React-Native {#react-native}

上周，弗罗茨瓦夫成为React Universe的舞台，重大公告令React Native社区振奋。其中最引人注目的是React Native 0.76的发布计划，预计10月底推出。此版本包含重大改进：新架构（New Architecture）将默认启用，并引入完全重构的React Native DevTools。现已提供[0.76候选版本](https://github.com/facebook/react-native/releases/tag/v0.76.0-rc.0)\!

[![React Native开发工具](/emails/issues/200/react-native-devtools.jpg)](https://www.youtube.com/watch?v=b48Lax2-jOQ&t=4350s)

- 🎥 [Alex Hunt - 全新React Native调试器（代号"Fusebox"）](https://www.youtube.com/watch?v=b48Lax2-jOQ&t=4350s)
  React Native的调试工具长期处于碎片化状态，而新版"React Native DevTools"通过基于Chrome DevTools协议（CDP）的后端提供统一体验。该架构获得IDE和Chrome DevTools支持，承诺更可靠的断点和稳定的热重载。它同时支持新旧架构，将成为React Native应用的终极调试方案。通过Chrome直接调试Hermes的功能将被淘汰，但性能和网络检测功能仍在开发中。

[![Hermes性能图表](/emails/issues/200/hermes-perf.jpg)](https://www.youtube.com/watch?v=pfeTPSai6vc&t=8702s)

- 🎥 [Tzvetan Mikov - Hermes: 通过运行时字节码翻译实现更佳性能](https://www.youtube.com/watch?v=pfeTPSai6vc&t=8702s)  
  去年在React Native Europe大会推出的静态Hermes引擎，通过挑战现有JavaScript引擎标准引发了广泛关注。该引擎仍处于开发阶段，采用创新方法：通过类型注解分析将JavaScript编译成本机机器码或类型化字节码，具备达到C++级别性能的潜力。但该工作流存在局限性：空中下载（OTA）更新功能将失效，当前Babel等JavaScript构建工具无法保留类型注解，且仍有大量未类型化代码在使用。为解决这些问题，Hermes团队推出了运行时字节码翻译技术，承诺将本机编译性能提升高达12倍。这些创新将在Hermes v2中提供，待Meta完成全面内部测试后发布。
