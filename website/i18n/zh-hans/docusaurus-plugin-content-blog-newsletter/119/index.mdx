## ⚛️ React {#react}

**[Vercel OG图像生成：在边缘快速创建动态社交卡片图像](https://vercel.com/blog/introducing-vercel-og-image-generation-fast-dynamic-social-card-images)**

Vercel推出全新高效解决方案，可在Vercel边缘函数中生成动态社交卡片。其优势在于：无需额外配置即可使用JSX、React无状态组件和Tailwind。强烈推荐[这个技术解析](https://twitter.com/shuding_/status/1579607964549513217)，非常有趣！他们甚至使用了React Native的布局引擎Yoga 🤭

**[React渲染的未来](https://prateeksurana.me/blog/future-of-rendering-in-react/)**

图文并茂的技术文章，详细解析了React 18及更高版本的渲染模式。涵盖SSR流式传输、Suspense、React lazy以及Server Components功能标志实现等核心机制。

**[解决React项目的常见痛点](https://dinosaurs-with-jetpacks.com/posts/react-problems)**

主张将JavaScript应用与框架（React等）解耦，倡导回归简洁开发范式。包含部分可能引发争议的激进观点。

**[在Storybook中使用Material UI](https://storybook.js.org/blog/material-ui-in-storybook/)**

本教程包含特别实用的技术方案：通过复用底层UI库的TypeScript类型，自动化生成Storybook控件。在属性透传场景中尤为高效，甚至支持`Pick`和`Omit`等辅助类型。

- 👤 Sathya Gunasekaran (ex V8) [joins the React core team](https://twitter.com/_gsathya/status/1578816177371250688), most likely to [work on the React-Forget compiler](https://twitter.com/Huxpro/status/1578865666010263552)
- 👥 [Next.js Conf 2022 speakers](https://twitter.com/vercel/status/1578401243243741185)
- 📖 Beta docs: [React.Children](https://twitter.com/dan_abramov/status/1578457525628047360) + [useId](https://twitter.com/dan_abramov/status/1577283961763467265) + [createElement](https://twitter.com/dan_abramov/status/1577463721193897985) + [useImperativeHandle](https://twitter.com/dan_abramov/status/1577717226269966336)
- 📖 [Next.js on Vercel docs improvements](https://twitter.com/leeerob/status/1577788730349436928)
- 🌶 [React and Next.js is DEAD — Something New is (Finally) Replacing It (For Good)](https://javascript.plainenglish.io/react-and-next-js-is-dead-something-new-is-finally-replacing-it-for-good-c792c48806f6): explains how the Qwik framework presents a superior model to React. Click-bait title, but really interesting article.
- 📜 [Build a React Custom Renderer - Part 1](https://www.markcodes.dev/posts/build-react-custom-renderer-part-1): new series that shows how to create a React renderer. For now it's quite easy to follow, but it does not yet support updates.
- 📜 [Remix, React, and State](https://blog.jim-nielsen.com/2022/remix-react-and-state/): gives a mental model for Remix that reconciles the React state in an end-to-end / fullstack way.
- 📜 [How to write your own state management library](https://judehunter.dev/blog/how-to-write-your-own-state-management-library): useful to understand how a state management library works, and how to implement a selector system.
- 📜 [How to fetch data in React with performance in mind](https://www.developerway.com/posts/how-to-fetch-data-in-react): explains why data-fetching in React is not as simple as it seems. performance is related to UX/perception. Explain waterfalls, browser request limits...
- 📜 [Avoid impossible UI states with React, Typescript and xState](https://whereisthemouse.com/avoid-impossible-ui-states-with-react-typescript-and-xstate): TypeScript's discriminated union-types avoid impossible states, and XState avoids impossible state transitions.
- 📜 [I changed my mind about writing new JavaScript frameworks](https://whitep4nth3r.com/blog/write-a-new-javascript-framework/): there are only 4 frameworks that support both SSG + SSR: Eleventy, Next.js, Gatsby, Redwood.
- 📜 [How to Use Sandpack for Code Demos](https://www.ctnicholas.dev/articles/how-to-use-sandpack-for-code-demos): article [updated with the latest Sandpack features](https://twitter.com/ctnicholasdev/status/1577683748878884864), this Codesandbox tool to create live code playgrounds. Used by React beta docs, Josh Comeau...
- 📦 [Tremor 1.0](https://www.tremor.so/): a library of React UI components to create dashboard analytics.
- 🐦 [W3C Trusted Types + React + XSS](https://twitter.com/sebastienlorber/status/1579485631260286977): a new HTML proposal could provide a native solution to prevent XSS vulnerabilities, and perhaps eventually remove the need for `dangerouslySetInnerHTML`?
