## ⚛️ React {#react}

[![useSignal() is the Future of Web Frameworks](/emails/issues/135/signals.jpg)](https://www.builder.io/blog/usesignal-is-the-future-of-web-frameworks)

**[`useSignal()`是Web框架的未来](https://www.builder.io/blog/usesignal-is-the-future-of-web-frameworks)**

Misko（Qwik和AngularJS的创建者）强调了signals的价值，并将其与React的工作机制进行了对比。React直接使用状态值，而signals使用getter函数，在调用该getter时创建订阅关系。

Signals能实现细粒度的自动渲染优化，无需记忆化处理。那些仅传递属性的中间组件无需重新渲染。

本文发表于[Angular宣布](https://twitter.com/angular/status/1625939902046117890)计划采用signals的次日（现已有[演示](https://twitter.com/sarah_edo/status/1628065696247857152)）。

Andrew Clark已[评论](https://twitter.com/acdlite/status/1626590880126889984)了在React中引入signals的可能性。他目前更倾向于押注[React Forget](https://www.youtube.com/watch?v=lGEMwh32soc)编译器，我们应能通过React Labs文章很快获得进展。我认为React逆势坚持[探索现有方向](https://twitter.com/acdlite/status/1625599973420154881)是明智之举。

[![宣布 Sandpack 2.0 及面向任意浏览器的 Node.js 运行时](/emails/issues/135/sandpack2.jpg)](https://codesandbox.io/blog/announcing-sandpack-2)

**[宣布 Sandpack 2.0 及面向任意浏览器的 Node.js 运行时](https://codesandbox.io/blog/announcing-sandpack-2)**

CodeSandbox 宣布了 Sandpack 的新版本，这是一个可在浏览器中创建交互式实时编程体验的工具（代码编辑器 + 预览）。Sandpack 通过专用包与 React 良好集成。v1 版本已被用于新版 React beta 文档站点，Josh Comeau 及许多其他开发者已在使用。

此 v2 版本基于新的 NodeBox 技术，可在浏览器中运行 Node.js。能够运行 Next.js、Vite、Astro 及其他技术栈。

这让人联想到 StackBlitz 的技术，其刚刚[宣布了 WebContainers API](https://blog.stackblitz.com/posts/webcontainer-api-is-here/)。Sandpack 文章解释了 NodeBox 与 WebContainers 技术的差异。
