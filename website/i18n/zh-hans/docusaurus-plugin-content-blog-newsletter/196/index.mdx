## 📱 React Native {#react-native}

[![React Native WebGPU](/emails/issues/196/rn-webgpu.jpg)](https://www.youtube.com/watch?v=i-DGY98cTjk)

**🎥 [React Native WebGPU](https://www.youtube.com/watch?v=i-DGY98cTjk)**

React Native WebGPU 正式推出，提供统一 API 以在 React Native 中利用 GPU 进行高级图形与计算！通过复用 Web API 标准，我们能够轻松集成现有 WebGPU 代码示例和库（如 ThreeJS）到 React Native 中。
除了图形处理，React Native WebGPU 还可用于矩阵计算等任务，使其在设备端机器学习算法中颇具潜力。该项目采用 Nitro Modules 构建，实现 TypeScript 到 C++ 的无缝互操作。目前处于技术预览阶段，预示着激动人心的进展！

---

**React Native 模块的未来**

React Native 开发者迎来激动人心的时刻！Marc Rousavy 正通过 NitroModules 革新原生代码编写方式 📦 [其在速度测试中远超现有解决方案](https://github.com/mrousavy/NitroBenchmarks)。他在 VisionCamera 中遇到了 JSI 的某些限制。该项目在 C++ 端已趋稳定，并已用于多个项目如 🐦 [react-native-unistyles](https://x.com/jpudysz/status/1822541758137303233) 和 react-native-webgpu，Swift 和 Kotlin 绑定仍在开发中。

👀 [与此同时，一项推动在 TurboModules 中使用 Swift 的新提案已落地](https://github.com/okwasniewski/discussions-and-proposals/blob/22edd2c7853aaf2ee654b57f4521e43d668103fb/proposals/0000-swift-turbo-modules.md)。这些变更将通过摆脱 Objective-C 和 C++ 来简化原生模块开发。随着这些更新，构建原生模块将大幅优化！
