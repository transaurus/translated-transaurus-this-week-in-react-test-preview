## ⚛️ React {#react}

[![Next.js 15.2](/emails/issues/224/next.jpg)](https://nextjs.org/blog/next-15-2)

[**Next.js 15.2**](https://nextjs.org/blog/next-15-2)

Vercel发布Next.js新次版本，带来显著的开发者体验提升：

- 重构错误UI和改进堆栈跟踪：利用React全新[`captureOwnerStack()`](https://react.dev/reference/react/captureOwnerStack) API打造调试体验
- 流式元数据：异步元数据不再阻塞页面渲染或客户端路由切换
- Turbopack性能优化：编译速度提升，内存占用降低
- React视图过渡(实验性)：支持React新视图过渡API
- Node.js中间件(实验性)：支持在中间件中使用Node.js运行时

[**TanStack Form 1.0**](https://tanstack.com/blog/announcing-tanstack-form-v1)

历经多年开发的TanStack新库正式发布稳定版，公告博文重点介绍以下特性：

- 框架无关的无头组件：支持5种框架，文中示例使用React
- 极致类型安全：`<form.Field>` API设计精妙
- 模式验证：支持Zod、Valibot、ArkType等标准模式
- 异步验证、内置防抖、AbortSignal中断机制
- 同构多平台：客户端/服务端共享验证逻辑

## 📱 React-Native {#react-native}

[![Lynx：释放原生能力](/emails/issues/224/lynx.jpg)](https://lynxjs.org/blog/lynx-unlock-native-for-more)

[**Lynx：释放原生能力**](https://lynxjs.org/blog/lynx-unlock-native-for-more)

字节跳动近日发布了其原生跨平台技术 Lynx，该技术已广泛应用于包括 TikTok 在内的生产级应用！通过 React 绑定，它成为 React Native 的有力竞争者，并预期支持其他框架。其架构设计颇具亮点：采用双线程模型和多种运行时环境，支持部分 React Native 尚未实现的 CSS 和 Web API，并提供便捷的内置 JSX 元素如 `<view>`。

初始功能集令人印象深刻，但也存在值得商榷的设计：如采用 React 17/Preact、在 Web 端使用自定义元素等。这项技术能否兑现承诺并与现有方案抗衡，尚需时间验证。无论如何，良性竞争终将推动生态发展。
