## ⚛️ React {#react}

[![Pigment CSS](/emails/issues/177/pigment.jpg)](https://github.com/mui/material-ui/tree/master/packages/pigment-css-react)

[**Pigment CSS**](https://github.com/mui/material-ui/tree/master/packages/pigment-css-react)

MUI 团队正在开发 Pigment，这是用于替代 Material UI v6 中 Emotion 的零运行时 CSS-in-JS 方案。首个版本已发布至 npm，包含 Next.js 的 `withPigment()` 插件。Olivier 在 🐦 [推文](https://twitter.com/olivtassinari/status/1765719897152127315) 中解释了创建新方案的动机及其与现有方案（Panda、StyleX、Tailwind）的对比。不同于其他方案，Pigment 将更便于现有 Emotion 和 Styled-Components 用户迁移。

[![Storybook 8](/emails/issues/177/storybook.jpg)](https://storybook.js.org/blog/storybook-8/)

[**Storybook 8**](https://storybook.js.org/blog/storybook-8/)

Storybook 的新主版本包含多项改进：

- 实验性 React Server Components 支持
- 更快的 React 控件自动生成（通过 `react-docgen`）
- "测试模式"下更快的静态构建（CLI 参数 `--test`）
- 内置可视化测试（通过 Chromatic）
- 重构的 Vite 支持，Vitest，Vite 5
- 优化的移动端/桌面端 UI/UX
